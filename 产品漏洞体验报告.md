# 企业级私有化多模态 RAG 系统 —— 零基础用户体验漏洞报告

> 角色：首次接触该项目的零基础用户  
> 目的：完全按照 README 与文档指引，从零开始体验部署、配置、使用全流程，记录发现的各类产品漏洞与体验问题  
> 体验时间：2026-06-17  
> 体验环境：Windows 11 + Git Bash，Docker 已预装  

---

## 一、总体结论

从“完全不懂”的用户视角体验后，发现该项目在**部署可用性、安全基线、核心功能实现、文档一致性、前端完整性、监控可观测性**六个维度均存在大量问题。 README 中描述的“一键启动”“五级权限穿透”“多模态 RAG”“企业级安全”等核心卖点，在实际体验中大多**无法直接跑通或仅为空壳实现**。本次共整理出 **100 条可验证的漏洞/体验问题**，其中致命/严重级别占比较高，建议优先修复部署阻塞与安全基线问题后再对外发布。

## 三、TOP 10 最需优先修复的问题

按“阻塞程度 + 安全风险 + 用户影响”排序，建议优先处理以下 10 条：

1. **README 中 `backend/.env.example` 路径错误**（#1）：用户第一步就失败，必须修正为 `.env.example` 或创建 `backend/.env.example`。
2. **默认账号不存在**（#86）：文档说 `admin/admin123`，数据库无此用户，需补充初始化种子或修正文档。
3. **默认 JWT 密钥硬编码**（#21）：未修改配置时任何人可伪造 Token，必须强制从环境读取。
4. **知识库/用户群/关键词接口无认证**（#24-#26）：核心管理接口完全开放，必须接入认证与权限校验。
5. **CORS `*` + credentials**（#23）：存在严重跨站风险，必须改为白名单并关闭 credentials 通配。
6. **没有自动数据库迁移**（#5）：首次启动数据库为空，所有业务接口 500，需增加 migration 服务或启动脚本。
7. **RabbitMQ guest 远程连接受限**（#6）：Worker 和 Backend 无法连接队列，需改为自定义用户或关闭 loopback 限制。
8. **Alertmanager 配置错误无法启动**（#76）：监控告警完全失效，需修正 `name: null`。
9. **后端 `/metrics` 404**（#13 / #77）：Prometheus 无法采集后端指标，需确保代码与镜像一致并暴露 `/metrics`。
10. **视频/音频/图片/链接摄取为空壳**（#41-#44 / #91）：核心卖点无法使用，需补齐实现或降低文档宣传口径。

## 四、体验总结

作为零基础用户，按照 README 完成“快速开始”的完整路径为：

1. `cp backend/.env.example backend/.env` → **失败**（文件不存在）。
2. 跳过配置直接 `docker compose up -d` → 能启动容器，但数据库为空、RabbitMQ 连接受限（取决于环境）、Alertmanager 启动失败、`/metrics` 不可用。
3. 访问 `http://localhost:3002` → 前端页面可打开，但用 `admin/admin123` 登录 → **失败**（用户不存在）。
4. 注册账号后进入系统 → 可创建知识库、上传文档，但视频/音频/图片/链接无法真正解析；搜索与问答依赖 mock 服务，效果有限。
5. 进入权限管理 → 可配置规则，但五级权限穿透实际拦截逻辑缺失。
6. 查看监控 → Prometheus 后端 target down、Grafana 看不到预置 dashboard、Alertmanager 未启动。

**结论**：当前项目更像是一个“技术方案 + 部分代码骨架”，距离 README 所宣称的“企业级私有化多模态 RAG 系统”尚有显著差距。建议先补齐部署可用性、安全基线和核心功能实现，再统一修订文档与前端文案，避免文档与实现脱节。
