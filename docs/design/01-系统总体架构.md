## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **模态原生保留** | 视频、图片保留原始像素/帧信息参与检索，不做有损的纯文本转换 |
| **权限前置过滤** | 所有检索请求先经过权限过滤，再执行向量搜索，确保零泄露 |
| **Pipeline可编排** | 各模态RAG Pipeline模块化设计，支持动态插拔和参数调优 |
| **统一与专用并存** | 底层检索引擎统一，上层处理逻辑专用化 |
| **渐进式实施** | MVP -> 扩展 -> 优化，分阶段验证价值 |
| **纵深安全防御** | 五级权限穿透 + 关键词敏感控制 + API安全网关，多层拦截 |

### 3.1 用户查询完整流程

```
用户查询 "Q3产品规划的销售额目标是多少？"
    |
    v
+----------+    +----------+    +----------+    +----------+    +----------+
| API网关  | -> | 鉴权中心 | -> | 权限服务 | -> | 查询处理 | -> | 检索引擎 |
+----------+    +----------+    +----------+    +----------+    +----------+
                                                                   |
                    +----------------------------------------------+
                    v
         +-----------------------------------------+
         | Step 1: 权限感知向量检索 (Milvus expr)   |
         |  - 文件类型过滤 + 文档黑名单 + 标签黑名单 |
         |  - 召回200候选Chunk                      |
         +-----------------------------------------+
                    |
                    v
         +-----------------------------------------+
         | Step 2: 批量权限缓存过滤 (Redis)         |
         |  - 文档级 + 标签级权限检查                |
         |  - 剩余120个通过, 40个PARTIAL            |
         +-----------------------------------------+
                    |
                    v
         +-----------------------------------------+
         | Step 3: 字段级精确过滤                   |
         |  - Excel列级 / Word段落级权限            |
         |  - 剩余112个                            |
         +-----------------------------------------+
                    |
                    v
         +-----------------------------------------+
         | Step 4: 关键词级别过滤                   |
         |  - Chunk.max_keyword_level vs user_level |
         |  - 拦截越级内容, 记录审计日志            |
         +-----------------------------------------+
                    |
                    v
         +-----------------------------------------+
         | Step 5: 权限感知Re-rank + 分组           |
         |  - 按权限级别分组, 组内Re-rank排序       |
         +-----------------------------------------+
                    |
                    v
         +-----------------------------------------+
         | Step 6: 权限分块压缩                     |
         |  - 同权限级别组内压缩                    |

         +-----------------------------------------+
         | Step 7: API安全评估                      |
         |  - L4->本地处理 / L3->脱敏 / L2->直接    |

                    |

         +-----------------------------------------+

         +-----------------------------------------+
```

## 4. 服务拆分

```
backend/
├── api-gateway/              # API网关 (Kong)
├── services/
│   ├── user-service/         # 用户与认证服务
│   │   └── 用户管理、OAuth2+JWT鉴权、用户群管理
│   ├── kb-service/           # 知识库管理服务
│   │   └── 知识库CRUD、配置管理
│   ├── doc-service/          # 文档管理服务
│   │   └── 文档上传、状态管理、元数据
│   ├── permission-service/   # 权限控制服务 (核心)
│   │   ├── 用户权限计算
│   │   ├── 用户群权限计算
│   │   ├── 字段级权限校验
│   │   ├── 权限缓存管理
│   │   └── 权限变更事件发布
│   ├── tag-service/          # 标签管理服务
│   │   └── 标签CRUD、自动标签分类
│   ├── ingest-service/       # 文档摄取处理服务
│   │   ├── parsers/          # 各类文档解析器
│   │   ├── chunkers/         # 分块策略
│   │   ├── embedders/        # 向量化适配器
│   │   └── keyword_annotator/# 关键词标注器
│   ├── search-service/       # 检索引擎服务 (核心)
│   │   ├── retrievers/       # 各类检索器
│   │   ├── fusion/           # 结果融合(RRF)
│   │   ├── reranker/         # 重排序(两阶段)
│   │   └── permission_filter/# 权限过滤器(L1/L2/L3)
│   ├── generate-service/     # 生成服务
│   │   ├── secure_api_gateway/  # API安全网关
│   │   ├── context_compressor/  # 权限感知压缩
│   │   └── response_validator/  # 返回结果校验
│   ├── keyword-service/      # 关键词敏感控制服务
│   │   ├── keyword_manager/     # 全局关键词配置
│   │   └── keyword_detector/    # 内容检测引擎
│   ├── eval-service/         # 评测服务
│   ├── audit-service/        # 审计日志服务
│   └── notification-service/ # 通知服务
├── shared/                   # 公共库
│   ├── models/               # 数据模型定义
│   ├── clients/              # 客户端封装
│   └── utils/                # 工具函数
└── workers/                  # 异步任务Worker
    ├── ingest-worker/        # 文档摄取Worker
    ├── embed-worker/         # 向量化Worker
    ├── index-worker/         # 索引更新Worker
    └── permission-sync-worker/  # 权限同步Worker
```
