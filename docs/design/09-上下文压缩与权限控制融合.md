## 1. 背景与问题定义

### 1.1 系统背景

在企业级私有化多模态RAG系统中，检索结果需经**上下文压缩**（如 LLMLingua、LongLLMLingua）后送入大语言模型。压缩的核心目标是：在Token预算内保留最相关的语义信息，降低推理成本并减少"Lost in the Middle"现象。

然而，企业知识库天然带有**多级权限控制**（L0-L4），压缩操作可能破坏原有的权限标记，导致敏感信息越界泄露。

### 1.2 核心矛盾

```
┌─────────────────────────────────────────────────────────────┐
│                        核心矛盾                              │
├─────────────────────────────────────────────────────────────┤
│  上下文压缩: 追求语义密度最大化，可能丢弃"低信息量"的标记      │
│         ↕ 冲突                                               │
│  权限控制: 要求标记绝对不可丢失，边界绝对不可跨越              │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 压缩策略概述

| 压缩策略 | 原理 | 适用场景 | 对权限的潜在威胁 |
|---------|------|---------|---------------|
| **LLMLingua** | 基于困惑度迭代删除低信息Token | 单轮问答 | 删除权限标记Token |
| **LongLLMLingua** | 分块压缩 + 重排序 | 长文档检索 | 截断打断权限标记 |
| **选择性上下文** | 基于熵过滤无关句子 | 多文档拼接 | 过滤掉权限边界句 |
| **Re-rank重排序** | 按相关性重排Chunk | 多路召回 | 打散同权限分组 |

## 2. 风险分析

### 2.1 风险总览

```
┌──────────────────────────────────────────────────────────────────────┐
│                        四大核心风险                                   │
├──────────────┬───────────────────────────────────────────────────────┤
│   R1         │  压缩导致权限标记丢失                                  │
│   R2         │  多轮对话历史泄露                                      │
│   R3         │  截断丢失标记 / 打断权限边界                           │
│   R4         │  重排序打散权限分组                                    │
└──────────────┴───────────────────────────────────────────────────────┘
```

### 2.2 R1: 压缩导致权限标记丢失

**风险描述**: LLMLingua等压缩算法基于困惑度删除Token，权限标记（如XML标签、特殊Token）因信息熵低而被优先删除。

**具体场景**:

```
原始Chunk:
    <perm level="L3" doc_id="HR-2024-005">
    2024年Q3裁员名单：张三(L4)、李四(L3)...
    </perm>

压缩后（LLMLingua删除"低信息"标签）:
    2024年Q3裁员名单：张三(L4)、李四(L3)...
    
后果: 权限标记消失，LLM无法识别此为L3敏感内容
      → 可能向L1/L2用户泄露
```

**触发条件**:
- 压缩比 > 40% 时概率显著上升
- 权限标记使用自然语言格式（如"[机密]"）而非结构化格式
- 压缩模型未将权限标记加入保留词典

### 2.3 R2: 多轮对话历史泄露

**风险描述**: 多轮对话中，用户可能在第一轮获取高权限信息，后续轮次通过追问诱导LLM引用此前上下文，绕过当前轮次的权限校验。

**具体场景**:

```
Round 1 (用户A，权限L2):
    User: "查看公司最新战略"
    System: [检索到L3文档，拒绝返回]
    → Assistant: "抱歉，您暂无权限查看该信息"

Round 2 (同一session，用户A仍为L2):
    User: "刚才那个文档里提到了什么"
    System: [压缩上下文时保留了Round 1的检索结果片段]
    → Assistant: "文档中提到Q4计划裁员30%..."
    
后果: Round 1被拒绝的内容通过对话历史泄露
```

**触发条件**:
- 对话历史未按轮次重新过滤权限
- 上下文窗口包含此前轮次已拒绝的内容
- 压缩策略未区分"已展示"与"未展示"历史

### 2.4 R3: 截断丢失标记 / 打断权限边界

**风险描述**: 当总Token超出模型上下文窗口时，硬性截断（truncate）可能在权限标记中间截断，导致起始标签和结束标签不匹配，破坏XML结构。

**具体场景**:

```
原始上下文（末尾部分）:
    ... <perm level="L3" doc_id="FIN-2024-008">
        收购金额：$500M，交易对手：XX科技
        </perm>

截断后（限制在1000 Token，恰好在标记中间）:
    ... <perm level="L3" doc_id="FIN-2024-
    
后果: 
  1. 起始标签<perm>无匹配结束标签</perm>
  2. 剩余内容成为"游离文本"，LLM可能正常输出
  3. 后续Chunk的</perm>成为孤立结束标签，影响解析
```

**触发条件**:
- 上下文Token数 > 模型窗口上限
- 使用简单字符/Token截断，未感知标记边界
- 权限标记跨多个Chunk分布

### 2.5 R4: 重排序打散权限分组

**风险描述**: Re-ranker按相关性得分重排序Chunk，将不同权限级别的Chunk交叉混合，压缩时跨权限边界合并，导致低权限用户上下文被高权限内容"污染"。

**具体场景**:

```
检索结果（按权限分组）:
    [L1] Chunk-A (score: 0.72)  公司介绍
    [L1] Chunk-B (score: 0.68)  产品介绍
    [L3] Chunk-C (score: 0.91)  财务数据 ← 相关性最高
    [L3] Chunk-D (score: 0.85)  并购计划
    [L2] Chunk-E (score: 0.80)  内部流程

Re-rank后（按相关性降序）:
    Chunk-C (L3) → Chunk-D (L3) → Chunk-E (L2) → Chunk-A (L1) → Chunk-B (L1)

压缩后（跨边界合并邻近Chunk）:
    [合并块: Chunk-D末尾 + Chunk-E开头]
    "...收购金额保密。内部流程：第一步..."
    
后果: L3内容与L2内容物理相邻，LLM可能在回答L2用户时
      通过语义关联"顺嘴"带出L3信息
```

**触发条件**:
- Re-ranker未考虑权限级别分组
- 压缩算法在Chunk边界处合并Token
- 多个权限级别的Chunk相关性得分接近

## 3. 核心设计原则

### 3.1 原则总览

| 原则编号 | 原则名称 | 核心要求 |
|---------|---------|---------|
| **P1** | 权限标记内联不可分 | 权限标记与内容绑定，压缩/截断时标记必须作为原子单元处理 |
| **P2** | 权限边界不跨越 | 不同权限级别的Chunk在压缩和排序中保持物理隔离 |
| **P3** | 历史上下文重新过滤 | 每轮对话独立校验历史可见性，不因历史已存在而放行 |
| **P4** | 截断策略权限感知 | Token截断必须识别标记边界，禁止在权限标记内部截断 |

### 3.2 P1: 权限标记内联不可分

```
非原子模式（危险）:
    [Header] level=L3
    [Content] 敏感内容...
    [Footer] end
    → 压缩可能删除Header/Footer，Content孤立泄露

原子模式（安全）:
    <perm level="L3">敏感内容</perm>
    → 将整个<perm>...</perm>视为不可拆分的压缩单元
      压缩发生在标签内部，标签本身不可删除/截断
```

**实现要点**:
- 权限标记采用XML格式，起始和结束标签配对
- 压缩算法在Token级别识别`<perm`和`</perm>`为保留Token
- 截断算法优先在`</perm>`之后寻找截断点

### 3.3 P2: 权限边界不跨越

```
隔离前（危险）:
    ┌─────────┬─────────┬─────────┐
    │ L1 Chunk│ L3 Chunk│ L1 Chunk│  → 压缩时跨边界，L3污染L1
    └─────────┴─────────┴─────────┘

隔离后（安全）:
    ┌─────────┐  ┌─────────┐  ┌─────────┐
    │ L1 Group│  │ L3 Group│  │ L1 Group│  → 组内压缩，组间不合并
    │(内压缩) │  │(内压缩) │  │(内压缩) │
    └─────────┘  └─────────┘  └─────────┘
```

**实现要点**:
- 压缩前按`perm_level`分组
- 每组独立执行压缩算法
- 组间插入不可压缩的分隔符

### 3.4 P3: 历史上下文重新过滤

```
错误做法（每轮追加，不复核）:
    Round1: [L1结果] → 压缩 → LLM
    Round2: [Round1压缩结果] + [L1结果] → 压缩 → LLM
    → Round1中已过滤的L3内容仍保留在历史中

正确做法（每轮重新过滤）:
    Round1: [L1结果] → 压缩 → LLM
    Round2: [Round1历史重新过滤为L1可见] + [L1结果] → 压缩 → LLM
    → 每轮基于当前用户权限重新计算可见历史
```

**实现要点**:
- 原始对话历史按轮次存储，保留完整权限标记
- `get_visible_history(user_perm)`每轮重新执行过滤
- 压缩操作在过滤后执行，不接触已过滤内容

### 3.5 P4: 截断策略权限感知

```
简单截断（危险）:
    文本: "...<perm level=\"L3\">机密内容</perm>..."
    截断点: 第50个字符，落在 "<perm" 中间
    结果: "...<perm lev" → 标记破坏，内容可能泄露

权限感知截断（安全）:
    候选截断点: 第50个字符
    检测: 落在标记内部 → 向前/向后寻找安全截断点
    安全点: "</perm>"之后第3个字符
    结果: "...<perm level=\"L3\">机密内容</perm> ..."
```

**实现要点**:
- 截断前解析所有`<perm>`标签位置
- 截断点必须落在`</perm>`之后的文本区域
- 无安全截断点时，整组`<perm>...</perm>`向后移动至下一窗口

## 4. 方案1: 权限分块压缩

### 4.1 设计思路

将传统"整体压缩"改为"分组压缩"：按权限级别对Chunk分组，组内使用LLMLingua等算法正常压缩，组间保持隔离，最后按权限级别降序拼接。

### 4.2 PermissionAwareCompressor 实现

```python
from typing import List, Dict, Tuple
from dataclasses import dataclass
import llmlingua


@dataclass
class PermissionChunk:
    """带权限标记的文本块"""
    content: str
    perm_level: str  # "L0", "L1", "L2", "L3", "L4"
    doc_id: str
    chunk_id: str


class PermissionAwareCompressor:
    """
    权限感知压缩器
    
    核心逻辑:
    1. 按 perm_level 分组
    2. 组内使用 LLMLingua 压缩
    3. 组间插入不可压缩分隔符
    4. 按权限级别降序输出（高权限在前，便于后续过滤）
    """
    
    PERM_LEVEL_ORDER = {"L0": 0, "L1": 1, "L2": 2, "L3": 3, "L4": 4}
    
    def __init__(self, base_compressor: llmlingua.PromptCompressor, 
                 target_compression_ratio: float = 0.5):
        self.base_compressor = base_compressor
        self.target_ratio = target_compression_ratio
        
    def compress(self, chunks: List[PermissionChunk], 
                 user_perm_level: str) -> str:
        """
        执行权限感知压缩
        
        Args:
            chunks: 检索到的带权限Chunk列表
            user_perm_level: 当前用户权限级别（如"L2"）
        
        Returns:
            压缩后的安全上下文字符串
        """
        # Step 1: 过滤用户不可见的Chunk
        visible_chunks = self._filter_by_permission(chunks, user_perm_level)
        
        # Step 2: 按权限级别分组
        groups = self._group_by_permission(visible_chunks)
        
        # Step 3: 每组独立压缩
        compressed_groups: Dict[str, str] = {}
        for level, group_chunks in groups.items():
            compressed = self._compress_group(group_chunks, level)
            compressed_groups[level] = compressed
            
        # Step 4: 按权限级别降序拼接
        sorted_levels = sorted(
            compressed_groups.keys(),
            key=lambda x: self.PERM_LEVEL_ORDER[x],
            reverse=True
        )
        
        result_parts = []
        for level in sorted_levels:
            # 插入组级权限标记（不可压缩区域）
            result_parts.append(f"<group level=\"{level}\">")
            result_parts.append(compressed_groups[level])
            result_parts.append(f"</group>")
            
        return "\n".join(result_parts)
    
    def _filter_by_permission(self, chunks: List[PermissionChunk], 
                              user_perm_level: str) -> List[PermissionChunk]:
        """过滤掉用户权限不足的Chunk"""
        user_level_num = self.PERM_LEVEL_ORDER[user_perm_level]
        return [
            c for c in chunks 
            if self.PERM_LEVEL_ORDER[c.perm_level] <= user_level_num
        ]
    
    def _group_by_permission(self, chunks: List[PermissionChunk]) -> Dict[str, List[PermissionChunk]]:
        """按权限级别分组"""
        groups: Dict[str, List[PermissionChunk]] = {}
        for chunk in chunks:
            groups.setdefault(chunk.perm_level, []).append(chunk)
        return groups
    
    def _compress_group(self, chunks: List[PermissionChunk], 
                        level: str) -> str:
        """
        对单个权限组执行压缩
        
        策略:
        - 将组内所有Chunk拼接为一个大文本
        - 使用LLMLingua压缩
        - 压缩前后包裹权限标记（标记本身不参与压缩）
        """
        # 提取纯内容（去掉已有权限标记，后续统一包裹）
        raw_contents = []
        for chunk in chunks:
            # 包裹单个Chunk的权限标记
            wrapped = (
                f'<perm level="{chunk.perm_level}" '
                f'doc_id="{chunk.doc_id}" '
                f'chunk_id="{chunk.chunk_id}">'
                f'{chunk.content}'
                f'</perm>'
            )
            raw_contents.append(wrapped)
            
        group_text = "\n".join(raw_contents)
        
        # 使用LLMLingua压缩（保留XML标签）
        compressed = self.base_compressor.compress_prompt(
            group_text,
            condition_compare=True,
            condition_in_question=['<perm', '</perm>'],  # 强制保留权限标记
            rank_method="longllmlingua",
            use_sentence_level_filter=False,
            context_budget="+100",  # 给标记预留空间
            dynamic_context_compression_ratio=self.target_ratio,
        )
        
        return compressed["compressed_prompt"]
```

### 4.4 组间隔离策略

| 隔离维度 | 策略 | 目的 |
|---------|------|------|
| **压缩隔离** | 每组独立调用LLMLingua | 防止跨权限级别的Token合并 |
| **标记隔离** | 组间插入`<group level="X">` | 明确标识权限边界 |
| **截断隔离** | 截断优先在组间边界执行 | 防止截断在组内破坏语义 |
| **重排序隔离** | 组间不重排序，仅组内优化 | 保持权限级别的物理聚集 |

## 5. 方案2: 多轮对话权限隔离

### 5.1 设计思路

多轮对话的每条历史消息都带有权限上下文。每次构建上下文时，基于当前用户的最新权限重新过滤历史，而非简单复用上轮的压缩结果。

### 5.2 PermissionAwareMessage 实现

```python
from typing import Optional, List, Dict, Any
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum


class MessageRole(Enum):
    USER = "user"
    ASSISTANT = "assistant"
    SYSTEM = "system"
    RETRIEVAL = "retrieval"  # 检索结果（特殊角色）


@dataclass
class PermissionAwareMessage:
    """
    权限感知消息
    
    每条消息记录:
    - 原始内容（完整内容，含所有权限标记）
    - 该消息生成时的用户权限级别
    - 该消息关联的检索结果权限分布
    """
    role: MessageRole
    content: str  # 原始完整内容（含所有权限标记）
    timestamp: datetime
    
    # 权限上下文
    user_perm_at_creation: str  # 消息创建时的用户权限
    max_content_perm: str       # 内容中最高权限级别
    
    # 检索结果元数据（仅RETRIEVAL角色使用）
    retrieved_chunks: List[Dict[str, Any]] = field(default_factory=list)
    
    # 该消息经压缩后的各权限版本（缓存）
    _compressed_versions: Dict[str, str] = field(default_factory=dict, repr=False)
    
    def get_visible_content(self, viewer_perm: str) -> Optional[str]:
        """
        获取指定权限级别可见的内容
        
        逻辑:
        1. 若 viewer_perm >= max_content_perm，返回完整内容
        2. 若 viewer_perm < max_content_perm，过滤掉高权限部分
        3. 若过滤后无内容，返回 None（该消息对该用户不可见）
        """
        from lxml import etree
        
        perm_order = {"L0": 0, "L1": 1, "L2": 2, "L3": 3, "L4": 4}
        viewer_level = perm_order.get(viewer_perm, -1)
        max_level = perm_order.get(self.max_content_perm, 5)
        
        if viewer_level >= max_level:
            return self.content
            
        # 解析XML，过滤高权限节点
        try:
            root = etree.fromstring(f"<root>{self.content}</root>")
            for perm_node in root.findall(".//perm"):
                node_level = perm_order.get(perm_node.get("level", "L0"), 0)
                if node_level > viewer_level:
                    perm_node.getparent().remove(perm_node)
                    
            visible = "".join(root.itertext()).strip()
            return visible if visible else None
        except etree.XMLSyntaxError:
            # 非XML格式，回退到整消息可见性判断
            return None if viewer_level < max_level else self.content
```

    3. 压缩操作在过滤后执行

        self.messages: List[PermissionAwareMessage] = []
        self._compression_cache: Dict[str, str] = {}

        self.messages.append(message)
        # 清空缓存，因为历史变了
        self._compression_cache.clear()
        
    def add_retrieval_result(self, chunks: List[PermissionChunk], 
                             user_perm: str):
        """
        添加检索结果作为特殊消息
        
        检索结果消息的内容为XML格式的原始Chunk，
        实际展示内容在get_visible_history时动态过滤
        """
        # 构建XML格式的检索内容
        parts = []
        max_perm = "L0"
        chunk_metas = []
        
        for chunk in chunks:
            parts.append(
                f'<perm level="{chunk.perm_level}" '
                f'doc_id="{chunk.doc_id}">'
                f'{chunk.content}'
                f'</perm>'
            )
            chunk_metas.append({
                "doc_id": chunk.doc_id,
                "perm_level": chunk.perm_level
            })
            if PermissionAwareCompressor.PERM_LEVEL_ORDER.get(chunk.perm_level, 0) > \
               PermissionAwareCompressor.PERM_LEVEL_ORDER.get(max_perm, 0):
                max_perm = chunk.perm_level
                
        content = "\n".join(parts)
        
        msg = PermissionAwareMessage(
            role=MessageRole.RETRIEVAL,
            content=content,
            timestamp=datetime.now(),
            user_perm_at_creation=user_perm,
            max_content_perm=max_perm,
            retrieved_chunks=chunk_metas
        )
        self.add_message(msg)
        
    def get_visible_history(self, current_user_perm: str,
                            max_tokens: int = 4000,
                            compressor: Optional[PermissionAwareCompressor] = None) -> List[Dict[str, str]]:
        """
        获取当前用户可见的对话历史
        
        处理流程:
        1. 遍历所有历史消息
        2. 每条消息按 current_user_perm 重新过滤
        3. 不可见的消息跳过（不放入历史）
        4. 可见消息按原始顺序排列
        5. 整体压缩（保留权限标记）
        6. 权限感知截断到max_tokens
        """
        visible_messages: List[PermissionAwareMessage] = []
        
        for msg in self.messages:
            # 重新过滤：基于当前用户权限，而非消息创建时的权限
            visible_content = msg.get_visible_content(current_user_perm)
            if visible_content is None:
                continue  # 该消息对当前用户完全不可见
                
            # 创建过滤后的消息副本（不修改原始消息）
            filtered_msg = PermissionAwareMessage(
                role=msg.role,
                content=visible_content,
                timestamp=msg.timestamp,
                user_perm_at_creation=msg.user_perm_at_creation,
                max_content_perm=current_user_perm  # 过滤后最高权限=当前用户权限
            )
            visible_messages.append(filtered_msg)
            
        # 构建标准消息格式
        result = []
        for msg in visible_messages:

                # 检索结果作为system消息注入
                role_str = "system"
                content = f"[检索结果]\n{msg.content}"
            else:
                content = msg.content
            result.append({"role": role_str, "content": content})
            
        # 压缩与截断
        if compressor:
            result = self._compress_history(result, compressor, max_tokens)
        else:
            result = self._safe_truncate(result, max_tokens)
            
        return result
    
    def _compress_history(self, messages: List[Dict[str, str]], 
                          compressor: PermissionAwareCompressor,
                          max_tokens: int) -> List[Dict[str, str]]:
        """对历史进行权限感知压缩"""
        # 将消息列表转为可压缩格式
        # ... 实现略，核心是在组内压缩，保留权限标记
        return messages

                       max_tokens: int) -> List[Dict[str, str]]:

## 6. 方案3: 权限标记内联+不可截断

### 6.1 设计思路

将权限标记设计为XML内联格式，并将`<perm>...</perm>`整体作为**原子单元**。截断算法必须感知标记边界，绝不在标记内部截断。

### 6.2 SecureContextBuilder 实现

```python
import re
from typing import List, Dict, Tuple, Optional
import tiktoken


class SecureContextBuilder:
    """
    安全上下文构建器
    
    核心能力:
    1. 解析并验证XML权限标记的完整性
    2. 实现权限感知截断（safe_truncate）
    3. 构建符合安全规范的最终Prompt
    """
    
    # 匹配 <perm level="X" ...> 和 </perm>
    PERM_OPEN_PATTERN = re.compile(
        r'<perm\s+level="(L[0-4])"\s+([^>]*)>'
    )
    PERM_CLOSE_PATTERN = re.compile(r'</perm>')
    
    # 强制保留Token（不参与压缩）
    RESERVED_TOKENS = {'<perm', '</perm>', 'level=', 'doc_id=', 'chunk_id='}
    
    def __init__(self, model_name: str = "gpt-4"):
        self.tokenizer = tiktoken.encoding_for_model(model_name)
        
    def validate_markup(self, text: str) -> Tuple[bool, List[str]]:
        """
        验证权限标记的XML完整性
        
        Returns:
            (是否有效, 错误列表)
        """
        errors = []
        
        # 使用栈检查标签配对
        stack = []
        pos = 0
        
        while pos < len(text):
            open_match = self.PERM_OPEN_PATTERN.search(text, pos)
            close_match = self.PERM_CLOSE_PATTERN.search(text, pos)
            
            # 找到下一个标签
            next_open = open_match.start() if open_match else float('inf')
            next_close = close_match.start() if close_match else float('inf')
            
            if next_open == float('inf') and next_close == float('inf'):
                break
                
            if next_open < next_close:
                # 遇到起始标签
                level = open_match.group(1)
                attrs = open_match.group(2)
                stack.append({
                    "level": level,
                    "attrs": attrs,
                    "start_pos": open_match.start()
                })
                pos = open_match.end()
            else:
                # 遇到结束标签
                if not stack:
                    errors.append(
                        f"孤立结束标签 at position {close_match.start()}: "
                        f"{text[max(0,close_match.start()-20):close_match.end()+20]}"
                    )
                else:
                    stack.pop()
                pos = close_match.end()
                
        # 未闭合的标签
        for unclosed in stack:
            errors.append(
                f"未闭合标签 at position {unclosed['start_pos']}: "
                f"level={unclosed['level']}"
            )
            
        return len(errors) == 0, errors
    
    def safe_truncate(self, text: str, max_tokens: int) -> str:
        """
        权限感知截断
        
        保证:
        1. 不在 <perm ...> 和 </perm> 内部截断
        2. 若截断点落在标记内，自动调整到最近的 </perm> 之后
        3. 若调整后超出max_tokens，整组<perm>移出当前窗口
        
        算法:
        1. 找到理论截断位置（按Token计数）
        2. 检查该位置是否在任何<perm>...</perm>内部
        3. 若在内部，向前找上一个</perm>之后作为安全截断点
        4. 若安全截断点导致内容过短，向后找下一个</perm>之后
        """
        tokens = self.tokenizer.encode(text)
        
        if len(tokens) <= max_tokens:
            return text
            
        # 理论截断位置（字符级）
        truncate_token_pos = max_tokens
        truncate_char_pos = self._token_to_char_pos(tokens, text, truncate_token_pos)
        
        # 查找所有<perm>区间
        perm_ranges = self._find_perm_ranges(text)
        
        # 检查截断点是否在perm内部
        for start, end in perm_ranges:
            if start < truncate_char_pos < end:
                # 截断点在perm内部，向前找安全点
                safe_pos = self._find_safe_truncate_point(
                    perm_ranges, truncate_char_pos, max_tokens, tokens
                )
                truncate_char_pos = safe_pos
                break
                
        return text[:truncate_char_pos]
    
    def _find_perm_ranges(self, text: str) -> List[Tuple[int, int]]:
        """查找所有<perm>...</perm>的字符区间"""
        ranges = []
        pos = 0
        
        while True:
            open_match = self.PERM_OPEN_PATTERN.search(text, pos)
            if not open_match:
                break
                
            close_match = self.PERM_CLOSE_PATTERN.search(text, open_match.end())
            if not close_match:
                break  # 未闭合，后续validate会报错
                
            ranges.append((open_match.start(), close_match.end()))
            pos = close_match.end()
            
        return ranges
    
    def _find_safe_truncate_point(self, perm_ranges: List[Tuple[int, int]], 
                                   current_pos: int, 
                                   max_tokens: int,
                                   tokens: List[int]) -> int:
        """
        寻找安全截断点
        
        策略: 优先向前找上一个</perm>之后的位置
        若导致内容损失过大（<50% max_tokens），则向后找下一个</perm>之后
        """
        # 向前找: 上一个perm结束后的位置
        forward_safe = 0
        for start, end in perm_ranges:
            if end <= current_pos:
                forward_safe = end
            else:
                break
                
        # 检查向前截断后的Token数
        forward_tokens = len(self.tokenizer.encode(text[:forward_safe]))
        
        if forward_tokens >= max_tokens * 0.5:
            # 向前截断可保留至少50%内容，接受
            return forward_safe
            
        # 向后找: 包含当前perm，在其结束后截断
        backward_safe = current_pos
        for start, end in perm_ranges:
            if start <= current_pos <= end:
                backward_safe = end
                break
                
        # 检查是否超出max_tokens过多（允许10%溢出，由后续处理）
        backward_tokens = len(self.tokenizer.encode(text[:backward_safe]))
        if backward_tokens <= int(max_tokens * 1.1):
            return backward_safe
            
        # 两边都不理想，选择向前安全点
        return forward_safe
    
    def _token_to_char_pos(self, tokens: List[int], text: str, token_pos: int) -> int:
        """将Token位置转为字符位置"""
        partial_tokens = tokens[:token_pos]
        partial_text = self.tokenizer.decode(partial_tokens)
        return len(partial_text)
    
    def build_secure_context(self, chunks: List[PermissionChunk],
                             user_perm: str,
                             max_tokens: int) -> str:
        """
        构建安全上下文（完整流程）
        
        1. 按权限过滤
        2. 包裹XML标记
        3. 验证标记完整性
        4. 安全截断
        """
        # 过滤
        perm_order = {"L0": 0, "L1": 1, "L2": 2, "L3": 3, "L4": 4}
        user_level = perm_order.get(user_perm, 0)
        
        parts = []
        for chunk in chunks:
            chunk_level = perm_order.get(chunk.perm_level, 5)
            if chunk_level > user_level:
                continue
            parts.append(
                f'<perm level="{chunk.perm_level}" '
                f'doc_id="{chunk.doc_id}" '
                f'chunk_id="{chunk.chunk_id}">'
                f'{chunk.content}'
                f'</perm>'
            )
            
        context = "\n".join(parts)
        
        # 验证
        valid, errors = self.validate_markup(context)
        if not valid:
            raise ValueError(f"权限标记验证失败: {errors}")
            
        # 截断
        return self.safe_truncate(context, max_tokens)
    
    def safe_truncate_messages(self, messages: List[Dict[str, str]], 
                               max_tokens: int) -> List[Dict[str, str]]:
        """
        对消息列表进行安全截断
        
        策略:
        - 优先截断最早的非系统消息
        - 每条消息内部使用safe_truncate
        - 绝不截断在<perm>标记内部
        """
        result = []
        remaining_tokens = max_tokens
        
        # 系统消息优先保留
        system_msgs = [m for m in messages if m["role"] == "system"]
        other_msgs = [m for m in messages if m["role"] != "system"]
        
        for msg in system_msgs:
            truncated = self.safe_truncate(msg["content"], remaining_tokens)
            tokens_used = len(self.tokenizer.encode(truncated))
            result.append({"role": msg["role"], "content": truncated})
            remaining_tokens -= tokens_used
            
        # 其他消息从最新开始保留
        for msg in reversed(other_msgs):
            if remaining_tokens <= 0:
                break
            truncated = self.safe_truncate(msg["content"], remaining_tokens)
            tokens_used = len(self.tokenizer.encode(truncated))
            result.insert(len(result), {"role": msg["role"], "content": truncated})
            remaining_tokens -= tokens_used
            
        return result
```

## 7. 方案4: 权限感知Re-rank+压缩顺序

### 7.1 设计思路

传统Re-ranker只按相关性排序，打散权限分组。权限感知Re-ranker在**组内重排序**，组间保持权限级别顺序，压缩时按权限级别从高到低依次处理。

### 7.2 PermissionAwareReranker 实现

```python
from typing import List, Dict, Tuple
from dataclasses import dataclass


@dataclass
class ScoredChunk(PermissionChunk):
    """带相关性得分的Chunk"""
    relevance_score: float  # Re-ranker给出的相关性得分
    retrieval_score: float  # 向量检索原始得分


class PermissionAwareReranker:
    """
    权限感知重排序器
    
    策略:
    1. 按权限级别分组
    2. 每组内部按相关性得分降序排列
    3. 组间按权限级别降序排列（高权限优先）
    4. 同权限组内，优先保留高相关性Chunk
    5. 跨权限级别不发生重排序
    """
    
    PERM_ORDER = {"L0": 0, "L1": 1, "L2": 2, "L3": 3, "L4": 4}
    
    def __init__(self, base_reranker=None):
        """
        Args:
            base_reranker: 底层Re-ranker（如ColBERT, CrossEncoder）
        """
        self.base_reranker = base_reranker
        
    def rerank(self, query: str, 
               chunks: List[PermissionChunk],
               user_perm: str) -> List[ScoredChunk]:
        """
        执行权限感知重排序
        
        步骤:
        1. 过滤不可见Chunk
        2. 使用base_reranker计算相关性得分
        3. 按权限分组内部排序
        4. 按权限级别拼接
        """
        # Step 1: 权限过滤
        visible = self._filter_by_permission(chunks, user_perm)
        
        # Step 2: 计算相关性得分
        scored = self._compute_scores(query, visible)
        
        # Step 3: 按权限分组 + 组内排序
        groups = self._group_and_sort(scored)
        
        # Step 4: 按权限级别降序拼接
        result = []
        for level in sorted(groups.keys(), 
                           key=lambda x: self.PERM_ORDER[x],
                           reverse=True):
            result.extend(groups[level])
            
        return result
    
    def _filter_by_permission(self, chunks: List[PermissionChunk], 
                              user_perm: str) -> List[PermissionChunk]:
        """过滤不可见Chunk"""
        user_level = self.PERM_ORDER.get(user_perm, -1)
        return [
            c for c in chunks 
            if self.PERM_ORDER.get(c.perm_level, 5) <= user_level
        ]
    
    def _compute_scores(self, query: str, 
                        chunks: List[PermissionChunk]) -> List[ScoredChunk]:
        """使用底层Re-ranker计算得分"""
        if self.base_reranker is None:
            # 无Re-ranker时，使用默认得分
            return [
                ScoredChunk(
                    content=c.content,
                    perm_level=c.perm_level,
                    doc_id=c.doc_id,
                    chunk_id=c.chunk_id,
                    relevance_score=1.0,
                    retrieval_score=1.0
                )
                for c in chunks
            ]
            
        # 调用底层Re-ranker
        texts = [c.content for c in chunks]
        scores = self.base_reranker.score(query, texts)
        
        return [
            ScoredChunk(
                content=c.content,
                perm_level=c.perm_level,
                doc_id=c.doc_id,
                chunk_id=c.chunk_id,
                relevance_score=scores[i],
                retrieval_score=getattr(c, 'retrieval_score', 0.0)
            )
            for i, c in enumerate(chunks)
        ]
    
    def _group_and_sort(self, scored_chunks: List[ScoredChunk]) -> Dict[str, List[ScoredChunk]]:
        """按权限分组并在组内按相关性排序"""
        groups: Dict[str, List[ScoredChunk]] = {}
        for chunk in scored_chunks:
            groups.setdefault(chunk.perm_level, []).append(chunk)
            
        # 组内按 relevance_score 降序
        for level in groups:
            groups[level].sort(key=lambda x: x.relevance_score, reverse=True)
            
        return groups
    
    def compress_in_order(self, chunks: List[ScoredChunk],
                          compressor: PermissionAwareCompressor,
                          max_tokens: int) -> str:
        """
        按权限顺序依次压缩
        
        策略:
        1. 按权限级别从高到低处理
        2. 每级压缩后检查Token预算
        3. 预算耗尽时，停止处理更低权限级别
        4. 保证高权限内容优先进入上下文（若用户有权查看）
        """
        groups: Dict[str, List[ScoredChunk]] = {}
        for chunk in chunks:
            groups.setdefault(chunk.perm_level, []).append(chunk)
            
        result_parts = []
        remaining_tokens = max_tokens
        
        for level in sorted(groups.keys(),
                           key=lambda x: self.PERM_ORDER[x],
                           reverse=True):
            group_chunks = groups[level]
            
            # 将ScoredChunk转回PermissionChunk
            perm_chunks = [
                PermissionChunk(c.content, c.perm_level, c.doc_id, c.chunk_id)
                for c in group_chunks
            ]
            
            # 压缩该组
            compressed = compressor._compress_group(perm_chunks, level)
            compressed_tokens = len(compressor.base_compressor.tokenizer.encode(compressed))
            
            if compressed_tokens <= remaining_tokens:
                result_parts.append(f'<group level="{level}">')
                result_parts.append(compressed)
                result_parts.append('</group>')
                remaining_tokens -= compressed_tokens
            else:
                # 预算不足，尝试进一步压缩该组
                reduced = self._emergency_compress(compressed, remaining_tokens)
                if reduced:
                    result_parts.append(f'<group level="{level}">')
                    result_parts.append(reduced)
                    result_parts.append('</group>')
                    remaining_tokens -= len(
                        compressor.base_compressor.tokenizer.encode(reduced)
                    )
                # 若仍不足，跳过该组及以下级别
                else:
                    break
                    
        return "\n".join(result_parts)
    
    def _emergency_compress(self, text: str, max_tokens: int) -> Optional[str]:
        """紧急压缩：在Token预算内保留尽可能多的内容"""
        # 使用更激进的压缩比，或简单截断（但保持标记完整）
        builder = SecureContextBuilder()
        truncated = builder.safe_truncate(text, max_tokens)
        return truncated if truncated.strip() else None
```

### 7.3 Re-rank 对比

| 维度 | 传统Re-ranker | PermissionAwareReranker |
|------|-------------|------------------------|
| **排序依据** | 仅相关性得分 | 相关性得分 + 权限级别 |
| **跨权限重排** | 允许 | 禁止 |
| **压缩行为** | 全局压缩，可能跨权限合并 | 分组压缩，组间隔离 |
| **截断风险** | 截断可能打散权限标记 | 截断优先发生在组间边界 |
| **Token分配** | 按相关性分配 | 按权限级别优先分配 |

## 10. 最终安全校验

### 10.1 SecureResponseValidator 实现

```python
import re
from typing import List, Dict, Tuple, Optional, Set
from difflib import SequenceMatcher


class SecureResponseValidator:
    """
    安全响应校验器
    
    作为最后一道防线，在LLM输出返回给用户前执行校验：
    1. 检查输出中是否包含用户无权访问的敏感内容
    2. 检查输出是否通过语义推理泄露了受限信息
    3. 检查权限标记是否意外暴露在输出中
    """
    
    PERM_ORDER = {"L0": 0, "L1": 1, "L2": 2, "L3": 3, "L4": 4}
    
    def __init__(self):
        # 敏感模式库（正则）
        self.sensitive_patterns = {
            "L4": [
                r'裁员\s*\d+%',
                r'收购金额\s*[:$]\s*\d+[MBT]?',
                r'IPO\s*计划.*\d{4}',
            ],
            "L3": [
                r'营收\s*[:$]\s*\d+[MBT]?',
                r'毛利率\s*[:：]\s*\d+%',
                r'客户流失.*\d+',
            ],
            "L2": [
                r'内部预算\s*\d+',
                r'未发布产品',
            ]
        }
        
    def validate(self, response: str, 
                 user_perm: str,
                 original_chunks: List[PermissionChunk],
                 conversation_history: Optional[List[Dict]] = None) -> Tuple[bool, str, List[str]]:
        """
        校验LLM输出是否安全
        
        Returns:
            (是否通过, 过滤后响应/错误提示, 警告列表)
        """
        warnings = []
        user_level = self.PERM_ORDER.get(user_perm, 0)
        
        # Check 1: 检查输出中是否包含未授权的原始文本片段
        leaked_fragments = self._check_chunk_leakage(response, original_chunks, user_level)
        if leaked_fragments:
            warnings.append(f"检测到未授权Chunk片段泄露: {leaked_fragments}")
            # 替换泄露片段
            response = self._sanitize_leakage(response, leaked_fragments)
            
        # Check 2: 检查输出是否匹配高于用户权限的敏感模式
        unauthorized_patterns = self._check_sensitive_patterns(response, user_level)
        if unauthorized_patterns:
            warnings.append(f"检测到未授权敏感模式: {unauthorized_patterns}")
            return False, "抱歉，生成的回答可能包含您无权访问的信息，已阻止。", warnings
            
        # Check 3: 检查权限标记是否意外暴露在输出中
        exposed_markers = self._check_exposed_markers(response)
        if exposed_markers:
            warnings.append(f"权限标记暴露在输出中: {exposed_markers}")
            response = self._remove_markers(response)
            
        # Check 4: 检查是否通过推理间接泄露（启发式）
        inference_leak = self._check_inference_leakage(
            response, original_chunks, user_level, conversation_history
        )
        if inference_leak:
            warnings.append(f"可能存在推理泄露风险: {inference_leak}")
            # 不阻止，但记录日志并降低置信度
            
        passed = len([w for w in warnings if "已阻止" in w]) == 0
        return passed, response, warnings
    
    def _check_chunk_leakage(self, response: str, 
                             chunks: List[PermissionChunk],
                             user_level: int) -> List[Tuple[str, str]]:
        """
        检查响应中是否包含用户无权访问的Chunk原文片段
        
        使用模糊匹配检测长片段泄露（>10字符）
        """
        leaked = []
        
        for chunk in chunks:
            chunk_level = self.PERM_ORDER.get(chunk.perm_level, 5)
            if chunk_level <= user_level:
                continue  # 用户有权访问，不算泄露
                
            # 查找长公共子串
            matcher = SequenceMatcher(None, response, chunk.content)
            for match in matcher.get_matching_blocks():
                if match.size >= 10:  # 10字符以上视为泄露
                    leaked_fragment = response[match.a: match.a + match.size]
                    leaked.append((chunk.chunk_id, leaked_fragment))
                    
        return leaked
    
    def _sanitize_leakage(self, response: str, 
                          leaked: List[Tuple[str, str]]) -> str:
        """清除检测到的泄露片段"""
        sanitized = response
        for chunk_id, fragment in leaked:
            sanitized = sanitized.replace(fragment, "[已过滤内容]")
        return sanitized
    
    def _check_sensitive_patterns(self, response: str, 
                                  user_level: int) -> List[str]:
        """检查响应中是否包含高于用户权限的敏感模式"""
        matched = []
        
        for level, patterns in self.sensitive_patterns.items():
            level_num = self.PERM_ORDER.get(level, 5)
            if level_num <= user_level:
                continue  # 用户权限足够，不检查
                
            for pattern in patterns:
                if re.search(pattern, response, re.IGNORECASE):
                    matched.append(f"L{level_num}模式: {pattern}")
                    
        return matched
    
    def _check_exposed_markers(self, response: str) -> List[str]:
        """检查权限标记XML是否暴露在输出中"""
        exposed = []
        
        # 查找不应该出现在用户可见输出中的标记
        marker_patterns = [
            r'<perm\s+level="L[0-4]"[^>]*>',
            r'</perm>',
            r'<group\s+level="L[0-4]"[^>]*>',
            r'</group>',
        ]
        
        for pattern in marker_patterns:
            matches = re.findall(pattern, response)
            exposed.extend(matches)
            
        return exposed
    
    def _remove_markers(self, response: str) -> str:
        """移除输出中的权限标记"""
        cleaned = response
        patterns = [
            r'<perm\s+level="L[0-4]"[^>]*>',
            r'</perm>',
            r'<group\s+level="L[0-4]"[^>]*>',
            r'</group>',
        ]
        for pattern in patterns:
            cleaned = re.sub(pattern, '', cleaned)
        return cleaned.strip()
    
    def _check_inference_leakage(self, response: str,
                                  chunks: List[PermissionChunk],
                                  user_level: int,
                                  history: Optional[List[Dict]]) -> List[str]:
        """
        启发式检查推理泄露
        
        场景: 用户无权访问L3文档，但LLM通过多个L1文档推理出L3结论
        检测方法（启发式）:
        - 响应中包含"因此"、"可以看出"等推理词 + 敏感数值
        - 响应中的关键信息无法追溯到用户可见的Chunk
        """
        warnings = []
        
        # 简单启发式: 若回答中包含具体数字，但可见Chunk中无该数字
        numbers_in_response = set(re.findall(r'\d+\.?\d*', response))
        
        visible_chunks = [
            c for c in chunks 
            if self.PERM_ORDER.get(c.perm_level, 5) <= user_level
        ]
        
        visible_text = " ".join([c.content for c in visible_chunks])
        numbers_in_visible = set(re.findall(r'\d+\.?\d*', visible_text))
        
        suspicious_numbers = numbers_in_response - numbers_in_visible
        
        # 过滤常见数字（年份等）
        common_numbers = {'2024', '2025', '2026', '1', '2', '3'}
        suspicious_numbers -= common_numbers
        
        if suspicious_numbers:
            warnings.append(
                f"响应中包含无法追溯到可见Chunk的数值: {suspicious_numbers}"
            )
            
        return warnings
```

### 10.3 校验结果处理策略

| 校验项 | 严重程度 | 处理方式 | 日志级别 |
|--------|---------|---------|---------|
| Chunk原文泄露 | **高** | 替换为`[已过滤内容]`，允许返回 | WARN |
| 敏感模式匹配（越权） | **致命** | 阻止返回，输出统一拒绝模板 | ERROR |
| 权限标记暴露 | **中** | 移除标记后返回 | WARN |
| 推理泄露（启发式） | **低** | 允许返回，降低置信度，记录审计 | INFO |

## 附录B: 性能考量

| 操作 | 额外开销 | 优化策略 |
|------|---------|---------|
| 权限分组 | O(n) | 检索阶段直接返回分组结果 |
| XML解析校验 | O(m) | 仅在DEBUG模式全量校验，生产环境采样 |

| 历史重新过滤 | O(r×s) | 缓存各权限级别的过滤结果 |
