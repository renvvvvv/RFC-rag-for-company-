## 2. 各服务职责说明

### 2.1 服务职责矩阵

| 服务 | 端口 | 核心职责 | 数据存储 |
|------|------|----------|----------|
| **api-gateway** | 8000 | 统一入口、路由、鉴权、限流、日志 | Kong config (DB-less) |
| **user-service** | 8100 | 用户管理、认证授权 (OAuth2/JWT)、组织架构 | PostgreSQL + Redis |
| **kb-service** | 8101 | 知识库CRUD、空间配额、统计 | PostgreSQL |
| **doc-service** | 8102 | 文档上传/下载/版本/预览、MinIO管理 | PostgreSQL + MinIO |
| **permission-service** | 8103 | RBAC+ABAC策略引擎、四级权限判定、字段过滤 | PostgreSQL + Redis |
| **tag-service** | 8104 | 标签定义、标签绑定、自动标签分类 | PostgreSQL |
| **ingest-service** | 8105 | 摄取任务编排、格式识别、Pipeline调度 | PostgreSQL |
| **search-service** | 8106 | 多路检索、RRF融合、Re-rank、权限过滤 | Milvus + Redis |
| **generate-service** | 8107 | LLM调用、上下文压缩、流式生成、安全校验 | — (无状态) |
| **keyword-service** | 8108 | 敏感词库管理、实时内容扫描、DFA引擎 | Redis + 内存 |
| **eval-service** | 8109 | 评测集管理、检索/生成评测、LLM-as-Judge | PostgreSQL |
| **audit-service** | 8110 | 审计日志采集、查询、归档 | PostgreSQL + 对象存储 |
| **notification-service** | 8111 | 通知模板、多渠道推送 (邮件/飞书/Webhook) | PostgreSQL + Redis |

### 2.2 核心服务详细说明

#### permission-service（权限控制服务）

> 核心服务。所有数据访问的"守门人"。

**核心功能模块：**

```
permission-service/
├── policy_engine/          # 策略决策引擎 (PDP)
│   ├── rbac_evaluator.py   # 角色权限评估
│   ├── abac_evaluator.py   # 属性权限评估
│   └── decision_combiner.py# 决策组合器 (Deny优先)
├── field_filter/           # 字段级权限
│   ├── word_filter.py      # Word段落过滤
│   └── excel_filter.py     # Excel行列过滤
├── tag_filter/             # 标签权限
│   └── tag_evaluator.py
└── cache/                  # 高性能缓存
    └── redis_bitmap.py     # 位图预计算
```

**对外接口：**

```python
# POST /v1/permissions/check
class PermissionCheckRequest(BaseModel):
```

### 4.4 核心任务定义示例

```python
# workers/ingest-worker/tasks/ingest_pipeline.py
from celery import chain, group, chord
from shared.celery_config import app
from shared.message.events import IngestCompletedEvent, EventType

@app.task(bind=True, max_retries=3, default_retry_delay=60)
def process_document(self, task_id: str, doc_id: str, file_key: str, file_type: str):
    """
    文档摄取主Pipeline
    使用Celery Canvas编排子任务
    """
    try:
        # Step 1: 解析
        parse_result = parse_document.delay(doc_id, file_key, file_type)
        
        # Step 2: 分块 (依赖解析结果)
        chunk_result = chunk_document.delay(doc_id, parse_result.get(timeout=300))
        
        # Step 3: 并行向量化 (Dense + Sparse)
        chunks = chunk_result.get(timeout=300)
        
        dense_job = embed_dense.delay(doc_id, chunks)
        sparse_job = embed_sparse.delay(doc_id, chunks)
        
        # Step 4: 索引入库 (依赖向量化完成)
        chord(
            group(dense_job, sparse_job),
            index_document.s(doc_id)
        ).apply_async()
        
        # 发布完成事件
        publish_event(IngestCompletedEvent(
            event_id=generate_uuid(),
            event_type=EventType.INGEST_COMPLETED,
            timestamp=now(),
            source_service="ingest-worker",
            payload={"task_id": task_id, "doc_id": doc_id},
            trace_id=get_trace_id()
        ))
        
    except Exception as exc:
        # 记录失败，进入死信队列
        self.retry(exc=exc)


# workers/embed-worker/tasks/embed_task.py
@app.task(bind=True, queue='q.embed.dense', time_limit=600)
def embed_dense(self, doc_id: str, chunks: List[Dict]):
    """GPU密集型: Dense向量生成"""
    from shared.embedders import DenseEmbedder
    
    embedder = DenseEmbedder()
    
    # 批处理以最大化GPU利用率
    batch_size = 64
    vectors = []
    
    for i in range(0, len(chunks), batch_size):
        batch = chunks[i:i + batch_size]
        texts = [c['text'] for c in batch]
        batch_vectors = embedder.encode(texts)
        vectors.extend(batch_vectors)
    
    return {
        "doc_id": doc_id,
        "vectors": vectors,
        "dimension": embedder.dimension
    }


# workers/index-worker/tasks/milvus_index_task.py
@app.task(bind=True, queue='q.index.milvus', max_retries=2)
def index_to_milvus(self, doc_id: str, dense_vectors: List, metadata: List[Dict]):
    """Milvus批量写入"""
    from pymilvus import Collection
    
    try:
        collection = Collection("rag_chunks")
        
        # 批量插入
        entities = [
            [m['chunk_id'] for m in metadata],
            [m['doc_id'] for m in metadata],
            [m['kb_id'] for m in metadata],
            dense_vectors,
            [m['text'] for m in metadata],
        ]
        
        collection.insert(entities)
        collection.flush()
        
        # 如果达到阈值，触发索引构建
        if collection.num_entities % 100000 < len(metadata):
            build_index.delay("rag_chunks")
            
    except Exception as exc:
        self.retry(exc=exc)
```

### 7.3 熔断与降级策略

```python
# shared/clients/circuit_breaker.py
import asyncio
from enum import Enum
from datetime import datetime, timedelta
from typing import Callable, Any

class CircuitState(Enum):
    CLOSED = "closed"       # 正常通行
    OPEN = "open"           # 熔断，拒绝请求
    HALF_OPEN = "half_open" # 试探性放行

class CircuitBreaker:
    """
    熔断器实现
    
    配置参数:
    - failure_threshold: 连续失败次数阈值 (默认5)
    - recovery_timeout: 熔断后恢复等待时间 (默认30s)
    - half_open_max_calls: 半开状态最大试探请求数 (默认3)
    """
    
    def __init__(
        self,
        name: str,
        failure_threshold: int = 5,
        recovery_timeout: int = 30,
        half_open_max_calls: int = 3,
        fallback: Optional[Callable] = None
    ):
        self.name = name
        self.failure_threshold = failure_threshold
        self.recovery_timeout = recovery_timeout
        self.half_open_max_calls = half_open_max_calls
        self.fallback = fallback
        
        self.state = CircuitState.CLOSED
        self.failure_count = 0
        self.last_failure_time = None
        self.half_open_calls = 0
        self._lock = asyncio.Lock()
    
    async def call(self, func: Callable, *args, **kwargs) -> Any:
        async with self._lock:
            if self.state == CircuitState.OPEN:
                if self._should_attempt_reset():
                    self.state = CircuitState.HALF_OPEN
                    self.half_open_calls = 0
                else:
                    # 熔断中，返回降级结果
                    if self.fallback:
                        return await self.fallback(*args, **kwargs)
                    raise CircuitBreakerOpen(
                        f"Circuit {self.name} is OPEN. "
                        f"Retry after {self.recovery_timeout}s"
                    )
            
            if self.state == CircuitState.HALF_OPEN:
                if self.half_open_calls >= self.half_open_max_calls:
                    if self.fallback:
                        return await self.fallback(*args, **kwargs)
                    raise CircuitBreakerOpen("Circuit half-open limit reached")
                self.half_open_calls += 1
        
        # 执行实际请求
        try:
            result = await func(*args, **kwargs)
            await self._on_success()
            return result
        except Exception as e:
            await self._on_failure()
            raise
    
    async def _on_success(self):
        async with self._lock:
            self.failure_count = 0
            self.last_failure_time = None
            if self.state == CircuitState.HALF_OPEN:
                self.state = CircuitState.CLOSED
                self.half_open_calls = 0
    
    async def _on_failure(self):
        async with self._lock:
            self.failure_count += 1
            self.last_failure_time = datetime.utcnow()
            
            if self.state == CircuitState.HALF_OPEN:
                self.state = CircuitState.OPEN
            elif self.failure_count >= self.failure_threshold:
                self.state = CircuitState.OPEN
    
    def _should_attempt_reset(self) -> bool:
        if not self.last_failure_time:
            return True
        elapsed = (datetime.utcnow() - self.last_failure_time).total_seconds()
        return elapsed >= self.recovery_timeout


# 使用示例: generate-service 调用 LLM
breaker = CircuitBreaker(
    name="llm_api",
    failure_threshold=3,
    recovery_timeout=60,
    fallback=llm_fallback_response  # 降级: 返回"服务繁忙，请稍后重试"
)

@router.post("/chat")
async def chat(request: GenerateRequest):
    try:
        response = await breaker.call(
            call_llm_api,
            query=request.query,
            context=request.contexts
        )
        return response
    except CircuitBreakerOpen:
        return {
            "success": False,
            "error_code": "LLM_SERVICE_UNAVAILABLE",
            "message": "生成服务暂时不可用，请稍后重试",
            "retry_after": 60
        }
```

### 7.4 各服务降级策略

| 服务 | 故障场景 | 降级策略 | 用户体验 |
|------|----------|----------|----------|
| **search-service** | Milvus超时 | 降级为纯BM25关键词检索 | 结果减少但仍可返回 |
| **search-service** | Re-rank超时 | 跳过重排，直接返回融合结果 | Top-K精度略有下降 |
| **generate-service** | LLM超时 | 返回检索结果摘要，不生成 | 显示"相关文档"列表 |
| **generate-service** | LLM熔断 | 缓存历史回答/模板回答 | 提示"服务繁忙" |
| **permission-service** | Redis故障 | 直查PostgreSQL + 默认拒绝 | 响应变慢但安全 |
| **permission-service** | 完全不可用 | 默认拒绝所有非管理员请求 | 系统进入只读安全模式 |
| **keyword-service** | 扫描超时 | 跳过扫描，记录审计日志 | 生成内容可能未过滤 |
| **notification-service** | 推送失败 | 持久化到队列，延迟重试 | 通知延迟，不丢失 |
| **embed-worker** | GPU故障 | 任务滞留队列，告警通知 | 新文档暂不索引 |
| **audit-service** | 写入失败 | 本地文件缓冲，稍后补写 | 审计不丢失 |

```python
# shared/utils/retry.py
from tenacity import (
    retry,
    stop_after_attempt,
    wait_exponential,
    retry_if_exception_type,
    before_sleep_log
)
import logging

logger = logging.getLogger(__name__)

# 数据库操作重试
@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=1, max=10),
    retry=retry_if_exception_type((DBConnectionError, DBTimeout)),
    before_sleep=before_sleep_log(logger, logging.WARNING),
    reraise=True
)
async def db_query_with_retry(query_func):
    return await query_func()

# 外部HTTP调用重试
@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=2, min=2, max=30),
    retry=retry_if_exception_type((HTTPTimeout, HTTP503Error)),
    before_sleep=before_sleep_log(logger, logging.WARNING)
)
async def http_call_with_retry(client, method, url, **kwargs):
    return await client.request(method, url, **kwargs)

# LLM调用重试 (更宽松，因为LLM经常波动)
@retry(
    stop=stop_after_attempt(2),  # LLM不重试太多次，避免成本爆炸
    wait=wait_exponential(multiplier=3, min=5, max=60),
    retry=retry_if_exception_type((LLMRateLimit, LLMTimeout)),
    before_sleep=before_sleep_log(logger, logging.INFO)
)
async def llm_call_with_retry(prompt):
    return await call_llm(prompt)
```
