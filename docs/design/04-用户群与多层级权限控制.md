## 1. 权限体系总览

```
五级权限穿透（从粗到细）：

Level 1: 文件类型权限
    "群A可以访问【视频】和【PDF】，不能访问【Excel】"
    
Level 2: 文档级别权限
    "群A可以访问文档D001、D003，不能访问D002"
    
Level 3: 字段/内容级别权限（Word/Excel专用）
    "群A可以看到Excel'工资表'的'姓名'和'部门'列，看不到'薪资'列"
    "群A可以看到Word文档的前3节，第4节'机密条款'不可见"
    
Level 4: 标签级别权限
    "群A可以访问标签为'公开'和'内部'的文档，不能访问'机密'标签"
    
Level 5: 关键词级别权限
    "群A安全级别=L2，包含'并购'关键词(Level 4)的Chunk不可见"
```

### 2.2 用户最终权限计算规则

```python
class PermissionEngine:
    """
    用户最终权限计算引擎
    """
    
    def compute_user_effective_permissions(self, user: User) -> EffectivePermission:
        """
        计算用户的有效权限
        
        规则:
        1. DENY 永远优先于 ALLOW
        2. 用户显式权限 > 群权限 > 角色默认权限
        3. 多个群的权限取并集（OR逻辑），但DENY覆盖ALLOW
        4. 安全级别: effective_level = max(个人level, 所有群的max_level)
        """
        
        # 1. 获取用户所属的所有群
        groups = self.get_user_groups(user.id)
        
        # 2. 计算安全级别
        user_personal_level = user.security_level  # 如 "L2"
        group_max_levels = [g.max_security_level for g in groups]
        effective_level = self._max_level([user_personal_level] + group_max_levels)
        
        # 3. 计算资源权限并集
        resource_perms = {}  # doc_id -> permission
        
        # 3.1 个人权限（最高优先级）
        personal_perms = self.get_user_personal_permissions(user.id)
        for target, perm in personal_perms.items():
            resource_perms[target] = perm
        
        # 3.2 群权限（合并）
        for group in groups:
            group_perms = self.get_group_permissions(group.id)
            for target, perm in group_perms.items():
                if target not in resource_perms:
                    resource_perms[target] = perm
                else:
                    # 合并规则: DENY覆盖一切
                    existing = resource_perms[target]
                    if existing != "DENY" and perm == "DENY":
                        resource_perms[target] = "DENY"
                    elif existing == "ALLOW" or perm == "ALLOW":
                        resource_perms[target] = "ALLOW"
        
        # 3.3 角色默认权限（最低优先级）
        role_perms = self.get_role_permissions(user.role_id)
        for target, perm in role_perms.items():
            if target not in resource_perms:
                resource_perms[target] = perm
        
        return EffectivePermission(
            user_id=user.id,
            security_level=effective_level,
            resource_permissions=resource_perms,
            group_ids=[g.id for g in groups]
        )
    
    def _max_level(self, levels: List[str]) -> str:
        """取最高安全级别"""
        level_order = {"L0": 0, "L1": 1, "L2": 2, "L3": 3, "L4": 4}
        max_val = max(level_order.get(l, 0) for l in levels)
        for k, v in level_order.items():
            if v == max_val:
                return k
        return "L0"
```

### 2.3 权限计算示例

```
用户张三:
  - 个人权限: 安全级别 L2
  - 所属群A（产品研发部）: max_security_level=L3, 允许访问 doc_001, doc_002
  - 所属群B（高管组）: max_security_level=L4, 允许访问 doc_003
  - 角色（普通员工）: 默认允许访问公开文档

有效权限:
  - 安全级别 = max(L2, L3, L4) = L4
  - 可访问文档 = doc_001 ∪ doc_002 ∪ doc_003 ∪ 公开文档
  
如果群A对 doc_002 设置了 DENY:
  - 可访问文档 = doc_001 ∪ doc_003 ∪ 公开文档
  （DENY覆盖ALLOW，doc_002不可访问）
```

## 4. Level 2: 文档级别权限

控制用户对具体文档的访问能力，支持白名单和黑名单两种模式。

```python
class DocumentPermission(BaseModel):
    """文档级权限"""
    doc_id: str
    target_type: str                   # "user" | "group" | "role"
    target_id: str
    permission: str                    # "NONE" | "READ" | "WRITE" | "ADMIN"
    inherit_from_kb: bool = True       # 是否继承知识库默认权限

class PermissionLevel(Enum):
    NONE = 0    # 无权限
    READ = 1    # 只读（可检索、预览）
    WRITE = 2   # 可编辑（修改元数据、标签）
    ADMIN = 3   # 完全控制（删除、权限管理）

# 权限继承链
# 系统默认(NONE) -> 知识库默认权限 -> 群权限 -> 个人显式权限（最高优先级）
```

## 5. Level 3: 字段/内容级别权限

这是系统最具差异化的权限设计，支持对Word段落和Excel单元格/列进行细粒度控制。

### 5.1 核心思路

- 文档解析时记录结构化位置信息
- 索引时每个Chunk携带来源位置元数据
- 检索时根据用户权限过滤掉无权限位置的Chunk
- 生成时仅使用有权限的Chunk作为上下文

## 7. Level 5: 关键词级别权限

与关键词敏感控制系统协同工作，作为内容级的最后一道防线。

```
全局关键词配置:
┌─────────────┬───────┬──────────┬────────────┐
│ 关键词      │ 级别  │ 分类     │ action     │
├─────────────┼───────┼──────────┼────────────┤
│ 薪资        │ L3    │ 财务     │ block      │
│ 并购        │ L4    │ 战略     │ block      │
│ 裁员        │ L3    │ HR       │ block      │
│ 客户名单    │ L4    │ 商业     │ block      │
│ 源代码      │ L4    │ 技术     │ block      │
│ 专利        │ L2    │ 知识产权 │ warn       │
└─────────────┴───────┴──────────┴────────────┘

Chunk入库时:
  - 检测包含的关键词
  - 标注 max_keyword_level = max(匹配到的关键词级别)
  - 存入metadata

检索时:
  - 用户 effective_level >= Chunk.max_keyword_level ?
  - 不满足 → 拦截，记录审计日志
```

## 8. 权限缓存设计

### 8.1 Redis缓存结构

```python
# 1. 用户-文档权限映射 (Hash)
# Key: acl:doc_perm:{user_id}
# Field: doc_id
# Value: ALLOW | DENY | PARTIAL
# TTL: 5分钟
HSET acl:doc_perm:user_123 doc_001 "ALLOW"

# 2. 群-文档权限映射 (Hash)
# Key: acl:doc_perm:group:{group_id}
HSET acl:doc_perm:group:group_001 doc_001 "ALLOW"

# 3. 用户标签权限 (String JSON)
# Key: acl:tag_perm:{user_id}
SET acl:tag_perm:user_123 '{"allowed":["公开","内部"],"denied":["机密"]}'
EXPIRE acl:tag_perm:user_123 300

# 4. 用户-字段权限配置 (String JSON)
# Key: acl:field_perm:{doc_id}:{user_id}
SET acl:field_perm:doc_003:user_123 '{"sheet_permissions":{...}}'
EXPIRE acl:field_perm:doc_003:user_123 600

# 5. 用户-文件类型权限 (Set)
# Key: acl:file_types:{user_id}
SADD acl:file_types:user_123 document pdf excel image video
EXPIRE acl:file_types:user_123 300

# 6. ACL版本号 (用于检测权限是否变更)
# Key: acl:version:{doc_id}
SET acl:version:doc_001 "v17"

# 7. 用户有效安全级别 (String)
# Key: acl:user_level:{user_id}
SET acl:user_level:user_123 "L4"
EXPIRE acl:user_level:user_123 300
```

### 8.2 缓存预热与失效策略

```python
class PermissionCacheManager:
    """权限缓存管理器"""
    
    async def warm_cache_for_user(self, user_id: str):
        """用户登录时预热权限缓存"""
        user = await self.get_user(user_id)
        groups = await self.get_user_groups(user_id)
        
        # 预热文档权限
        all_perms = {}
        for group in groups:
            group_perms = await self.get_group_document_permissions(group.id)
            all_perms.update(group_perms)
        
        personal_perms = await self.get_user_personal_permissions(user_id)
        all_perms.update(personal_perms)
        
        # 批量写入Redis
        if all_perms:
            pipe = self.redis.pipeline()
            key = f"acl:doc_perm:{user_id}"
            for doc_id, perm in all_perms.items():
                pipe.hset(key, doc_id, perm)
            pipe.expire(key, 300)
            await pipe.execute()
        
        # 预热标签权限
        effective_tags = self._compute_effective_tag_permissions(user, groups)
        await self.redis.setex(
            f"acl:tag_perm:{user_id}",
            300,
            json.dumps(effective_tags)
        )
        
        # 预热安全级别
        effective_level = self._compute_effective_level(user, groups)
        await self.redis.setex(
            f"acl:user_level:{user_id}",
            300,
            effective_level
        )
    
    async def invalidate_user_cache(self, user_id: str):
        """使某个用户的所有权限缓存失效"""
        patterns = [
            f"acl:doc_perm:{user_id}",
            f"acl:tag_perm:{user_id}",
            f"acl:field_perm:*:{user_id}",
            f"acl:file_types:{user_id}",
            f"acl:user_level:{user_id}",
        ]
        for pattern in patterns:
            await self._delete_keys_by_pattern(pattern)
    
    async def invalidate_document_cache(self, doc_id: str):
        """文档权限变更时，使所有相关用户的缓存失效"""
        # 获取能访问该文档的所有用户
        affected_users = await self.get_users_with_doc_access(doc_id)
        for user_id in affected_users:
            await self.redis.hdel(f"acl:doc_perm:{user_id}", doc_id)
            await self.redis.delete(f"acl:field_perm:{doc_id}:{user_id}")
        
        # 更新ACL版本号
        new_version = f"v{int(time.time())}"
        await self.redis.set(f"acl:version:{doc_id}", new_version)
```

## 9. 权限变更同步机制

### 9.1 事件定义

```python
class PermissionChangeEvent(Enum):
    DOC_PERMISSION_CHANGED = "doc_perm_changed"
    GROUP_PERMISSION_CHANGED = "group_perm_changed"
    ROLE_PERMISSION_CHANGED = "role_perm_changed"
    USER_ROLE_CHANGED = "user_role_changed"
    USER_GROUP_CHANGED = "user_group_changed"
    TAG_PERMISSION_CHANGED = "tag_perm_changed"
    FIELD_PERMISSION_CHANGED = "field_perm_changed"
    KEYWORD_CONFIG_CHANGED = "keyword_config_changed"
```

### 9.2 事件处理器

```python
class PermissionEventHandler:
    """权限变更事件处理器"""
    
    async def handle(self, event: Dict):
        event_type = event["type"]
        handlers = {
            PermissionChangeEvent.DOC_PERMISSION_CHANGED.value: self._handle_doc_perm_change,
            PermissionChangeEvent.GROUP_PERMISSION_CHANGED.value: self._handle_group_perm_change,
            PermissionChangeEvent.USER_GROUP_CHANGED.value: self._handle_user_group_change,
            PermissionChangeEvent.KEYWORD_CONFIG_CHANGED.value: self._handle_keyword_config_change,
        }
        
        handler = handlers.get(event_type)
        if handler:
            await handler(event)
    
    async def _handle_doc_perm_change(self, event: Dict):
        """文档权限变更"""
        doc_id = event["doc_id"]
        
        # 1. 使该文档的所有权限缓存失效
        await self.cache_manager.invalidate_document_cache(doc_id)
        
        # 2. 更新向量数据库中Chunk的acl_version
        new_version = await self._generate_new_acl_version(doc_id)
        await self._update_chunk_acl_version(doc_id, new_version)
        
        # 3. 重新计算受影响用户的权限缓存（异步）
        affected_users = await self._get_users_with_doc_access(doc_id)
        for user_id in affected_users:
            await self.cache_manager.warm_cache_for_user(user_id)
    
    async def _handle_group_perm_change(self, event: Dict):
        """群权限变更"""
        group_id = event["group_id"]
        
        # 群权限变化影响所有该群成员
        members = await self._get_group_members(group_id)
        for user_id in members:
            await self.cache_manager.invalidate_user_cache(user_id)
            await self.cache_manager.warm_cache_for_user(user_id)
    
    async def _handle_user_group_change(self, event: Dict):
        """用户群成员变更"""
        user_id = event["user_id"]
        
        # 用户加入/退出群，需要重新计算所有权限
        await self.cache_manager.invalidate_user_cache(user_id)
        await self.cache_manager.warm_cache_for_user(user_id)
    
    async def _handle_keyword_config_change(self, event: Dict):
        """关键词配置变更"""
        # 需要重新标注所有Chunk的关键词
        # 触发异步重标注任务
        await self.task_queue.send_task(
            "reannotate_all_chunks",
            {"changed_keywords": event.get("keywords", [])}
        )
```

### 9.3 消息队列拓扑

```
Exchange: permission.events (topic)

Queues:
  - permission.doc.changed        → doc_id级别处理
  - permission.group.changed      → group_id级别处理  
  - permission.user.changed       → user_id级别处理
  - permission.keyword.changed    → 全局关键词重标注

Consumers:
  - permission-sync-worker (4 instances)
    - 消费所有队列
    - 执行缓存失效和预热
```
