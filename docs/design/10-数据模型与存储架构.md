### 1.2 实体关系详细说明

| 实体A | 关系 | 实体B | 基数 | 说明 |
|-------|------|-------|------|------|
| users | 1:N | group_members | 一个用户可加入多个群组 | 用户-群组成员 |
| user_groups | 1:N | group_members | 一个群组可有多个成员 | 群组-群组成员 |
| users | 1:N | knowledge_bases | 一个用户可创建多个知识库 | 用户-知识库(创建者) |
| knowledge_bases | 1:N | documents | 一个知识库含多个文档 | 知识库-文档 |
| documents | 1:N | chunks | 一个文档含多个片段 | 文档-片段 |
| documents | N:M | tags | 通过 document_tags 关联 | 文档-标签 |
| chunks | N:M | tags | 通过 chunk_tags 关联 | 片段-标签 |
| users | 1:N | document_permissions | 一个用户有多条文档权限 | 用户-文档权限 |
| users | 1:N | field_permissions | 一个用户有多条字段权限 | 用户-字段权限 |
| users | 1:N | group_permissions | 一个用户有多条群组权限 | 用户-群组权限 |
| users | 1:N | file_type_permissions | 一个用户有多条文件类型权限 | 用户-文件类型权限 |
| users | 1:N | tag_permissions | 一个用户有多条标签权限 | 用户-标签权限 |
| users | 1:N | audit_logs | 一个用户产生多条审计日志 | 用户-审计日志 |

## 2. PostgreSQL 关键表结构

### 2.1 用户与群组

```sql
-- ============================================
-- 2.1.1 用户表
-- ============================================
CREATE TABLE users (
    id              BIGSERIAL PRIMARY KEY,
    username        VARCHAR(64) NOT NULL UNIQUE,
    email           VARCHAR(128) NOT NULL UNIQUE,
    password_hash   VARCHAR(255) NOT NULL,
    display_name    VARCHAR(128),
    avatar_url      VARCHAR(512),
    status          VARCHAR(16) NOT NULL DEFAULT 'active'
                        CHECK (status IN ('active', 'inactive', 'suspended', 'deleted')),
    role            VARCHAR(32) NOT NULL DEFAULT 'user'
                        CHECK (role IN ('super_admin', 'admin', 'user', 'guest')),
    max_keyword_level INT NOT NULL DEFAULT 5
                        CHECK (max_keyword_level BETWEEN 0 AND 5),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_login_at   TIMESTAMPTZ
);

COMMENT ON TABLE users IS '系统用户表';
COMMENT ON COLUMN users.max_keyword_level IS '用户可访问的最高敏感等级(0-5, 0=最高敏感)';

-- ============================================
-- 2.1.2 用户群组表
-- ============================================
CREATE TABLE user_groups (
    id              BIGSERIAL PRIMARY KEY,
    name            VARCHAR(128) NOT NULL,
    description     TEXT,
    owner_id        BIGINT NOT NULL REFERENCES users(id) ON DELETE RESTRICT,
    status          VARCHAR(16) NOT NULL DEFAULT 'active'
                        CHECK (status IN ('active', 'inactive', 'archived')),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE user_groups IS '用户群组，用于批量权限管理';

-- ============================================
-- 2.1.3 群组成员表
-- ============================================
CREATE TABLE group_members (
    id              BIGSERIAL PRIMARY KEY,
    group_id        BIGINT NOT NULL REFERENCES user_groups(id) ON DELETE CASCADE,
    user_id         BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role            VARCHAR(16) NOT NULL DEFAULT 'member'
                        CHECK (role IN ('owner', 'admin', 'member')),
    joined_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (group_id, user_id)
);

COMMENT ON TABLE group_members IS '群组成员关联表';
```

### 2.2 知识库与文档

```sql
-- ============================================
-- 2.2.1 知识库表
-- ============================================
CREATE TABLE knowledge_bases (
    id              BIGSERIAL PRIMARY KEY,
    name            VARCHAR(256) NOT NULL,
    description     TEXT,
    owner_id        BIGINT NOT NULL REFERENCES users(id) ON DELETE RESTRICT,
    embedding_model VARCHAR(128) NOT NULL DEFAULT 'bge-m3',
    chunk_size      INT NOT NULL DEFAULT 512,
    chunk_overlap   INT NOT NULL DEFAULT 50,
    status          VARCHAR(16) NOT NULL DEFAULT 'active'
                        CHECK (status IN ('active', 'inactive', 'archived')),
    default_perm_level VARCHAR(16) NOT NULL DEFAULT 'private'
                        CHECK (default_perm_level IN ('public', 'internal', 'private', 'restricted')),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE knowledge_bases IS '知识库定义表';

-- ============================================
-- 2.2.2 文档表
-- ============================================
CREATE TABLE documents (
    id              BIGSERIAL PRIMARY KEY,
    kb_id           BIGINT NOT NULL REFERENCES knowledge_bases(id) ON DELETE CASCADE,
    title           VARCHAR(512) NOT NULL,
    file_name       VARCHAR(512) NOT NULL,
    file_type       VARCHAR(32) NOT NULL,
    file_size       BIGINT NOT NULL DEFAULT 0,
    file_path       VARCHAR(1024),              -- MinIO 对象存储路径
    mime_type       VARCHAR(128),
    content_text    TEXT,                       -- 解析后的纯文本内容(可选存储)
    status          VARCHAR(32) NOT NULL DEFAULT 'pending'
                        CHECK (status IN ('pending', 'parsing', 'chunking', 'indexing',
                                          'completed', 'failed', 'stored_only')),
    stored_only     BOOLEAN NOT NULL DEFAULT FALSE,
    -- stored_only: 仅存储原始文件，不参与向量化索引和检索
    parser_config   JSONB DEFAULT '{}',         -- 解析器配置参数
    meta_info       JSONB DEFAULT '{}',         -- 业务元数据(来源、作者、部门等)
    created_by      BIGINT NOT NULL REFERENCES users(id),
    updated_by      BIGINT REFERENCES users(id),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    completed_at    TIMESTAMPTZ,
    acl_version     BIGINT NOT NULL DEFAULT 1   -- ACL版本号，用于缓存一致性
);

COMMENT ON TABLE documents IS '文档表，存储原始文件元数据和解析状态';
COMMENT ON COLUMN documents.stored_only IS 'TRUE=仅存储不参与索引；FALSE=正常解析索引';
COMMENT ON COLUMN documents.acl_version IS 'ACL版本号，权限变更时递增，用于Redis缓存校验';

CREATE INDEX idx_documents_kb_id ON documents(kb_id);
CREATE INDEX idx_documents_status ON documents(status);
CREATE INDEX idx_documents_stored_only ON documents(stored_only) WHERE stored_only = TRUE;
CREATE INDEX idx_documents_created_at ON documents(created_at DESC);
```

### 2.3 文本片段

```sql
-- ============================================
-- 2.3.1 片段表
-- ============================================
CREATE TABLE chunks (
    id              BIGSERIAL PRIMARY KEY,
    doc_id          BIGINT NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    kb_id           BIGINT NOT NULL REFERENCES knowledge_bases(id) ON DELETE CASCADE,
    chunk_index     INT NOT NULL,               -- 文档内片段序号
    content         TEXT NOT NULL,              -- 片段文本内容
    content_type    VARCHAR(16) NOT NULL DEFAULT 'text'
                        CHECK (content_type IN ('text', 'table', 'image_desc', 'audio_desc')),
    -- 位置信息(用于溯源定位)
    position_info   JSONB NOT NULL DEFAULT '{}',
    -- position_info 示例:
    -- {"page": 3, "bbox": [100, 200, 400, 300], "section": "1.2 概述"}
    -- {"start_char": 1500, "end_char": 2000, "line_start": 45, "line_end": 62}
    -- {"frame_start": 120, "frame_end": 180, "timestamp": "00:02:00"}

    -- 敏感控制
    max_keyword_level INT NOT NULL DEFAULT 5
                        CHECK (max_keyword_level BETWEEN 0 AND 5),
    -- max_keyword_level: 该片段包含的最高敏感关键词等级(0=最高敏感, 5=公开)

    -- 标签(冗余存储，加速过滤)
    tags            BIGINT[] DEFAULT '{}',      -- 关联标签ID数组

    -- 权限版本
    doc_acl_version BIGINT NOT NULL DEFAULT 1,  -- 关联文档的ACL版本快照

    -- 向量引用
    vector_id       VARCHAR(64),                -- Milvus 向量记录ID
    vector_status   VARCHAR(16) NOT NULL DEFAULT 'pending'
                        CHECK (vector_status IN ('pending', 'indexed', 'failed', 'deleted')),

    -- 状态
    status          VARCHAR(16) NOT NULL DEFAULT 'active'
                        CHECK (status IN ('active', 'inactive', 'deprecated')),

    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE chunks IS '文档分片表，存储文本片段及向量化状态';
COMMENT ON COLUMN chunks.position_info IS '片段在原始文档中的位置信息，格式因文档类型而异';
COMMENT ON COLUMN chunks.max_keyword_level IS '片段敏感等级，由敏感关键词扫描结果决定';
COMMENT ON COLUMN chunks.tags IS '冗余标签ID数组，避免JOIN加速权限过滤';
COMMENT ON COLUMN chunks.doc_acl_version IS '创建时文档ACL版本，与document.acl_version匹配才有效';

CREATE INDEX idx_chunks_doc_id ON chunks(doc_id);
CREATE INDEX idx_chunks_kb_id ON chunks(kb_id);
CREATE INDEX idx_chunks_vector_status ON chunks(vector_status);
CREATE INDEX idx_chunks_status ON chunks(status);
CREATE INDEX idx_chunks_max_keyword_level ON chunks(max_keyword_level);
CREATE INDEX idx_chunks_gin_tags ON chunks USING GIN(tags);

-- 部分索引：仅存储文件不产生片段
CREATE INDEX idx_chunks_doc_id_not_stored ON chunks(doc_id)
    WHERE doc_id NOT IN (SELECT id FROM documents WHERE stored_only = TRUE);
```

### 2.4 标签系统

```sql
-- ============================================
-- 2.4.1 标签定义表
-- ============================================
CREATE TABLE tags (
    id              BIGSERIAL PRIMARY KEY,
    kb_id           BIGINT NOT NULL REFERENCES knowledge_bases(id) ON DELETE CASCADE,
    name            VARCHAR(128) NOT NULL,
    color           VARCHAR(7) DEFAULT '#1890ff',  -- 标签颜色 HEX
    description     TEXT,
    parent_id       BIGINT REFERENCES tags(id) ON DELETE SET NULL,
    status          VARCHAR(16) NOT NULL DEFAULT 'active'
                        CHECK (status IN ('active', 'inactive')),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (kb_id, name)
);

COMMENT ON TABLE tags IS '标签定义表，知识库内唯一';

-- ============================================
-- 2.4.2 文档标签关联
-- ============================================
CREATE TABLE document_tags (
    id              BIGSERIAL PRIMARY KEY,
    doc_id          BIGINT NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    tag_id          BIGINT NOT NULL REFERENCES tags(id) ON DELETE CASCADE,
    tagged_by       BIGINT REFERENCES users(id),
    tagged_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (doc_id, tag_id)
);

CREATE INDEX idx_document_tags_doc_id ON document_tags(doc_id);
CREATE INDEX idx_document_tags_tag_id ON document_tags(tag_id);

-- ============================================
-- 2.4.3 片段标签关联
-- ============================================
CREATE TABLE chunk_tags (
    id              BIGSERIAL PRIMARY KEY,
    chunk_id        BIGINT NOT NULL REFERENCES chunks(id) ON DELETE CASCADE,
    tag_id          BIGINT NOT NULL REFERENCES tags(id) ON DELETE CASCADE,
    tagged_by       BIGINT REFERENCES users(id),
    tagged_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (chunk_id, tag_id)
);

CREATE INDEX idx_chunk_tags_chunk_id ON chunk_tags(chunk_id);
CREATE INDEX idx_chunk_tags_tag_id ON chunk_tags(tag_id);
```

### 2.5 五级权限体系

```sql
-- ============================================
-- 2.5.1 文件类型权限
-- ============================================
CREATE TABLE file_type_permissions (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    file_type       VARCHAR(32) NOT NULL,       -- pdf, docx, xlsx, mp4, jpg 等
    permission      VARCHAR(16) NOT NULL
                        CHECK (permission IN ('deny', 'read', 'download', 'full')),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, file_type)
);

COMMENT ON TABLE file_type_permissions IS '文件类型级权限(一级)';
COMMENT ON COLUMN file_type_permissions.permission IS 'deny=禁止, read=仅预览, download=可下载, full=完全访问';

-- ============================================
-- 2.5.2 文档权限
-- ============================================
CREATE TABLE document_permissions (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    doc_id          BIGINT NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    permission      VARCHAR(16) NOT NULL
                        CHECK (permission IN ('deny', 'read', 'annotate', 'edit', 'admin')),
    granted_by      BIGINT REFERENCES users(id),
    expires_at      TIMESTAMPTZ,                -- 临时权限过期时间
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, doc_id)
);

COMMENT ON TABLE document_permissions IS '文档级权限(二级)';
COMMENT ON COLUMN document_permissions.permission IS 'deny=禁止, read=可读, annotate=可批注, edit=可编辑, admin=完全管理';

CREATE INDEX idx_doc_permissions_user_id ON document_permissions(user_id);
CREATE INDEX idx_doc_permissions_doc_id ON document_permissions(doc_id);
CREATE INDEX idx_doc_permissions_expires ON document_permissions(expires_at)
    WHERE expires_at IS NOT NULL;

-- ============================================
-- 2.5.3 字段权限
-- ============================================
CREATE TABLE field_permissions (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    doc_id          BIGINT NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    field_name      VARCHAR(64) NOT NULL,       -- content_text, meta_info.source, file_path 等
    permission      VARCHAR(16) NOT NULL
                        CHECK (permission IN ('deny', 'partial', 'read', 'edit')),
    mask_rule       JSONB,                      -- 脱敏规则，如 {"type": "regex", "pattern": "\\d{4}(\\d{8})"}
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, doc_id, field_name)
);

COMMENT ON TABLE field_permissions IS '字段级权限(三级)，支持细粒度字段控制和脱敏';
COMMENT ON COLUMN field_permissions.mask_rule IS '字段脱敏规则JSON';

CREATE INDEX idx_field_permissions_user_doc ON field_permissions(user_id, doc_id);

-- ============================================
-- 2.5.4 标签权限
-- ============================================
CREATE TABLE tag_permissions (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    tag_id          BIGINT NOT NULL REFERENCES tags(id) ON DELETE CASCADE,
    permission      VARCHAR(16) NOT NULL
                        CHECK (permission IN ('deny', 'read', 'use', 'manage')),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, tag_id)
);

COMMENT ON TABLE tag_permissions IS '标签级权限(四级)，控制用户对标签可见及使用权限';
COMMENT ON COLUMN tag_permissions.permission IS 'deny=禁止, read=可见, use=可使用, manage=可管理';

-- ============================================
CREATE TABLE group_permissions (
    id              BIGSERIAL PRIMARY KEY,

                        CHECK (permission IN ('deny', 'view', 'join', 'manage')),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, group_id)

COMMENT ON TABLE group_permissions IS '群组级权限(五级)，控制用户对群组的访问和操作';
COMMENT ON COLUMN group_permissions.permission IS 'deny=禁止, view=可查看, join=可加入, manage=可管理';
```

### 2.6 敏感关键词控制

```sql
-- ============================================
-- 2.6.1 敏感关键词分级表
-- ============================================
CREATE TABLE sensitive_keywords (
    id              BIGSERIAL PRIMARY KEY,
    keyword         VARCHAR(256) NOT NULL,
    level           INT NOT NULL DEFAULT 5
                        CHECK (level BETWEEN 0 AND 5),
    -- level: 0=绝密, 1=机密, 2=秘密, 3=内部, 4=受限, 5=公开
    category        VARCHAR(64) NOT NULL DEFAULT 'general',
    -- category: general, political, pornography, violence, privacy, commercial 等
    match_mode      VARCHAR(16) NOT NULL DEFAULT 'exact'
                        CHECK (match_mode IN ('exact', 'fuzzy', 'regex', 'semantic')),
    match_pattern   VARCHAR(512),               -- regex 或扩展匹配模式
    action          VARCHAR(32) NOT NULL DEFAULT 'block'
                        CHECK (action IN ('block', 'mask', 'audit', 'warn', 'upgrade')),
    -- action: block=阻断检索, mask=内容脱敏, audit=仅审计, warn=警告并记录, upgrade=升级审批
    description     TEXT,
    created_by      BIGINT REFERENCES users(id),
    is_active       BOOLEAN NOT NULL DEFAULT TRUE,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (keyword, level, category)
);

COMMENT ON TABLE sensitive_keywords IS '敏感关键词分级控制表';
COMMENT ON COLUMN sensitive_keywords.level IS '敏感等级(0=绝密, 5=公开)';
COMMENT ON COLUMN sensitive_keywords.match_mode IS '匹配模式：精确/模糊/正则/语义';
COMMENT ON COLUMN sensitive_keywords.action IS '触发动作：阻断/脱敏/审计/警告/升级';

CREATE INDEX idx_sensitive_keywords_level ON sensitive_keywords(level);
CREATE INDEX idx_sensitive_keywords_active ON sensitive_keywords(is_active) WHERE is_active = TRUE;
CREATE INDEX idx_sensitive_keywords_category ON sensitive_keywords(category);

-- ============================================
-- 2.6.2 关键词匹配记录(可选，用于审计)
-- ============================================
CREATE TABLE keyword_match_logs (
    id              BIGSERIAL PRIMARY KEY,
    keyword_id      BIGINT NOT NULL REFERENCES sensitive_keywords(id),
    chunk_id        BIGINT REFERENCES chunks(id) ON DELETE SET NULL,
```

## 3. Milvus 向量数据库 Schema

### 3.1 架构设计原则

| 原则 | 说明 |
|------|------|
| **权限字段冗余** | 向量记录中冗余存储 `doc_acl_version`, `max_keyword_level`, `tags`，实现检索时过滤 |
| **分集合存储** | 文本向量与图片向量独立集合，避免字段混乱 |
| **标量过滤优先** | 检索表达式先过滤权限和标签，再执行向量ANN搜索 |

### 3.4 Milvus 检索表达式示例

```python
from pymilvus import Collection

# 加载集合
text_col = Collection("text_collection")
text_col.load()

# 构建权限过滤表达式
user_max_level = 3          # 当前用户允许访问的最高敏感等级(数字越小越敏感)
allowed_doc_ids = [10, 20, 30, 40]   # 用户有权限的文档ID列表(由权限服务计算)

required_acl_version = 5    # 当前有效ACL版本

# 检索表达式：权限前置过滤
expr = f"""
    status == 'active'
    AND max_keyword_level >= {user_max_level}
    AND doc_acl_version == {required_acl_version}
    AND doc_id in {allowed_doc_ids}
    AND array_contains_any(tags, {allowed_tags})
"""

# 执行检索
results = text_col.search(
    data=[query_embedding],      # [1, 768] 查询向量
    anns_field="embedding",
    param={"metric_type": "COSINE", "params": {"ef": 128}},
    limit=100,
    expr=expr,                   # 标量过滤表达式
```

### 4.3 缓存读写示例

```python
import json
import redis
from typing import Dict, Optional

r = redis.Redis(host='localhost', port=6379, db=0, decode_responses=True)

# ============================================
# 4.3.1 文档权限缓存
# ============================================
def cache_doc_permissions(user_id: int, perms: Dict[int, str], ttl: int = 300):
    """
    缓存用户对各文档的权限
    perms: {doc_id: permission_str}
    """
    key = f"acl:doc_perm:{user_id}"
    if not perms:
        r.setex(key, ttl, "__empty__")
        return
    r.hset(key, mapping=perms)
    r.expire(key, ttl)

def get_doc_permission(user_id: int, doc_id: int) -> Optional[str]:
    """获取用户对指定文档的权限"""
    key = f"acl:doc_perm:{user_id}"
    perm = r.hget(key, str(doc_id))
    return perm

# ============================================
# 4.3.2 标签权限缓存
# ============================================
def cache_tag_permissions(user_id: int, perms: Dict[int, str], ttl: int = 300):
    """
    缓存用户对各标签的权限
    perms: {tag_id: permission_str}
    """
    key = f"acl:tag_perm:{user_id}"
    r.hset(key, mapping=perms)
    r.expire(key, ttl)

# ============================================
# 4.3.3 字段权限缓存
# ============================================
def cache_field_permissions(user_id: int, doc_id: int, fields: Dict[str, str], ttl: int = 300):
    """
    缓存用户对指定文档的各字段权限
    fields: {field_name: permission_str}
    """
    key = f"acl:field_perm:{user_id}:{doc_id}"
    r.hset(key, mapping=fields)
    r.expire(key, ttl)

# ============================================
# 4.3.4 文件类型权限缓存
# ============================================
def cache_file_type_permissions(user_id: int, types: Dict[str, str], ttl: int = 600):
    """
    缓存用户对各文件类型的权限
    types: {file_extension: permission_str}
    """
    key = f"acl:file_types:{user_id}"
    r.hset(key, mapping=types)
    r.expire(key, ttl)

# ============================================
# 4.3.5 ACL版本缓存
# ============================================
def cache_acl_version(doc_id: int, version: int, ttl: int = 60):
    """缓存文档ACL版本号，用于Milvus检索时校验"""
    key = f"acl:version:{doc_id}"
    r.setex(key, ttl, str(version))

def get_acl_version(doc_id: int) -> Optional[int]:
    val = r.get(f"acl:version:{doc_id}")
    return int(val) if val else None

# ============================================
# 4.3.6 用户基础权限缓存
# ============================================
def cache_user_perm_meta(user_id: int, meta: dict, ttl: int = 60):
    """
    缓存用户基础权限元数据
    meta: {role, max_keyword_level, kb_ids, group_ids}
    """
    key = f"user:perm:{user_id}"
    r.setex(key, ttl, json.dumps(meta))

def get_user_perm_meta(user_id: int) -> Optional[dict]:
    val = r.get(f"user:perm:{user_id}")
    return json.loads(val) if val else None
```

### 4.4 缓存失效策略

| 触发场景 | 失效操作 |
|----------|----------|
| 文档权限变更 | `DEL acl:doc_perm:*` + `INCR acl:version:{doc_id}` |
| 标签权限变更 | `DEL acl:tag_perm:*` |
| 字段权限变更 | `DEL acl:field_perm:*:{doc_id}` |
| 用户角色/等级变更 | `DEL user:perm:{user_id}` + `DEL acl:*:{user_id}` |
| 权限服务定时任务 | 全量扫描过期 Key，按 TTL 自然淘汰 |

```
│  │                                                                                 ││

                                        │

│  │  保留策略: 7~90 天数据                                                           ││
│  └─────────────────────────────────────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────────────────────────────────────┘
                                        │
                                        ▼ 自动归档策略

│  ┌─────────────────────────────────────────────────────────────────────────────────┐│

│  │  • 90 天以上历史数据                                                              ││
│  │  • 向量索引从 Milvus 卸载（保留原始向量在 MinIO 归档桶）                           ││
│  │  • 审计日志按月分区，旧分区可导出为 Parquet 压缩存储                               ││
│  │  • 需重新激活时才加载回温数据层                                                    ││
│  │                                                                                 ││
│  │  存储介质: HDD / 对象存储低频层 / 磁带库(可选)                                     ││
│  │  保留策略: 90 天以上至永久保留（合规要求）                                         ││
│  └─────────────────────────────────────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 各层数据分布

| 数据类型 | 热层 | 温层 | 冷层 | 迁移策略 |
|----------|------|------|------|----------|
| users | ✓ 全量 | — | — | 常驻热层 |
| documents (元数据) | ✓ 最近7天 | ✓ 7~90天 | ✓ 归档 | 按 updated_at |
| chunks (内容) | ✓ 最近7天 | ✓ 7~90天 | ✓ 压缩归档 | 按 created_at |
| chunks (向量) | ✓ Milvus 内存 | ✓ Milvus 磁盘 | ✓ MinIO 归档 | 按访问频率 |
| 原始文件 | ✓ MinIO 热桶 | ✓ MinIO 标准 | ✓ MinIO 低频 | 生命周期策略 |
| audit_logs | ✓ 当月分区 | ✓ 近3月分区 | ✓ 历史分区导出 | 按月滚动 |
| Redis 缓存 | ✓ 活跃数据 | — | — | TTL 淘汰 |

## 8. 数据一致性策略

### 8.1 一致性模型

| 场景 | 一致性要求 | 实现方式 |
|------|-----------|----------|
| 用户-权限关系 | 强一致性 | PostgreSQL 事务 + 缓存失效广播 |
| 文档-片段关系 | 强一致性 | PG 外键级联 + 软删除 |
| PG-Milvus 向量 | 最终一致性 | 异步 Worker 同步 + 状态机校验 |
| PG-Redis 缓存 | 最终一致性 | 写穿透 + TTL + 版本号校验 |
| PG-MinIO 文件 | 最终一致性 | 异步上传 + MD5 校验 + 定时清理 |

### 8.2 版本号一致性校验

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   PostgreSQL    │     │     Redis       │     │    Milvus       │
│                 │     │                 │     │                 │
│ documents.acl   │────►│ acl:version:55  │────►│ doc_acl_version │
│ _version = 5    │     │ = "5"           │     │ = 5             │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │
        │ 权限变更时:
        │ UPDATE documents SET acl_version = acl_version + 1 WHERE id = 55;
        │
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ 失效广播:                                                         │
│  1. DEL acl:doc_perm:*   (清除所有用户文档权限缓存)                │
│  2. SET acl:version:55 = "6"                                     │
│  3. 后台 perm-sync-worker 逐步更新 Milvus 中 doc_id=55 的向量记录 │
│     (期间旧版本记录因 doc_acl_version=5 ≠ 6 被检索表达式过滤)       │
└─────────────────────────────────────────────────────────────────┘
```

### 8.3 异常恢复流程

| 异常场景 | 检测方式 | 恢复动作 |
|----------|----------|----------|
| PG 写成功，Milvus 写失败 | chunks.vector_status='failed' | 定时任务重试索引 |
| Milvus 已索引，PG 记录丢失 | 孤儿向量检测脚本 | 标记删除或重新关联 |
| Redis 缓存与 PG 不一致 | acl_version 不匹配 | 强制刷新缓存，回源PG |
| MinIO 文件缺失 | 定时扫描 file_path | 标记 documents.status='failed' |
| 敏感等级扫描遗漏 | 增量关键词检测 | 重新扫描历史chunks并更新max_keyword_level |
