## 3. 检索评测

### 3.1 指标体系

| 指标 | 公式 | 说明 | 优秀值 |
|------|------|------|--------|
| **Recall@K** | 期望文档中被召回的比例 | 召回率 | > 0.85 |
| **Precision@K** | 召回结果中相关文档的比例 | 精确率 | > 0.70 |
| **MRR** | 1 / 第一个相关文档的排名 | 平均倒数排名 | > 0.75 |
| **NDCG@K** | 加权排序质量 | 归一化折损累计增益 | > 0.80 |
| **Hit Rate** | 至少召回1个相关文档的比例 | 命中率 | > 0.90 |

### 3.2 指标计算实现

```python
class RetrievalEvaluator:
    """检索评测器"""
    
    def evaluate(
        self,
        dataset: EvalDataset,
        retriever: Retriever,
        k_values: List[int] = [5, 10, 20]
    ) -> RetrievalMetrics:
        results = []
        
        for question in dataset.questions:
            # 执行检索
            retrieved = retriever.retrieve(question.query, top_k=max(k_values))
            retrieved_doc_ids = [r.doc_id for r in retrieved]
            
            # 计算各项指标
            for k in k_values:
                result = {
                    "question_id": question.id,
                    "k": k,
                    "recall": self._recall_at_k(
                        question.expected_docs, retrieved_doc_ids, k
                    ),
                    "precision": self._precision_at_k(
                        question.expected_docs, retrieved_doc_ids, k
                    ),
                    "mrr": self._mrr(question.expected_docs, retrieved_doc_ids),
                    "hit": self._hit_rate(
                        question.expected_docs, retrieved_doc_ids, k
                    )
                }
                results.append(result)
        
        # 聚合结果
        return self._aggregate_metrics(results, k_values)
    
    def _recall_at_k(
        self,
        expected: List[str],
        retrieved: List[str],
        k: int
    ) -> float:
        """Recall@K = |期望∩召回| / |期望|"""
        retrieved_k = set(retrieved[:k])
        expected_set = set(expected)
        if not expected_set:
            return 0.0
        return len(expected_set & retrieved_k) / len(expected_set)
    
    def _precision_at_k(
        self,
        expected: List[str],
        retrieved: List[str],
        k: int
    ) -> float:
        """Precision@K = |期望∩召回| / K"""
        retrieved_k = set(retrieved[:k])
        expected_set = set(expected)
        if k == 0:
            return 0.0
        return len(expected_set & retrieved_k) / k
    
    def _mrr(self, expected: List[str], retrieved: List[str]) -> float:
        """MRR = 1 / 第一个相关文档的排名"""
        expected_set = set(expected)
        for rank, doc_id in enumerate(retrieved, start=1):
            if doc_id in expected_set:
                return 1.0 / rank
        return 0.0
    
    def _ndcg_at_k(
        self,
        expected: List[str],
        retrieved: List[str],
        k: int
    ) -> float:
        """NDCG@K = DCG / IDCG"""
        expected_set = set(expected)
        
        # DCG = sum(relevance_i / log2(i+1))
        dcg = 0.0
        for i, doc_id in enumerate(retrieved[:k], start=1):
            rel = 1.0 if doc_id in expected_set else 0.0
            dcg += rel / math.log2(i + 1)
        
        # IDCG = ideal DCG (所有相关文档排在前面的情况)
        idcg = 0.0
        for i in range(min(len(expected), k)):
            idcg += 1.0 / math.log2(i + 2)
        
        return dcg / idcg if idcg > 0 else 0.0
```

## 4. 生成评测

### 4.1 指标体系

| 指标 | 说明 | 评测方式 |
|------|------|----------|
| **Answer Relevance** | 答案与问题的相关程度 | LLM-as-Judge |
| **Faithfulness** | 答案是否忠实于检索上下文 | LLM-as-Judge |
| **Context Precision** | 检索上下文中有用信息的比例 | 人工+LLM |
| **Context Recall** | 回答问题所需的上下文是否被召回 | LLM-as-Judge |
| **Answer Correctness** | 答案事实正确性 | 对比期望答案 |

### 4.2 LLM-as-Judge实现

```python
class LLMJudge:
    """使用minimax-m3作为评判模型"""
    
    JUDGE_PROMPT_TEMPLATE = """
你是一个严格的答案质量评判专家。请根据以下信息对AI助手的回答进行评分。

【检索上下文】
{context}

【用户问题】
{query}

【AI助手回答】
{answer}

请从以下维度进行评分（1-5分）：

1. 相关性 (Relevance): 答案是否直接回答了用户的问题？
   - 5分: 完全相关，直接回答
   - 3分: 部分相关
   - 1分: 完全不相关

2. 忠实度 (Faithfulness): 答案中的信息是否都能在检索上下文中找到依据？
   - 5分: 所有信息都有依据
   - 3分: 大部分有依据
   - 1分: 包含编造信息

3. 完整性 (Completeness): 答案是否完整覆盖了问题的所有方面？
   - 5分: 完全完整
   - 3分: 部分完整
   - 1分: 严重缺失

请以JSON格式输出评分：
{{
  "relevance": <1-5>,
  "faithfulness": <1-5>,
  "completeness": <1-5>,
  "overall": <1-5>,
  "reasoning": "简要说明评分的理由"
}}
"""
    
    def __init__(self, llm_client):
        self.llm = llm_client  # minimax-m3
    
    async def judge(
        self,
        query: str,
        answer: str,
        context: str
    ) -> JudgeResult:
        prompt = self.JUDGE_PROMPT_TEMPLATE.format(
            context=context,
            query=query,
            answer=answer
        )
        
        response = await self.llm.generate(prompt)
        
        # 解析JSON评分
        try:
            scores = json.loads(self._extract_json(response))
            return JudgeResult(**scores)
        except:
            # 解析失败时返回默认值
            return JudgeResult(
                relevance=3, faithfulness=3, completeness=3, overall=3,
                reasoning="解析失败"
            )
```

## 6. Bad Case分析

### 6.1 自动聚类

```python
class BadCaseAnalyzer:
    """Bad Case分析器"""
    
    def analyze(self, eval_results: List[EvalResult]) -> BadCaseReport:
        bad_cases = [r for r in eval_results if not r.is_pass]
        
        # 按失败原因分类
        categories = defaultdict(list)
        for case in bad_cases:
            reason = self._classify_failure_reason(case)
            categories[reason].append(case)
        
        # 按查询意图聚类
        intent_clusters = self._cluster_by_intent(bad_cases)
        
        return BadCaseReport(
            total_bad_cases=len(bad_cases),
            failure_categories=dict(categories),
            intent_clusters=intent_clusters,
            recommendations=self._generate_recommendations(categories)
        )
    
    def _classify_failure_reason(self, case: EvalResult) -> str:
        """分类失败原因"""
        if case.recall < 0.5:
            return "检索失败: 未召回相关文档"
        elif case.precision < 0.5:
            return "检索失败: 召回太多无关文档"
        elif case.faithfulness < 3:
            return "生成失败: 答案包含编造信息"
        elif case.relevance < 3:
            return "生成失败: 答案未回答用户问题"
        elif case.intercepted:
            return "权限误拦截: 合法查询被拦截"
        else:
            return "其他失败"
```

### 6.2 根因定位

```
Bad Case分析示例:

查询: "销售部Q3的营收目标"
失败原因: 检索失败（未召回相关文档）

根因分析:
1. 问题中的"Q3"和"营收目标"被分到了不同的Chunk
2. 用户使用了"营收"而非"销售额"，同义词未扩展
3. 相关文档是Excel表格，表格分块策略未覆盖"行级聚合描述"

优化建议:
1. 增加"营收"↔"销售额"的同义词映射
2. 优化Excel分块策略，增加按部门分组的区域Chunk
3. 调整查询改写模块，扩展时间表达
```

class FeedbackCollector:
    """反馈收集器"""
    
    async def collect_explicit_feedback(
        self,
        query_id: str,
        feedback_type: str,  # "thumbs_up" | "thumbs_down"

        self,

            "follow_up": metrics.follow_up,           # 是否追问
            "clicked_sources": metrics.clicked_sources  # 点击了哪些来源
        })

## 8. 趋势报告

```python
class TrendReport:
    """趋势报告生成器"""
    
    def generate(self, eval_history: List[EvalRun]) -> TrendReportData:
        """
        生成趋势报告
        """
        # 按时间排序
        sorted_runs = sorted(eval_history, key=lambda r: r.run_at)
        
        # 提取各指标的时间序列
        trends = {
            "recall@10": [r.metrics.recall_at_10 for r in sorted_runs],
            "mrr": [r.metrics.mrr for r in sorted_runs],
            "ndcg@10": [r.metrics.ndcg_at_10 for r in sorted_runs],
            "faithfulness": [r.metrics.faithfulness for r in sorted_runs],
        }
        
        # 计算趋势
        for metric, values in trends.items():
            if len(values) >= 2:
                first_half = sum(values[:len(values)//2]) / (len(values)//2)
                second_half = sum(values[len(values)//2:]) / (len(values) - len(values)//2)
                change_pct = (second_half - first_half) / first_half * 100
                
                print(f"{metric}: {change_pct:+.1f}% 变化")
        
        return TrendReportData(
            time_series=trends,
            runs=sorted_runs,
            summary=self._generate_summary(trends)
        )
```
