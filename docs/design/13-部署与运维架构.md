## 3. 监控指标与告警

### 3.1 监控指标清单

| 类别 | 指标名称 | 说明 | 告警阈值 |
|------|----------|------|----------|
| **API性能** | 请求延迟P99 | API响应时间 | > 2s |
| | 错误率 | 5xx比例 | > 1% |
| | QPS | 每秒请求数 | 按容量规划 |
| | 并发连接数 | 同时连接数 | > 80%容量 |
| **检索质量** | 平均召回分数 | 检索效果 | 下降趋势 |
| | Re-rank命中率 | 重排序效果 | < 80% |
| | 空结果率 | 无结果查询比例 | > 10% |
| **系统资源** | CPU使用率 | 节点CPU | > 80% |
| | 内存使用率 | 节点内存 | > 85% |
| | GPU显存 | GPU使用率 | > 90% |
| | 磁盘使用率 | 存储使用 | > 85% |
| | 网络I/O | 带宽使用 | > 80%容量 |
| **业务指标** | 文档处理队列堆积 | 待处理文档数 | > 1000 |
| | 索引延迟 | 文档到可检索时间 | > 5min |
| | 权限检查延迟 | 单次权限查询时间 | > 100ms |
| | API安全拦截率 | 被拦截的请求比例 | 监控趋势 |

### 3.2 Prometheus告警规则

```yaml
# alerting-rules.yml
groups:
- name: rag-alerts
  rules:
  - alert: HighAPIErrorRate
    expr: rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m]) > 0.01
    for: 2m
    labels:
      severity: critical
    annotations:
      summary: "API错误率过高"
      description: "错误率: {{ $value | humanizePercentage }}"

  - alert: HighLatencyP99
    expr: histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[5m])) > 2
    for: 3m
    labels:
      severity: warning
    annotations:
      summary: "API P99延迟过高"
      description: "P99延迟: {{ $value }}s"

  - alert: DocumentQueueBacklog
    expr: celery_tasks_pending{queue="ingest"} > 1000
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "文档处理队列堆积"
      description: "堆积任务: {{ $value }}"

  - alert: PermissionCheckSlow
    expr: histogram_quantile(0.95, rate(permission_check_duration_seconds_bucket[5m])) > 0.1
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "权限检查延迟高"
      description: "P95延迟: {{ $value }}s"

  - alert: DiskSpaceLow
    expr: (node_filesystem_avail_bytes / node_filesystem_size_bytes) < 0.15
    for: 5m
    labels:
      severity: critical
    annotations:
      summary: "磁盘空间不足"
      description: "剩余空间: {{ $value | humanizePercentage }}"
```

## 6. CI/CD流水线

```yaml
# .github/workflows/ci.yml
name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Install dependencies
        run: |
          pip install -r backend/requirements.txt
          pip install pytest pytest-asyncio
      
      - name: Run unit tests
        run: pytest backend/tests/ -v --cov=backend/src --cov-report=xml
      
      - name: Run lint
        run: |
          flake8 backend/src
          black --check backend/src
      
      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      - name: Build frontend
        run: |
          cd frontend
          npm ci
          npm run build
      
      - name: Frontend tests
        run: |
          cd frontend
          npm test -- --coverage

  build:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      
      - name: Build and push images
        run: |
          docker build -t registry.company.com/rag/app-backend:${{ github.sha }} ./backend
          docker build -t registry.company.com/rag/frontend:${{ github.sha }} ./frontend
          docker push registry.company.com/rag/app-backend:${{ github.sha }}
          docker push registry.company.com/rag/frontend:${{ github.sha }}

  deploy:
    needs: build
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      
      - name: Deploy to K8s
        run: |
          kubectl set image deployment/app-backend app-backend=registry.company.com/rag/app-backend:${{ github.sha }} -n rag-system
          kubectl set image deployment/frontend frontend=registry.company.com/rag/frontend:${{ github.sha }} -n rag-system
          kubectl rollout status deployment/app-backend -n rag-system
          kubectl rollout status deployment/frontend -n rag-system
```

## 7. 故障恢复

| 故障场景 | 影响 | 检测方式 | 恢复策略 | RTO |
|----------|------|----------|----------|-----|
| **PostgreSQL主库故障** | 写操作中断 | 健康检查 | 自动切换至standby | < 1min |
| **Milvus节点故障** | 检索延迟增加 | 心跳检测 | 自动重平衡至其他节点 | < 30s |
| **Redis故障** | 缓存失效 | 连接检测 | 降级为直连DB，重启Redis | < 2min |
| **RabbitMQ故障** | 异步任务堆积 | 连接检测 | 任务本地队列缓冲，重启MQ | < 3min |
| **MinIO节点故障** | 无（多副本） | 心跳检测 | 自动Healing | < 5min |
| **embed-worker故障** | 向量化延迟 | 任务堆积告警 | 重启Worker，任务重分配 | < 5min |
| **minimax-m3 API故障** | 生成服务中断 | 超时检测 | 降级为本地模型/缓存回复 | < 10s |
