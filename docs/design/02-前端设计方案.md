## 1. 前端架构

### 1.1 技术栈选型

| 层级 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| 框架 | React | 18.x | 函数组件 + Hooks 为主 |
| 语言 | TypeScript | 5.x | 严格模式，全量类型覆盖 |
| UI库 | Ant Design | 5.x | 企业级组件，主题定制 |
| 状态管理 | Zustand | 4.x | 轻量、无样板代码、支持中间件 |

| 网络请求 | Axios | 1.x | 拦截器统一处理认证/错误 |
| 构建工具 | Vite | 5.x | 极速HMR，Rollup打包 |
| 代码规范 | ESLint + Prettier | - | Airbnb规范 + 自定义规则 |

## 2. 上传中心设计

### 2.1 支持的文件类型与处理策略

| 文件类型 | 扩展名 | 处理方式 | 预览方式 | 参与检索 | 备注 |
|----------|--------|----------|----------|----------|------|
| **文档** | .doc, .docx | 解析为结构化段落+表格；提取样式层级 | 在线Word预览（mammoth.js渲染HTML） | ✅ 是 | 保留标题层级用于权限控制 |
| **PDF** | .pdf | 页面渲染+文本提取+布局分析；生成页码映射 | PDF.js分页预览 + 文本层叠加 | ✅ 是 | 支持页码级引用 |
| **表格** | .xls, .xlsx, .csv | 工作表解析+单元格级索引；识别表头/数据区域 | 在线表格预览（虚拟滚动表格） | ✅ 是 | 支持字段级权限 |
| **图片** | .jpg, .png, .gif, .webp, .bmp | 保留原图+生成缩略图+OCR文字识别+Vision LLM描述 | 图片画廊（缩略图网格 + 灯箱放大） | ✅ 是 | 多维度向量索引 |
| **音频** | .mp3, .wav, .m4a, .flac, .ogg | 仅存储，不入检索索引；状态标记 `stored_only` | 音频播放器（波形图 + 播放控制） | ❌ 否 | 后续迭代引入ASR后可转检索 |
| **视频** | .mp4, .avi, .mov, .mkv, .wmv | 关键帧提取+Vision LLM画面描述+音频轨预留 | 视频播放器+时间轴+关键帧缩略图 | ✅ 是 | 15s切片策略 |
| **网页链接** | URL | 网页抓取+正文提取+链接去重 | iframe预览 / 快照缩略图 | ✅ 是 | 支持定期重新抓取 |
| **压缩包** | .zip, .rar, .7z | 解压后递归处理内部文件；生成文件树 | 文件树预览（可展开/折叠） | ✅ 是 | 内部文件独立参与检索 |

### 2.2 上传流程设计

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           6步上传处理流程                                    │
└─────────────────────────────────────────────────────────────────────────────┘

    用户拖拽/点击选择文件
              │
              ▼
    ┌─────────────────┐
    │  Step 1: 格式校验 │ ◄──── 前端快速校验（扩展名 + MIME类型 + 大小限制）
    │  ───────────────  │         非法格式 ──► 拒绝并提示具体原因
    │  • 扩展名白名单    │         超大文件 ──► 触发分片上传标记
    │  • MIME类型匹配    │
    │  • 单文件大小限制  │
    │  • 批量总数限制    │
    └─────────────────┘
              │ 校验通过
              ▼
    ┌─────────────────┐
    │  Step 2: 病毒扫描 │ ◄──── 后端集成ClamAV / 第三方扫描API
    │  ───────────────  │         风险文件 ──► 隔离区 + 告警通知
    │  • 哈希黑名单比对  │
    │  • 实时病毒扫描    │
    │  • 敏感内容检测    │
    └─────────────────┘
              │ 扫描通过
              ▼
    ┌─────────────────┐
    │  Step 3: 分片上传 │ ◄──── MinIO S3 Multipart Upload API
    │  ───────────────  │         网络中断 ──► 断点续传恢复
    │  • 计算文件MD5    │         秒传命中 ──► 跳过上传直接关联
    │  • 分片大小: 5MB  │
    │  • 并发上传: 3片  │
    │  • 断点续传标记    │
    └─────────────────┘
              │ 上传完成
              ▼
    ┌─────────────────┐
    │  Step 4: 异步入队 │ ◄──── 提交到RabbitMQ / Celery任务队列
    │  ───────────────  │
    │  • 生成任务ID     │
    │  • 解析路由分发    │   ──► 文档Pipeline
    │  • 优先级队列     │   ──► 图片Pipeline
    │  • 死信队列兜底   │   ──► 视频Pipeline
    └─────────────────┘       ──► ...
              │
              ▼
    ┌─────────────────┐
    │  Step 5: WebSocket│ ◄──── 实时推送处理进度
    │  实时进度推送     │
    │  ───────────────  │         前端订阅: ws://api/upload/{taskId}/progress
    │  • 上传进度 0-30% │         事件类型:
    │  • 解析进度 30-60%│           - progress: { stage, percent, detail }
    │  • 索引进度 60-90%│           - completed: { docId, status }
    │  • 完成 90-100%   │           - error: { code, message, retryable }
    └─────────────────┘
              │
              ▼
    ┌─────────────────┐
    │  Step 6: 处理完成 │ ◄──── 入库 + 索引 + 通知
    │  ───────────────  │
    │  • 状态更新: done │         成功 ──► 知识库列表刷新 + 通知中心消息
    │  • 元数据入库     │         失败 ──► 错误详情页 + 重试按钮
    │  • 向量索引完成   │         部分 ──► 警告状态（如某sheet解析失败）
    │  • 通知中心推送   │
    └─────────────────┘
```

// src/hooks/useUploadWebSocket.ts
import { useEffect, useRef, useCallback } from 'react';
import { useUploadStore } from '@/stores/useUploadStore';

const WS_URL = import.meta.env.VITE_WS_URL || 'ws://localhost:8080/ws/upload';

export const useUploadWebSocket = () => {
  const wsRef = useRef<WebSocket | null>(null);
  const { updateTaskProgress, completeTask, failTask } = useUploadStore();

  useEffect(() => {
    const ws = new WebSocket(WS_URL);
    wsRef.current = ws;

    ws.onopen = () => {
      console.log('[WebSocket] 上传进度通道已连接');
    };

    ws.onmessage = (event) => {
      const message = JSON.parse(event.data);
      switch (message.type) {
        case 'progress':
          updateTaskProgress(message.taskId, message.stage, message.percent, message.detail);
          break;
        case 'completed':
          completeTask(message.taskId, message.result);
          break;
        case 'error':
          failTask(message.taskId, message.error);
          break;
      }
    };

    ws.onerror = (err) => {
      console.error('[WebSocket] 连接错误:', err);
    };

    ws.onclose = () => {
      console.log('[WebSocket] 连接已关闭');
    };

    return () => {
      ws.close();
    };
  }, [updateTaskProgress, completeTask, failTask]);

  const subscribeTask = useCallback((taskId: string) => {
    if (wsRef.current?.readyState === WebSocket.OPEN) {
      wsRef.current.send(JSON.stringify({ action: 'subscribe', taskId }));
    }
  }, []);

  return { subscribeTask };
};

// src/stores/useUploadStore.ts
import { create } from 'zustand';
import { immer } from 'zustand/middleware/immer';
import { UploadTask, UploadStage } from '@/types/upload';

interface UploadState {
  queue: UploadTask[];
  activeCount: number;
  addTask: (file: File) => string;
  updateTaskProgress: (id: string, stage: UploadStage, progress: number, detail?: string) => void;
  completeTask: (id: string, result: { docId: string; url: string }) => void;
  failTask: (id: string, error: { code: string; message: string; retryable: boolean }) => void;
  cancelTask: (id: string) => void;
  removeTask: (id: string) => void;
}

export const useUploadStore = create<UploadState>()(
  immer((set) => ({
    queue: [],
    activeCount: 0,

    addTask: (file) => {
      const id = `upload_${Date.now()}_${Math.random().toString(36).slice(2, 9)}`;
      set((state) => {
        state.queue.unshift({
          id,
          file,
          category: getFileCategory(file.name),
          status: UploadStage.VALIDATING,
          progress: 0,
          createdAt: Date.now(),
          updatedAt: Date.now(),
        });
        state.activeCount += 1;
      });
      return id;
    },

    updateTaskProgress: (id, stage, progress, detail) => {
      set((state) => {
        const task = state.queue.find((t) => t.id === id);
        if (task) {
          task.status = stage;
          task.progress = progress;
          task.stageDetail = detail;
          task.updatedAt = Date.now();
        }
      });
    },

    completeTask: (id, result) => {
      set((state) => {
        const task = state.queue.find((t) => t.id === id);
        if (task) {
          task.status = UploadStage.COMPLETED;
          task.progress = 100;
          task.result = result;
          task.updatedAt = Date.now();
          state.activeCount -= 1;
        }
      });
    },

    failTask: (id, error) => {
      set((state) => {
        const task = state.queue.find((t) => t.id === id);
        if (task) {
          task.status = UploadStage.FAILED;
          task.error = error;
          task.updatedAt = Date.now();
          state.activeCount -= 1;
        }
      });
    },

    cancelTask: (id) => {
      set((state) => {
        const task = state.queue.find((t) => t.id === id);
        if (task && task.status !== UploadStage.COMPLETED) {
          task.status = UploadStage.CANCELLED;
          state.activeCount -= 1;
        }
      });
    },

    removeTask: (id) => {
      set((state) => {
        const idx = state.queue.findIndex((t) => t.id === id);
        if (idx !== -1) {
          const task = state.queue[idx];
          if (task.status !== UploadStage.COMPLETED && task.status !== UploadStage.FAILED) {
            state.activeCount -= 1;
          }
          state.queue.splice(idx, 1);
        }
      });
    },
  }))
);

### 3.2 评测指标体系

#### 3.2.1 检索质量指标（Retrieval Metrics）

| 指标名称 | 英文 | 计算公式 | 含义说明 | 目标值 |
|----------|------|----------|----------|--------|
| **Recall@K** | Recall@K | `|Rel ∩ Retrieved_K| / |Rel|` | Top-K结果中相关文档占所有相关文档的比例 | ≥ 0.85 |
| **Precision@K** | Precision@K | `|Rel ∩ Retrieved_K| / K` | Top-K结果中相关文档所占比例 | ≥ 0.70 |
| **MRR** | Mean Reciprocal Rank | `avg(1/rank_i)` | 首个相关文档排名的倒数均值，越接近1越好 | ≥ 0.75 |
| **NDCG@K** | Normalized DCG | `DCG@K / IDCG@K` | 考虑排序位置的相关性加权指标 | ≥ 0.80 |
| **Hit Rate** | Hit Rate@K | `hits / total_queries` | 至少召回1个相关文档的查询占比 | ≥ 0.90 |

#### 3.2.2 生成质量指标（Generation Metrics）

| 指标名称 | 英文 | 评估方式 | 含义说明 | 评分范围 |
|----------|------|----------|----------|----------|
| **答案相关性** | Answer Relevance | LLM-as-Judge | 答案与问题的相关程度 | 1-5分 |
| **忠实度** | Faithfulness | LLM-as-Judge + 规则 | 答案是否基于检索内容，无 hallucination | 0-1分 |
| **上下文精确度** | Context Precision | 公式计算 | 检索到的上下文中与问题相关的比例 | 0-1分 |
| **上下文召回率** | Context Recall | 公式计算 | 问题所需信息在检索上下文中的覆盖程度 | 0-1分 |
| **答案正确性** | Answer Correctness | LLM-as-Judge / 人工 | 答案事实准确性（对比标准答案） | 0-1分 |

// src/stores/usePermissionStore.ts
import { create } from 'zustand';
import { immer } from 'zustand/middleware/immer';
import { RolePermissionConfig, Action, FileType } from '@/types/permission';

interface PermissionState {
  roles: RolePermissionConfig[];
  currentRoleId: string | null;
  draft: RolePermissionConfig | null;
  isDirty: boolean;

  setCurrentRole: (roleId: string) => void;
  updateFileTypeAction: (fileType: FileType, action: Action, granted: boolean) => void;
  updateDocPermission: (updates: Partial<DocPermission>) => void;
  addFieldPermission: (fieldPerm: FieldPermission) => void;
  updateTagPermission: (allowed: string[], denied: string[]) => void;
  saveDraft: () => Promise<void>;
  resetDraft: () => void;
}

export const usePermissionStore = create<PermissionState>()(
  immer((set, get) => ({
    roles: [],
    currentRoleId: null,
    draft: null,
    isDirty: false,

    setCurrentRole: (roleId) => {
      const role = get().roles.find((r) => r.roleId === roleId);
      if (role) {
        set((state) => {
          state.currentRoleId = roleId;
          state.draft = JSON.parse(JSON.stringify(role));
          state.isDirty = false;
        });
      }
    },

    updateFileTypeAction: (fileType, action, granted) => {
      set((state) => {
        if (!state.draft) return;
        const ftp = state.draft.fileTypePermissions.find((p) => p.fileType === fileType);
        if (ftp) {
          if (granted && !ftp.actions.includes(action)) {
            ftp.actions.push(action);
          } else if (!granted) {
            ftp.actions = ftp.actions.filter((a) => a !== action);
          }
        } else if (granted) {
          state.draft.fileTypePermissions.push({ fileType, actions: [action] });
        }
        state.isDirty = true;
      });
    },

    updateDocPermission: (updates) => {
      set((state) => {
        if (state.draft) {
          Object.assign(state.draft.docPermission, updates);
          state.isDirty = true;
        }
      });
    },

    addFieldPermission: (fieldPerm) => {
      set((state) => {
        if (state.draft) {
          const idx = state.draft.fieldPermissions.findIndex(
            (p) => p.docId === fieldPerm.docId
          );
          if (idx >= 0) {
            state.draft.fieldPermissions[idx] = fieldPerm;
          } else {
            state.draft.fieldPermissions.push(fieldPerm);
          }
          state.isDirty = true;
        }
      });
    },

    updateTagPermission: (allowed, denied) => {
      set((state) => {
        if (state.draft) {
          state.draft.tagPermission.allowedTagIds = allowed;
          state.draft.tagPermission.deniedTagIds = denied;
          state.isDirty = true;
        }
      });
    },

    saveDraft: async () => {
      const { draft } = get();
      if (!draft) return;
      // TODO: 调用API保存
      // await permissionApi.saveRoleConfig(draft);
      set((state) => {
        const idx = state.roles.findIndex((r) => r.roleId === draft.roleId);
        if (idx >= 0) {
          state.roles[idx] = JSON.parse(JSON.stringify(draft));
        }
        state.isDirty = false;
      });
    },

    resetDraft: () => {
      const { currentRoleId, roles } = get();
      const role = roles.find((r) => r.roleId === currentRoleId);
      if (role) {
        set((state) => {
          state.draft = JSON.parse(JSON.stringify(role));
          state.isDirty = false;
        });
      }
    },
  }))
);

// src/hooks/usePermission.ts
import { useCallback } from 'react';
import { useUserStore } from '@/stores/useUserStore';
import { FileType, Action } from '@/types/permission';

export const usePermission = () => {
  const { user, permissions } = useUserStore();

  const can = useCallback(
    (action: Action, resource: { type: FileType } | { docId: string } | { tagId: string }) => {
      if (!permissions) return false;
      // Level 1: 文件类型权限检查
      if ('type' in resource) {
        const ftp = permissions.fileTypePermissions.find((p) => p.fileType === resource.type);
        return ftp ? ftp.actions.includes(action) : false;
      }
      // Level 2/3/4: 更细粒度检查...
      return true;
    },
    [permissions]
  );

  const canUpload = useCallback(
    (fileType: FileType) => can(Action.UPLOAD, { type: fileType }),
    [can]
  );

  const canRead = useCallback(
    (fileType: FileType) => can(Action.READ, { type: fileType }),
    [can]
  );

  return { user, can, canUpload, canRead };
};

### 6.2 状态管理规范

#### 6.2.1 Zustand Store 设计模式

```typescript
// Store设计原则：
// 1. 按业务域拆分Store，避免单一Store过大
// 2. 使用 immer middleware 保证不可变性
// 3. 异步操作封装在 Store actions 中，组件只调用 action
// 4. 选择器使用浅比较，避免不必要的重渲染

// src/stores/useSearchStore.ts
import { create } from 'zustand';
import { immer } from 'zustand/middleware/immer';
import { devtools } from 'zustand/middleware';
import { SearchResult, SearchFilter } from '@/types/search';
import { searchApi } from '@/api/search';

interface SearchState {
  // State
  query: string;
  results: SearchResult[];
  filters: SearchFilter;
  loading: boolean;
  error: string | null;
  total: number;
  page: number;
  pageSize: number;

  // Actions
  setQuery: (query: string) => void;
  setFilters: (filters: Partial<SearchFilter>) => void;
  search: () => Promise<void>;
  loadMore: () => Promise<void>;
  reset: () => void;
}

const initialState = {
  query: '',
  results: [],
  filters: { fileTypes: [], tags: [], dateRange: null },
  loading: false,
  error: null,
  total: 0,
  page: 1,
  pageSize: 20,
};

export const useSearchStore = create<SearchState>()(
  devtools(
    immer((set, get) => ({
      ...initialState,

      setQuery: (query) => {
        set((state) => {
          state.query = query;
          state.page = 1;
        });
      },

      setFilters: (filters) => {
        set((state) => {
          Object.assign(state.filters, filters);
          state.page = 1;
        });
      },

      search: async () => {
        const { query, filters, page, pageSize } = get();
        if (!query.trim()) return;

        set((state) => {
          state.loading = true;
          state.error = null;
          state.results = [];
        });

        try {
          const res = await searchApi.search({ query, filters, page, pageSize });
          set((state) => {
            state.results = res.data.items;
            state.total = res.data.total;
            state.loading = false;
          });
        } catch (err: any) {
          set((state) => {
            state.error = err.message || '检索失败';
            state.loading = false;
          });
        }
      },

      loadMore: async () => {
        const { query, filters, page, pageSize, results, total } = get();
        if (results.length >= total) return;

        const nextPage = page + 1;
        set((state) => {
          state.loading = true;
        });

        try {
          const res = await searchApi.search({ query, filters, page: nextPage, pageSize });
          set((state) => {
            state.results.push(...res.data.items);
            state.page = nextPage;
            state.loading = false;
          });
        } catch (err: any) {
          set((state) => {
            state.error = err.message;
            state.loading = false;
          });
        }
      },

      reset: () => {
        set((state) => {
          Object.assign(state, initialState);
        });
      },
    })),
    { name: 'search-store' }
  )
);
```

  baseURL: API_BASE_URL,
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
  },

// 请求拦截器：注入Token
apiClient.interceptors.request.use(
  (config: InternalAxiosRequestConfig) => {
    const token = useUserStore.getState().token;
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

// 响应拦截器：统一错误处理
apiClient.interceptors.response.use(
  (response) => response.data,
  (error: AxiosError) => {
    const status = error.response?.status;
    const data = error.response?.data as any;

    switch (status) {
      case 401:
        message.error('登录已过期，请重新登录');
        useUserStore.getState().logout();
        window.location.href = '/login';
        break;
      case 403:
        message.error('无权限执行此操作');
        break;
      case 422:
        message.error(`参数错误: ${data?.message || '请检查输入'}`);
        break;
      case 500:
        message.error('服务器内部错误，请联系管理员');
        break;
      default:
        message.error(data?.message || '网络错误，请稍后重试');
    }

    return Promise.reject(error);
  }
);
