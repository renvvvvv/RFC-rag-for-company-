## 1. 设计目标

### 1.1 问题背景

企业知识库中包含大量敏感信息：员工薪资、并购重组、裁员计划、客户名单、核心源代码、未公开专利等。传统的文档级权限控制（"用户能否访问文档D"）粒度太粗，无法阻止以下风险场景：

- **场景A**: 一份全员可读的《公司年度总结》中，意外嵌入了"Q4裁员30%"的段落
- **场景B**: 普通员工通过RAG问答，让LLM从多个公开文档中推理出"某核心客户流失"
- **场景C**: 权限系统允许访问某技术文档，但文档中包含核心算法专利的详细描述

### 1.2 设计目标

关键词分级敏感控制是**独立于文档权限的另一维度**的安全防线：

| 维度 | 文档权限系统 | 关键词敏感控制系统 |
|------|-------------|-------------------|
| **控制粒度** | 文档/字段/标签级 | 内容语义级 |
| **决策依据** | 用户角色+资源属性 | 内容敏感度+用户 clearance |
| **拦截时机** | 检索前、检索中 | Chunk入库标注 + 检索过滤 + 生成拦截 |
| **动态性** | 相对静态（按文档配置） | 全局动态更新，即时生效 |
| **防护目标** | "用户能看哪些文档" | "用户能看到什么级别的内容" |

**核心设计原则：**

```
纵深防御: 文档权限(L1-L4) 是第一道防线，关键词敏感控制是第二道防线
              ↓
    Chunk入库时标注 max_keyword_level（静态预计算）
              ↓
    检索时过滤 chunk_max_level > user_effective_level（动态拦截）
              ↓
    生成后校验答案中是否泄露越级信息（最终防线）
```

```python
from pydantic import BaseModel, Field
from typing import List, Optional, Set
from enum import Enum
from datetime import datetime

class KeywordLevel(int, Enum):
    """敏感级别定义"""
    L0_PUBLIC = 0       # 公开 - 任何人可访问
    L1_INTERNAL = 1     # 内部 - 企业内部员工
    L2_CONFIDENTIAL = 2 # 保密 - 特定部门/项目组
    L3_SECRET = 3       # 机密 - 高管/核心人员
    L4_TOP_SECRET = 4   # 绝密 - 极少数授权人员

class MatchType(str, Enum):
    """匹配规则类型"""
    EXACT = "exact"           # 精确匹配: "薪资" 只匹配 "薪资"
    SUBSTRING = "substring"   # 子串匹配: "薪资" 匹配 "薪资待遇"
    REGEX = "regex"           # 正则匹配: 复杂模式
    FUZZY = "fuzzy"           # 模糊匹配: 同音/形近字 (拼音相似度>0.85)

class Action(str, Enum):
    """命中后的动作"""
    BLOCK = "block"           # 完全拦截，不可见
    MASK = "mask"             # 脱敏显示: "张**" / "****公司"
    AUDIT = "audit"           # 记录审计日志，但允许访问
    WARN = "warn"             # 附加警告标识

class Modality(str, Enum):
    """作用模态"""
    DOCUMENT = "document"
    EXCEL = "excel"
    IMAGE = "image"
    VIDEO = "video"
    LINK = "link"
    ALL = "all"

class SensitiveKeyword(BaseModel):
    """
    全局敏感关键词配置模型
    
    由系统管理员在安全后台配置，全局生效。
    支持变体词、分类归属、多模态作用范围、差异化处置动作。
    """
    id: str = Field(..., description="全局唯一标识")
    keyword: str = Field(..., description="主关键词", max_length=100)
    level: KeywordLevel = Field(..., description="敏感级别 0-4")
    category: str = Field(..., description="分类: 人事/财务/技术/法务/战略/...")
    
    # 匹配规则
    match_type: MatchType = Field(default=MatchType.SUBSTRING, description="匹配类型")
    match_pattern: Optional[str] = Field(default=None, description="正则模式(match_type=regex时生效)")
    
    # 变体词库
    variants: List[str] = Field(default=[], description="变体词列表: 同义词/缩写/黑话")
    
    # 作用范围
    apply_to_modalities: List[Modality] = Field(default=[Modality.ALL], description="生效模态")
    apply_to_kbs: Optional[List[str]] = Field(default=None, description="限定知识库(空=全局)")
    
    # 处置动作
    action: Action = Field(default=Action.BLOCK, description="命中后的默认动作")
    
    # 元信息
    created_by: str = Field(..., description="创建者")
    created_at: datetime = Field(default_factory=datetime.utcnow)
    updated_at: datetime = Field(default_factory=datetime.utcnow)
    enabled: bool = Field(default=True, description="是否启用")
    
    # 扩展属性
    description: Optional[str] = Field(default=None, description="业务说明")
    confidence_threshold: float = Field(default=1.0, description="模糊匹配置信度阈值(0-1)")

# 用户Clearance级别定义
class UserClearance(BaseModel):
    """用户敏感内容访问许可级别"""
    user_id: str
    max_keyword_level: KeywordLevel = KeywordLevel.L0_PUBLIC
    
    # 细粒度例外: 允许访问特定高敏分类
    category_exceptions: List[str] = Field(default=[], description="例外分类")
    
    # 细粒度例外: 允许访问特定关键词(白名单)
    keyword_whitelist: Set[str] = Field(default=set(), description="关键词白名单")
    
    # 生效时间窗口 (用于临时授权)
    valid_from: Optional[datetime] = None
    valid_until: Optional[datetime] = None
    
    @property
    def effective_level(self) -> int:
        """计算当前有效级别"""
        now = datetime.utcnow()
        if self.valid_from and now < self.valid_from:
            return KeywordLevel.L0_PUBLIC.value
        if self.valid_until and now > self.valid_until:
            return KeywordLevel.L0_PUBLIC.value
        return self.max_keyword_level.value
```

### 3.2 变体词匹配策略

```python
class VariantMatcher:
    """
    关键词变体匹配器
    
    支持多维度变体扩展:
    1. 同义词扩展: "薪资" → ["薪酬", "工资", "年收入"]
    2. 英文对照: "并购" → ["M&A", "acquisition", "merger"]
    3. 行业黑话: "裁员" → ["优化", "缩编", "结构性调整"]
    4. 拼音模糊: "cesuan" → "测算" (FUZZY模式下)
    """
    
    def __init__(self, keyword: SensitiveKeyword):
        self.keyword = keyword
        self.variant_set = self._build_variant_set()
    
    def _build_variant_set(self) -> Set[str]:
        """构建包含主词和变体的完整匹配集合"""
        variants = {self.keyword.keyword.lower()}
        variants.update(v.lower() for v in self.keyword.variants)
        return variants
    
    def match(self, text: str) -> bool:
        """对给定文本进行匹配检测"""
        text_lower = text.lower()
        
        if self.keyword.match_type == MatchType.EXACT:
            return self.keyword.keyword.lower() in text_lower.split()
        
        elif self.keyword.match_type == MatchType.SUBSTRING:
            return any(v in text_lower for v in self.variant_set)
        
        elif self.keyword.match_type == MatchType.REGEX:
            if self.keyword.match_pattern:
                import re
                return bool(re.search(self.keyword.match_pattern, text))
            return False
        
        elif self.keyword.match_type == MatchType.FUZZY:
            return self._fuzzy_match(text_lower)
        
        return False
    
    def _fuzzy_match(self, text: str) -> bool:
        """基于拼音相似度的模糊匹配"""
        # 使用 pypinyin 计算拼音相似度
        from pypinyin import lazy_pinyin
        
        text_pinyin = ''.join(lazy_pinyin(text))
        for variant in self.variant_set:
            variant_pinyin = ''.join(lazy_pinyin(variant))
            # 编辑距离相似度
            similarity = self._seq_similarity(text_pinyin, variant_pinyin)
            if similarity >= self.keyword.confidence_threshold:
                return True
        return False
    
    @staticmethod
    def _seq_similarity(a: str, b: str) -> float:
        """序列相似度 (1 - 归一化编辑距离)"""
        import difflib
        return difflib.SequenceMatcher(None, a, b).ratio()
```

## 4. Chunk入库标注流程

### 4.1 整体流程

```
文档解析完成 → 生成Chunk列表
       |
       v
+---------------------------------------------------+
| Step 1: 加载全局关键词库                           |
|   - 从PostgreSQL读取所有 enabled=True 的关键词     |
|   - 按 match_type 分组预处理                        |
|   - 构建 AC自动机 (用于 SUBSTRING/EXACT 类型)      |
+---------------------------------------------------+
       |
       v
+---------------------------------------------------+
| Step 2: 批量关键词匹配                             |
|   - 对每个Chunk的 content 执行多模式匹配           |
|   - 同时匹配: 主词 + 变体词 + 正则模式             |
|   - 记录匹配到的关键词ID、级别、位置                 |
+---------------------------------------------------+
       |
       v
+---------------------------------------------------+
| Step 3: 计算 Chunk 敏感元数据                      |
|   - sensitive_keywords: [{kw_id, keyword, level}]  |
|   - max_keyword_level: max(命中关键词级别)          |
|   - keyword_categories: 命中关键词的分类集合         |
+---------------------------------------------------+
       |
       v
+---------------------------------------------------+
| Step 4: 向量化与入库                               |
|   - Chunk元数据写入 PostgreSQL                     |
|   - Dense/Sparse向量写入 Milvus                    |
|   - metadata中携带 max_keyword_level 等字段         |
+---------------------------------------------------+
```

### 4.2 AC自动机高效多模式匹配

```python
from typing import List, Dict, Tuple, Optional
from dataclasses import dataclass
from collections import deque, defaultdict

@dataclass
class MatchResult:
    """关键词匹配结果"""
    keyword_id: str
    keyword: str
    level: int
    category: str
    start_pos: int
    end_pos: int
    matched_text: str
    action: Action

class AhoCorasickNode:
    """AC自动机节点"""
    __slots__ = ['children', 'failure_link', 'outputs']
    
    def __init__(self):
        self.children: Dict[str, 'AhoCorasickNode'] = {}
        self.failure_link: Optional['AhoCorasickNode'] = None
        self.outputs: List[Dict] = []  # [{keyword_id, keyword, level, ...}]

class KeywordACTrie:
    """
    AC自动机多模式匹配引擎
    
    时间复杂度: O(n + m + z)
    - n: 文本长度
    - m: 所有模式串总长度  
    - z: 匹配结果数量
    
    适用于 EXACT 和 SUBSTRING 类型关键词的高效批量匹配。
    """
    
    def __init__(self):
        self.root = AhoCorasickNode()
        self._built = False
    
    def add_keyword(self, keyword: SensitiveKeyword):
        """添加关键词到Trie"""
        # 添加主词
        self._add_pattern(keyword.keyword, keyword)
        # 添加变体词
        for variant in keyword.variants:
            self._add_pattern(variant, keyword, is_variant=True)
    
    def _add_pattern(self, pattern: str, keyword: SensitiveKeyword, is_variant: bool = False):
        """添加单个模式串"""
        node = self.root
        for char in pattern.lower():
            if char not in node.children:
                node.children[char] = AhoCorasickNode()
            node = node.children[char]
        
        node.outputs.append({
            "keyword_id": keyword.id,
            "keyword": keyword.keyword,
            "level": keyword.level.value,
            "category": keyword.category,
            "action": keyword.action,
            "matched_variant": pattern if is_variant else None
        })
    
    def build(self):
        """构建failure link (BFS)"""
        queue = deque()
        
        # 第一层节点的failure指向root
        for char, node in self.root.children.items():
            node.failure_link = self.root
            queue.append(node)
        
        # BFS构建
        while queue:
            current = queue.popleft()
            
            for char, child in current.children.items():
                # 计算failure link
                failure = current.failure_link
                while failure is not None and char not in failure.children:
                    failure = failure.failure_link
                
                if failure is None:
                    child.failure_link = self.root
                else:
                    child.failure_link = failure.children[char]
                    # 继承failure节点的输出
                    child.outputs.extend(child.failure_link.outputs)
                
                queue.append(child)
        
        self._built = True
    
    def search(self, text: str) -> List[MatchResult]:
        """
        在文本中搜索所有匹配关键词
        
        Args:
            text: 待搜索文本
            
        Returns:
            MatchResult列表，按出现位置排序
        """
        if not self._built:
            self.build()
        
        results = []
        node = self.root
        text_lower = text.lower()
        
        for i, char in enumerate(text_lower):
            # 跟随failure link直到找到匹配转移或回到root
            while node is not None and char not in node.children:
                node = node.failure_link
            
            if node is None:
                node = self.root
                continue
            
            node = node.children[char]
            
            # 收集当前节点的所有匹配输出
            for output in node.outputs:
                matched_len = len(output.get("matched_variant", output["keyword"]))
                start_pos = i - matched_len + 1
                results.append(MatchResult(
                    keyword_id=output["keyword_id"],
                    keyword=output["keyword"],
                    level=output["level"],
                    category=output["category"],
                    start_pos=max(0, start_pos),
                    end_pos=i + 1,
                    matched_text=text[max(0, start_pos):i+1],
                    action=output["action"]
                ))
        
        return results
```

### 4.3 annotate_chunk 实现

```python
import re
from typing import List, Dict, Set, Optional
from dataclasses import dataclass, field

@dataclass
class ChunkSensitivityAnnotation:
    """Chunk敏感内容标注结果"""
    # 命中关键词详情
    sensitive_keywords: List[Dict] = field(default_factory=list)
    
    # 聚合指标
    max_keyword_level: int = 0  # 0-4, 0表示无敏感词
    keyword_categories: Set[str] = field(default_factory=set)
    
    # 统计信息
    total_matches: int = 0
    level_distribution: Dict[int, int] = field(default_factory=dict)
    
    # 是否需要特殊处理
    requires_masking: bool = False
    mask_positions: List[Tuple[int, int]] = field(default_factory=list)

class KeywordAnnotator:
    """
    Chunk关键词标注器
    
    职责:
    1. 加载全局关键词配置
    2. 对Chunk内容进行多模式匹配
    3. 计算聚合敏感指标
    4. 将结果注入Chunk元数据
    """
    
    def __init__(self, db_session):
        self.db = db_session
        self.ac_trie: Optional[KeywordACTrie] = None
        self.regex_keywords: List[SensitiveKeyword] = []
        self.fuzzy_keywords: List[SensitiveKeyword] = []
        self._load_keywords()
    
    def _load_keywords(self):
        """加载并预处理全局关键词"""
        keywords = self.db.query(SensitiveKeyword).filter(
            SensitiveKeyword.enabled == True
        ).all()
        
        # 分类构建索引
        self.ac_trie = KeywordACTrie()
        self.regex_keywords = []
        self.fuzzy_keywords = []
        
        for kw in keywords:
            if kw.match_type in (MatchType.EXACT, MatchType.SUBSTRING):
                self.ac_trie.add_keyword(kw)
            elif kw.match_type == MatchType.REGEX:
                self.regex_keywords.append(kw)
            elif kw.match_type == MatchType.FUZZY:
                self.fuzzy_keywords.append(kw)
        
        self.ac_trie.build()
    
    def annotate_chunk(self, chunk_content: str, modality: Modality = Modality.ALL) -> ChunkSensitivityAnnotation:
        """
        对单个Chunk进行关键词敏感标注
        
        Args:
            chunk_content: Chunk文本内容
            modality: 当前Chunk的模态类型
            
        Returns:
            ChunkSensitivityAnnotation 标注结果
        """
        annotation = ChunkSensitivityAnnotation()
        all_matches: List[MatchResult] = []
        
        # 1. AC自动机匹配 (EXACT + SUBSTRING)
        if self.ac_trie:
            ac_matches = self.ac_trie.search(chunk_content)
            all_matches.extend(ac_matches)
        
        # 2. 正则匹配
        for kw in self.regex_keywords:
            if kw.match_pattern and self._modality_match(kw, modality):
                for m in re.finditer(kw.match_pattern, chunk_content):
                    all_matches.append(MatchResult(
                        keyword_id=kw.id,
                        keyword=kw.keyword,
                        level=kw.level.value,
                        category=kw.category,
                        start_pos=m.start(),
                        end_pos=m.end(),
                        matched_text=m.group(),
                        action=kw.action
                    ))
        
        # 3. 去重: 同一位置保留最高级别匹配
        all_matches = self._deduplicate_matches(all_matches)
        
        # 4. 构建标注结果
        annotation.sensitive_keywords = [
            {
                "keyword_id": m.keyword_id,
                "keyword": m.keyword,
                "level": m.level,
                "category": m.category,
                "matched_text": m.matched_text,
                "position": (m.start_pos, m.end_pos),
                "action": m.action.value
            }
            for m in all_matches
        ]
        
        if all_matches:
            annotation.max_keyword_level = max(m.level for m in all_matches)
            annotation.keyword_categories = set(m.category for m in all_matches)
            annotation.total_matches = len(all_matches)
            annotation.level_distribution = self._calc_level_distribution(all_matches)
            annotation.requires_masking = any(
                m.action == Action.MASK for m in all_matches
            )
            annotation.mask_positions = [
                (m.start_pos, m.end_pos) for m in all_matches if m.action == Action.MASK
            ]
        
        return annotation
    
    def _deduplicate_matches(self, matches: List[MatchResult]) -> List[MatchResult]:
        """
        匹配结果去重:
        - 重叠位置保留最高级别匹配
        - 同级别保留最长匹配
        """
        if not matches:
            return []
        
        # 按位置排序
        matches.sort(key=lambda m: (m.start_pos, -m.level, -(m.end_pos - m.start_pos)))
        
        filtered = []
        last_end = -1
        
        for m in matches:
            if m.start_pos >= last_end:
                filtered.append(m)
                last_end = m.end_pos
            else:
                # 重叠情况: 检查是否级别更高
                if m.level > filtered[-1].level:
                    filtered[-1] = m
                    last_end = m.end_pos
        
        return filtered
    
    def _modality_match(self, keyword: SensitiveKeyword, chunk_modality: Modality) -> bool:
        """检查关键词是否对当前模态生效"""
        if Modality.ALL in keyword.apply_to_modalities:
            return True
        return chunk_modality in keyword.apply_to_modalities
    
    def _calc_level_distribution(self, matches: List[MatchResult]) -> Dict[int, int]:
        """计算级别分布"""
        dist = defaultdict(int)
        for m in matches:
            dist[m.level] += 1
        return dict(dist)


# ============ Chunk入库时的标注调用 ============

def process_chunk_with_annotation(
    chunk: Chunk,
    annotator: KeywordAnnotator,
    db_session
) -> Chunk:
    """
    Chunk入库前标注流程
    
    在Pipeline的"索引入库"阶段之前调用。
    """
    # 执行关键词标注
    annotation = annotator.annotate_chunk(
        chunk_content=chunk.content,
        modality=chunk.modality
    )
    
    # 将标注结果注入Chunk元数据
    chunk.metadata["sensitive_keywords"] = annotation.sensitive_keywords
    chunk.metadata["max_keyword_level"] = annotation.max_keyword_level
    chunk.metadata["keyword_categories"] = list(annotation.keyword_categories)
    chunk.metadata["keyword_match_count"] = annotation.total_matches
    chunk.metadata["level_distribution"] = annotation.level_distribution
    
    # 如果包含需要MASK的关键词，预处理content
    if annotation.requires_masking:
        chunk.content = apply_content_masking(
            chunk.content, 
            annotation.mask_positions
        )
        chunk.metadata["content_masked"] = True
    
    # 持久化到PostgreSQL
    db_session.add(ChunkMetadata(
        chunk_id=chunk.chunk_id,
        doc_id=chunk.doc_id,
        max_keyword_level=annotation.max_keyword_level,
        sensitive_keywords=annotation.sensitive_keywords,
        keyword_categories=list(annotation.keyword_categories),
        # ... 其他字段
    ))
    
    return chunk


def apply_content_masking(content: str, mask_positions: List[Tuple[int, int]]) -> str:
    """
    对敏感内容进行脱敏处理
    
    策略:
    - 人名/公司名: 张** / ****科技
    - 数字: ****
    - 其他: [已脱敏]
    """
    if not mask_positions:
        return content
    
    # 从后向前替换，避免位置偏移
    result = list(content)
    for start, end in sorted(mask_positions, reverse=True):
        masked_len = end - start
        if masked_len <= 2:
            replacement = "*" * masked_len
        elif masked_len <= 8:
            replacement = content[start] + "*" * (masked_len - 1)
        else:
            replacement = content[start:start+2] + "****" + content[end-2:end]
        result[start:end] = list(replacement)
    
    return "".join(result)
```

### 5.2 KeywordLevelFilter 实现

```python
from typing import List, Dict, Optional
from dataclasses import dataclass
from enum import Enum
import time

class FilterDecision(str, Enum):
    ALLOW = "allow"
    BLOCK = "block"
    MASK = "mask"

@dataclass
class FilterAuditRecord:
    """过滤审计记录"""
    user_id: str
    query: str
    chunk_id: str
    doc_id: str
    decision: FilterDecision
    reason: str
    chunk_max_level: int
    user_level: int
    matched_keywords: List[Dict]
    timestamp: float

class KeywordLevelFilter:
    """
    检索阶段关键词级别过滤器
    
    职责:
    1. 根据用户clearance级别过滤越权Chunk
    2. 记录拦截审计日志
    3. 支持分类例外和白名单绕过
    4. 提供被拦截原因说明
    """
    
    def __init__(self, audit_service):
        self.audit = audit_service
        self.block_stats = {
            "total_requests": 0,
            "total_blocked": 0,
            "level_breakdown": {i: 0 for i in range(5)}
        }
    
    def filter_chunks(
        self,
        chunks: List[Chunk],
        user: UserClearance,
        query: str,
        query_context: Optional[Dict] = None
    ) -> tuple[List[Chunk], List[FilterAuditRecord]]:
        """
        过滤Chunk列表，返回允许的Chunk和审计记录
        
        Args:
            chunks: 检索召回的候选Chunk
            user: 用户Clearance信息
            query: 原始查询文本
            query_context: 查询上下文（如知识库ID等）
            
        Returns:
            (allowed_chunks, audit_records)
        """
        allowed = []
        audit_records = []
        user_level = user.effective_level
        
        for chunk in chunks:
            decision, reason, matched = self._evaluate_chunk(chunk, user)
            
            if decision == FilterDecision.ALLOW:
                allowed.append(chunk)
            else:
                # 记录拦截
                record = FilterAuditRecord(
                    user_id=user.user_id,
                    query=query,
                    chunk_id=chunk.chunk_id,
                    doc_id=chunk.doc_id,
                    decision=decision,
                    reason=reason,
                    chunk_max_level=chunk.metadata.get("max_keyword_level", 0),
                    user_level=user_level,
                    matched_keywords=matched,
                    timestamp=time.time()
                )
                audit_records.append(record)
                self.block_stats["total_blocked"] += 1
                self.block_stats["level_breakdown"][
                    chunk.metadata.get("max_keyword_level", 0)
                ] += 1
        
        self.block_stats["total_requests"] += len(chunks)
        
        # 异步写入审计日志
        if audit_records:
            self.audit.log_keyword_filter_events(audit_records)
        
        return allowed, audit_records
    
    def _evaluate_chunk(
        self, 
        chunk: Chunk, 
        user: UserClearance
    ) -> tuple[FilterDecision, str, List[Dict]]:
        """
        评估单个Chunk的访问权限
        
        决策优先级:
        1. 用户白名单检查 (keyword_whitelist)
        2. 分类例外检查 (category_exceptions)
        3. 级别比较 (chunk_max_level <= user_level ?)
        """
        chunk_max_level = chunk.metadata.get("max_keyword_level", 0)
        chunk_keywords = chunk.metadata.get("sensitive_keywords", [])
        user_level = user.effective_level
        
        # 快捷路径: 无敏感词
        if chunk_max_level == 0:
            return FilterDecision.ALLOW, "No sensitive keywords", []
        
        # 收集命中的高敏关键词
        exceeded_keywords = [
            kw for kw in chunk_keywords 
            if kw["level"] > user_level
        ]
        
        # 规则1: 白名单绕过
        whitelist_hits = [
            kw for kw in exceeded_keywords
            if kw["keyword"] in user.keyword_whitelist or kw["keyword_id"] in user.keyword_whitelist
        ]
        if whitelist_hits and len(whitelist_hits) == len(exceeded_keywords):
            return (
                FilterDecision.ALLOW, 
                f"All exceeded keywords in user whitelist: {[k['keyword'] for k in whitelist_hits]}",
                exceeded_keywords
            )
        
        # 规则2: 分类例外绕过
        exceeded_categories = set(kw["category"] for kw in exceeded_keywords)
        if exceeded_categories and exceeded_categories.issubset(set(user.category_exceptions)):
            return (
                FilterDecision.ALLOW,
                f"Categories in user exception list: {exceeded_categories}",
                exceeded_keywords
            )
        
        # 规则3: 级别比较
        if chunk_max_level <= user_level:
            return FilterDecision.ALLOW, "Chunk level within user clearance", chunk_keywords
        
        # 规则4: 拦截
        blocking_keywords = [kw for kw in exceeded_keywords if kw["keyword"] not in user.keyword_whitelist]
        reason = (
            f"Chunk max_keyword_level({chunk_max_level}) > "
            f"user_effective_level({user_level}). "
            f"Blocking keywords: {[k['keyword'] for k in blocking_keywords]}"
        )
        
        return FilterDecision.BLOCK, reason, exceeded_keywords
    
    def get_filter_stats(self) -> Dict:
        """获取过滤统计"""
        total = self.block_stats["total_requests"]
        blocked = self.block_stats["total_blocked"]
        return {
            "total_requests": total,
            "total_blocked": blocked,
            "block_rate": blocked / total if total > 0 else 0,
            "level_breakdown": self.block_stats["level_breakdown"]
        }


# ============ 在检索服务中的集成调用 ============

async def search_with_keyword_filter(
    query: str,
    user_id: str,
    kb_id: str,
    top_k: int = 10,
    search_service: SearchService = None,
    permission_service: PermissionService = None,
    keyword_filter: KeywordLevelFilter = None
) -> SearchResponse:
    """
    带关键词过滤的检索入口
    """
    # 1. 获取用户Clearance
    user_clearance = await permission_service.get_user_clearance(user_id)
    
    # 2. 构建检索表达式 (包含关键词级别预过滤)
    # 利用Milvus标量过滤能力，在向量检索阶段就排除明显越级的内容
    max_allowed_level = user_clearance.effective_level
    milvus_expr = f"kb_id == '{kb_id}' and max_keyword_level <= {max_allowed_level}"
    
    # 3. 执行混合检索
    candidates = await search_service.hybrid_search(
        query=query,
        expr=milvus_expr,
        top_k=top_k * 3  # 多召回一些，后续精过滤
    )
    
    # 4. 关键词级别精过滤 (二次校验 + 审计)
    allowed_chunks, audit_records = keyword_filter.filter_chunks(
        chunks=candidates,
        user=user_clearance,
        query=query
    )
    
    # 5. Re-rank
    ranked = await search_service.rerank(allowed_chunks, query, top_k=top_k)
    
    # 6. 组装响应
    return SearchResponse(
        results=ranked,
        total_candidates=len(candidates),
        filtered_count=len(candidates) - len(allowed_chunks),
        audit_records=len(audit_records)
    )
```

## 6. 问答时关键词拦截

### 6.1 为什么需要最终防线

检索阶段的关键词过滤基于**Chunk的预标注级别**，但存在以下绕过风险：

1. **推理泄露**: LLM可能从多个L1/L2 Chunk中推理出L3级别的结论
2. **编造风险**: LLM可能"幻觉"出高敏感信息（虽然与检索内容无关，但造成恐慌）
3. **提示注入**: 恶意用户通过jailbreak让LLM输出敏感内容
4. **标注遗漏**: 新关键词尚未触发全量重标注，存在漏网之鱼

因此，**生成后的答案必须进行最终校验**。

### 6.2 ResponseKeywordInterceptor 实现

```python
from typing import List, Dict, Optional, Tuple
from dataclasses import dataclass
from enum import Enum
import re

class InterceptDecision(str, Enum):
    PASS = "pass"           # 答案安全，直接返回
    MASK = "mask"           # 答案包含敏感词，脱敏后返回
    BLOCK = "block"         # 答案越级，拦截并返回安全提示
    REVIEW = "review"       # 可疑，需人工审核

@dataclass
class InterceptionResult:
    decision: InterceptDecision
    original_answer: str
    processed_answer: Optional[str] = None
    matched_keywords: List[Dict] = None
    risk_score: float = 0.0  # 0-1
    reason: str = ""
    suggested_safe_response: Optional[str] = None

class ResponseKeywordInterceptor:
    """
    问答响应关键词拦截器 —— 最终安全防线
    
    三道检查:
    1. 答案关键词检测 (基于AC自动机)
    2. LLM编造检测 (基于上下文忠实度)
    3. 推理泄露检测 (高敏推断识别)
    """
    
    def __init__(
        self,
        keyword_annotator: KeywordAnnotator,
        audit_service: AuditService,
        llm_client = None
    ):
        self.annotator = keyword_annotator
        self.audit = audit_service
        self.llm = llm_client
    
    async def intercept(
        self,
        answer: str,
        context_chunks: List[Chunk],
        user: UserClearance,
        query: str,
        session_id: str
    ) -> InterceptionResult:
        """
        拦截主入口
        
        Returns:
            InterceptionResult: 包含处理决策和(如需要)修改后的答案
        """
        # === 检查1: 答案直接关键词检测 ===
        answer_annotation = self.annotator.annotate_chunk(answer)
        
        if answer_annotation.max_keyword_level > user.effective_level:
            exceeded = [
                kw for kw in answer_annotation.sensitive_keywords
                if kw["level"] > user.effective_level
            ]
            
            # 检查是否来自检索上下文（正常推理）还是LLM编造
            is_hallucinated = self._detect_hallucination(answer, context_chunks, exceeded)
            
            if is_hallucinated:
                # LLM编造了高敏内容 —— 严重违规
                await self._log_security_event(
                    user, query, answer, exceeded, "HALLUCINATED_SENSITIVE",
                    session_id
                )
                return InterceptionResult(
                    decision=InterceptDecision.BLOCK,
                    original_answer=answer,
                    matched_keywords=exceeded,
                    risk_score=0.95,
                    reason="LLM hallucinated sensitive content not present in retrieved context",
                    suggested_safe_response=self._generate_safe_response(
                        user.effective_level, "hallucination"
                    )
                )
            else:
                # 答案基于检索上下文推导，但级别超限
                # 尝试脱敏
                masked_answer = self._mask_answer(answer, exceeded)
                await self._log_security_event(
                    user, query, answer, exceeded, "INFERRED_SENSITIVE",
                    session_id
                )
                return InterceptionResult(
                    decision=InterceptDecision.MASK,
                    original_answer=answer,
                    processed_answer=masked_answer,
                    matched_keywords=exceeded,
                    risk_score=0.75,
                    reason="Answer contains inferred sensitive content exceeding user clearance",
                    suggested_safe_response=masked_answer
                )
        
        # === 检查2: 推理泄露检测 ===
        inference_risk = await self._detect_inference_leakage(
            answer, context_chunks, user.effective_level
        )
        if inference_risk["risk_score"] > 0.7:
            await self._log_security_event(
                user, query, answer, [], "INFERENCE_LEAKAGE",
                session_id, extra={"inference_risk": inference_risk}
            )
            return InterceptionResult(
                decision=InterceptDecision.BLOCK,
                original_answer=answer,
                risk_score=inference_risk["risk_score"],
                reason=f"Potential inference leakage detected: {inference_risk['reason']}",
                suggested_safe_response=self._generate_safe_response(
                    user.effective_level, "inference"
                )
            )
        
        # === 检查3: 安全模式评估 ===
        safety_score = self._calculate_safety_score(answer, answer_annotation)
        if safety_score < 0.3:
            return InterceptionResult(
                decision=InterceptDecision.REVIEW,
                original_answer=answer,
                risk_score=1.0 - safety_score,
                reason="Low safety score triggers manual review",
                suggested_safe_response=self._generate_safe_response(
                    user.effective_level, "review"
                )
            )
        
        # 通过所有检查
        return InterceptionResult(
            decision=InterceptDecision.PASS,
            original_answer=answer,
            risk_score=0.0,
            reason="All security checks passed"
        )
    
    def _detect_hallucination(
        self, 
        answer: str, 
        context_chunks: List[Chunk],
        exceeded_keywords: List[Dict]
    ) -> bool:
        """
        检测LLM是否编造了敏感内容
        
        策略:
        1. 检查答案中的高敏关键词是否出现在检索上下文中
        2. 如果关键词在上下文中存在但级别未超限 → 正常推理
        3. 如果关键词在上下文中不存在 → 可能编造
        """
        # 合并所有上下文文本
        context_text = " ".join(c.content for c in context_chunks)
        context_lower = context_text.lower()
        
        for kw_info in exceeded_keywords:
            kw_text = kw_info["matched_text"].lower()
            # 检查关键词原文是否出现在上下文中
            if kw_text not in context_lower:
                # 进一步检查变体
                keyword_obj = self.annotator.db.query(SensitiveKeyword).get(kw_info["keyword_id"])
                if keyword_obj:
                    all_forms = {keyword_obj.keyword.lower()}
                    all_forms.update(v.lower() for v in keyword_obj.variants)
                    if not any(form in context_lower for form in all_forms):
                        return True  # 高度怀疑编造
        
        return False
    
    async def _detect_inference_leakage(
        self,
        answer: str,
        context_chunks: List[Chunk],
        user_level: int
    ) -> Dict:
        """
        检测答案是否从低敏Chunk中推理出了高敏结论
        
        示例风险:
        - 上下文: "Q3销售额下降30%" + "成本上升20%"
        - 答案: "因此Q3预计净亏损500万" (L2级别结论)
        - 用户级别: L1 → 不应看到亏损信息
        """
        # 简单启发式: 答案中的敏感词级别是否显著高于上下文平均级别
        context_max_levels = [
            c.metadata.get("max_keyword_level", 0) for c in context_chunks
        ]
        avg_context_level = sum(context_max_levels) / len(context_max_levels) if context_max_levels else 0
        
        answer_annotation = self.annotator.annotate_chunk(answer)
        answer_max_level = answer_annotation.max_keyword_level
        
        # 如果答案级别显著高于上下文，可能是推理泄露
        if answer_max_level > avg_context_level + 1:
            return {
                "risk_score": 0.8,
                "reason": f"Answer level({answer_max_level}) significantly exceeds context level({avg_context_level:.1f})"
            }
        
        # 可使用LLM进行二次判断
        if self.llm and answer_max_level > user_level:
            prompt = f"""
            判断以下答案是否包含用户不应看到的敏感信息:
            
            用户权限级别: L{user_level}
            检索上下文最高敏感级别: L{max(context_max_levels) if context_max_levels else 0}
            
            答案: {answer}
            
            请判断:
            1. 答案是否包含超越用户权限级别的敏感信息?
            2. 这些信息是否可以从检索上下文中合理推断?
            3. 风险评分 (0-1)
            
            返回JSON: {{"is_leakage": bool, "risk_score": float, "reason": str}}
            """
            try:
                response = await self.llm.complete(prompt)
                result = json.loads(response)
                return result
            except Exception:
                pass
        
        return {"risk_score": 0.0, "reason": "No significant inference risk detected"}
    
    def _mask_answer(self, answer: str, keywords: List[Dict]) -> str:
        """对答案中的敏感词进行脱敏"""
        result = answer
        # 从后向前替换
        for kw in sorted(keywords, key=lambda k: k["position"][0], reverse=True):
            start, end = kw["position"]
            result = result[:start] + "[已脱敏]" + result[end:]
        return result
    
    def _calculate_safety_score(self, answer: str, annotation: ChunkSensitivityAnnotation) -> float:
        """计算答案安全评分"""
        if annotation.max_keyword_level == 0:
            return 1.0
        # 级别越高分数越低
        return max(0, 1.0 - annotation.max_keyword_level * 0.2)
    
    def _generate_safe_response(self, user_level: int, reason_type: str) -> str:
        """生成安全提示回复"""
        templates = {
            "hallucination": (
                "很抱歉，系统检测到回答中可能包含不准确的信息。"
                "由于您的当前权限级别为 L{}，我无法为您提供相关内容。"
                "如有需要，请联系管理员申请更高权限。"
            ),
            "inference": (
                "根据您的权限级别(L{})，我无法提供该问题的完整回答，"
                "因为相关内容涉及更高保密级别的信息。"
                "建议您通过正式渠道向相关部门咨询。"
            ),
            "review": (
                "您的请求已提交审核。"
                "由于内容涉及敏感信息，需要管理员确认后才能回复。"
            ),
            "default": (
                "抱歉，根据您的访问权限，我无法回答此问题。"
            )
        }
        template = templates.get(reason_type, templates["default"])
        return template.format(user_level)
    
    async def _log_security_event(
        self,
        user: UserClearance,
        query: str,
        answer: str,

            "keywords": keywords,

            "extra": extra or {}
        })
```

```python
class GenerateService:
    """生成服务 - 集成关键词拦截"""

        self.interceptor = interceptor
        self.llm = llm_client
    
    async def generate_answer(
        self,
        query: str,
        context_chunks: List[Chunk],
        user: UserClearance,
        session_id: str
    ) -> GenerateResponse:
        """
        带安全拦截的答案生成流程
        """
        # 1. 组装Prompt (上下文只包含已通过权限过滤的Chunk)
        prompt = self._build_prompt(query, context_chunks)
        
        # 2. 调用LLM生成
        raw_answer = await self.llm.complete(prompt)
        
        # 3. 最终防线: 响应关键词拦截
        interception = await self.interceptor.intercept(
            answer=raw_answer,
            context_chunks=context_chunks,
            user=user,
            query=query,
            session_id=session_id
        )
        
        # 4. 根据拦截决策处理
        if interception.decision == InterceptDecision.PASS:
            return GenerateResponse(
                answer=raw_answer,
                sources=[c.chunk_id for c in context_chunks],
                safety_status="PASS"
            )
        
        elif interception.decision == InterceptDecision.MASK:
            return GenerateResponse(
                answer=interception.processed_answer,
                sources=[c.chunk_id for c in context_chunks],
                safety_status="MASKED",
                safety_notice="部分内容已脱敏处理"
            )
        
        elif interception.decision in (InterceptDecision.BLOCK, InterceptDecision.REVIEW):
            return GenerateResponse(
                answer=interception.suggested_safe_response,
                sources=[],  # 不返回来源
                safety_status="BLOCKED",
                safety_notice=interception.reason,
                requires_review=(interception.decision == InterceptDecision.REVIEW)
            )
```

## 7. 动态更新机制

### 7.1 问题定义

当管理员在后台新增/修改/删除关键词时，系统中已入库的Chunk需要更新敏感标注。直接全量重跑所有文档成本极高，需要高效的增量更新机制。

### 7.2 更新策略设计

```
关键词变更事件
       |
       v
+------------------------------+
| 1. 变更分类与影响评估         |
|   - 新增关键词?              |
|   - 级别上调? (如 L2→L3)     |
|   - 级别下调? (如 L3→L2)     |
|   - 删除/禁用?               |
|   - 变体词变更?              |
+------------------------------+
       |
       v
+------------------------------+
| 2. 影响范围计算               |
|   - 匹配模式: EXACT影响小     |
|   - SUBSTRING/REGEX影响大     |
|   - 限定知识库?               |
+------------------------------+
       |
       v
+----------+------------+-----------+
| 影响小   | 影响中      | 影响大    |
|(EXACT)   |(SUBSTRING) |(全局正则) |
+----------+------------+-----------+
| 定向重标 | 批量扫描   | 全量重标  |
| 精确匹配 | 候选过滤   | (低峰期)  |
+----------+------------+-----------+
```

### 7.3 增量更新实现

```python
from enum import Enum
from typing import List, Set, Optional
from dataclasses import dataclass
import asyncio

class ChangeType(str, Enum):
    CREATE = "create"
    UPDATE = "update"
    DELETE = "disable"
    LEVEL_UP = "level_up"      # 级别上调 (影响更大)
    LEVEL_DOWN = "level_down"  # 级别下调 (影响更小)
    VARIANTS_CHANGE = "variants_change"

@dataclass
class KeywordChangeEvent:
    """关键词变更事件"""
    change_type: ChangeType
    keyword: SensitiveKeyword
    old_keyword: Optional[SensitiveKeyword] = None  # UPDATE时有用

class ChunkReannotationEngine:
    """
    Chunk增量重标注引擎
    
    根据关键词变更类型，选择最优的重标注策略，
    最小化计算和IO开销。
    """
    
    def __init__(self, db_session, milvus_client, annotator: KeywordAnnotator):
        self.db = db_session
        self.milvus = milvus_client
        self.annotator = annotator
        self.batch_size = 500
    
    async def handle_keyword_change(self, event: KeywordChangeEvent):
        """处理关键词变更事件"""
        keyword = event.keyword
        
        if event.change_type == ChangeType.CREATE:
            await self._handle_create(keyword)
        elif event.change_type == ChangeType.LEVEL_UP:
            await self._handle_level_up(keyword, event.old_keyword)
        elif event.change_type == ChangeType.LEVEL_DOWN:
            await self._handle_level_down(keyword, event.old_keyword)
        elif event.change_type == ChangeType.DELETE:
            await self._handle_delete(keyword)
        elif event.change_type == ChangeType.VARIANTS_CHANGE:
            await self._handle_variants_change(keyword, event.old_keyword)
    
    async def _handle_create(self, keyword: SensitiveKeyword):
        """
        新增关键词: 只需扫描可能命中的Chunk
        
        策略:
        - EXACT/SUBSTRING: 使用数据库LIKE查询快速定位候选
        - REGEX: 全表扫描 (但可限定知识库)
        """
        candidate_chunk_ids = await self._find_candidates(keyword)
        
        if not candidate_chunk_ids:
            return
        
        # 批量重标注
        await self._reannotate_chunks(candidate_chunk_ids, incremental=True)
        
        # 更新统计
        print(f"[CREATE] Keyword '{keyword.keyword}' processed {len(candidate_chunk_ids)} chunks")
    
    async def _handle_level_up(self, keyword: SensitiveKeyword, old: SensitiveKeyword):
        """
        级别上调 (如 L2→L3):
        
        只需更新已命中该关键词的Chunk的 max_keyword_level，
        无需重新匹配文本。
        """
        # 找到所有已包含此关键词的Chunk
        affected = self.db.query(ChunkMetadata).filter(
            ChunkMetadata.sensitive_keywords.contains([{"keyword_id": keyword.id}])
        ).all()
        
        for chunk_meta in affected:
            # 重新计算 max_level
            chunk_meta.max_keyword_level = self._recalc_max_level(chunk_meta)
            chunk_meta.updated_at = datetime.utcnow()
        
        self.db.commit()
        
        # 同步更新Milvus标量字段
        await self._sync_milvus_scalar([c.chunk_id for c in affected])
        
        print(f"[LEVEL_UP] Keyword '{keyword.keyword}' affected {len(affected)} chunks")
    
    async def _handle_level_down(self, keyword: SensitiveKeyword, old: SensitiveKeyword):
        """
        级别下调 (如 L3→L2):
        
        可能需要降低某些Chunk的 max_keyword_level，
        但前提是这些Chunk没有其他同级别或更高级别关键词。
        """
        affected = self.db.query(ChunkMetadata).filter(
            ChunkMetadata.sensitive_keywords.contains([{"keyword_id": keyword.id}])
        ).all()
        
        recalc_needed = []
        for chunk_meta in affected:
            # 检查是否还有其他关键词维持原级别
            other_max = max(
                (kw["level"] for kw in chunk_meta.sensitive_keywords if kw["keyword_id"] != keyword.id),
                default=0
            )
            if other_max < old.level.value:
                # 需要重新计算
                recalc_needed.append(chunk_meta)
        
        if recalc_needed:
            for chunk_meta in recalc_needed:
                chunk_meta.max_keyword_level = self._recalc_max_level(chunk_meta)
            self.db.commit()
            await self._sync_milvus_scalar([c.chunk_id for c in recalc_needed])
        
        print(f"[LEVEL_DOWN] Keyword '{keyword.keyword}' affected {len(recalc_needed)} chunks")
    
    async def _handle_delete(self, keyword: SensitiveKeyword):
        """
        删除/禁用关键词:
        
        从Chunk的 sensitive_keywords 列表中移除此关键词，
        并重新计算 max_keyword_level。
        """
        affected = self.db.query(ChunkMetadata).filter(
            ChunkMetadata.sensitive_keywords.contains([{"keyword_id": keyword.id}])
        ).all()
        
        for chunk_meta in affected:
            # 移除该关键词
            chunk_meta.sensitive_keywords = [
                kw for kw in chunk_meta.sensitive_keywords
                if kw["keyword_id"] != keyword.id
            ]
            # 更新分类集合
            chunk_meta.keyword_categories = list(set(
                kw["category"] for kw in chunk_meta.sensitive_keywords
            ))
            # 重新计算级别
            chunk_meta.max_keyword_level = self._recalc_max_level(chunk_meta)
            chunk_meta.updated_at = datetime.utcnow()
        
        self.db.commit()
        await self._sync_milvus_scalar([c.chunk_id for c in affected])
        
        print(f"[DELETE] Keyword '{keyword.keyword}' affected {len(affected)} chunks")
    
    async def _handle_variants_change(self, keyword: SensitiveKeyword, old: SensitiveKeyword):
        """
        变体词变更:
        
        需要重新扫描所有可能受影响的Chunk。
        优化: 使用新旧变体集合的差集缩小扫描范围。
        """
        old_variants = set(old.variants if old else [])
        new_variants = set(keyword.variants)
        
        added_variants = new_variants - old_variants
        removed_variants = old_variants - new_variants
        
        # 扫描新增变体可能命中的Chunk
        if added_variants:
            for variant in added_variants:
                candidates = await self._find_candidates_by_text(variant, keyword.apply_to_kbs)
                await self._reannotate_chunks(candidates, incremental=True)
        
        # 对于移除的变体，需要检查是否还有其他变体/主词维持命中
        if removed_variants:
            # 保守策略: 重新标注所有包含该主词的Chunk
            affected = self.db.query(ChunkMetadata).filter(
                ChunkMetadata.sensitive_keywords.contains([{"keyword_id": keyword.id}])
            ).all()
            await self._reannotate_chunks([c.chunk_id for c in affected], incremental=True)
    
    async def _find_candidates(self, keyword: SensitiveKeyword) -> List[str]:
        """
        快速定位可能命中的Chunk候选
        
        策略:
        1. 对EXACT/SUBSTRING类型，使用数据库LIKE查询
        2. 对REGEX类型，全量扫描但可限定知识库
        """
        kb_filter = keyword.apply_to_kbs
        
        if keyword.match_type in (MatchType.EXACT, MatchType.SUBSTRING):
            # 构建LIKE模式
            patterns = [f"%{keyword.keyword}%"]
            for v in keyword.variants:
                patterns.append(f"%{v}%")
            
            query = self.db.query(Chunk.chunk_id, Chunk.content).filter(
                Chunk.content.op("ILIKE")(patterns[0])  # 简化示意
            )
            if kb_filter:
                query = query.filter(Chunk.kb_id.in_(kb_filter))
            
            results = query.all()
            return [r.chunk_id for r in results]
        
        elif keyword.match_type == MatchType.REGEX:
            # REGEX类型: 需要全量扫描，但可限定知识库范围
            query = self.db.query(Chunk.chunk_id, Chunk.content)
            if kb_filter:
                query = query.filter(Chunk.kb_id.in_(kb_filter))
            
            results = query.all()
            candidate_ids = []
            pattern = re.compile(keyword.match_pattern, re.IGNORECASE)
            for chunk_id, content in results:
                if pattern.search(content):
                    candidate_ids.append(chunk_id)
            return candidate_ids
        
        return []
    
    async def _reannotate_chunks(self, chunk_ids: List[str], incremental: bool = True):
        """批量重标注Chunk"""
        for i in range(0, len(chunk_ids), self.batch_size):
            batch = chunk_ids[i:i + self.batch_size]
            
            chunks = self.db.query(Chunk).filter(Chunk.chunk_id.in_(batch)).all()
            
            for chunk in chunks:
                # 重新标注
                annotation = self.annotator.annotate_chunk(
                    chunk.content,
                    chunk.modality
                )
                
                # 更新元数据
                chunk.metadata["sensitive_keywords"] = annotation.sensitive_keywords
                chunk.metadata["max_keyword_level"] = annotation.max_keyword_level
                chunk.metadata["keyword_categories"] = list(annotation.keyword_categories)
            
            self.db.commit()
            
            # 同步Milvus
            await self._sync_milvus_scalar(batch)
    
    async def _sync_milvus_scalar(self, chunk_ids: List[str]):
        """同步Milvus标量字段"""
        chunk_metas = self.db.query(ChunkMetadata).filter(
            ChunkMetadata.chunk_id.in_(chunk_ids)
        ).all()
        
        for meta in chunk_metas:
            self.milvus.update_entity(
                collection_name="text_chunks",
                entities=[{
                    "chunk_id": meta.chunk_id,
                    "max_keyword_level": meta.max_keyword_level,
                    "keyword_categories": meta.keyword_categories
                }]
            )
    
    def _recalc_max_level(self, chunk_meta: ChunkMetadata) -> int:
        """重新计算Chunk的最高敏感级别"""
        if not chunk_meta.sensitive_keywords:
            return 0
        return max(kw["level"] for kw in chunk_meta.sensitive_keywords)


# ============ 后台管理API ============

class KeywordAdminAPI:
    """关键词管理后台API"""
    
    def __init__(self, reannotation_engine: ChunkReannotationEngine):
        self.engine = reannotation_engine
    
    async def create_keyword(self, keyword_data: dict) -> SensitiveKeyword:
        """创建新关键词"""
        keyword = SensitiveKeyword(**keyword_data)
        self.db.add(keyword)
        self.db.commit()
        
        # 触发增量重标注
        await self.engine.handle_keyword_change(KeywordChangeEvent(
            change_type=ChangeType.CREATE,
            keyword=keyword
        ))
        
        return keyword
    
    async def update_keyword(self, keyword_id: str, updates: dict) -> SensitiveKeyword:
        """更新关键词"""
        old = self.db.query(SensitiveKeyword).get(keyword_id)
        
        # 记录变更前状态
        old_snapshot = SensitiveKeyword(**old.dict())
        
        # 应用更新
        for k, v in updates.items():
            setattr(old, k, v)
        old.updated_at = datetime.utcnow()
        self.db.commit()
        
        # 判断变更类型
        change_type = self._classify_change(old_snapshot, old)
        
        # 触发增量重标注
        await self.engine.handle_keyword_change(KeywordChangeEvent(
            change_type=change_type,
            keyword=old,
            old_keyword=old_snapshot
        ))
        
        return old
    
    def _classify_change(self, old: SensitiveKeyword, new: SensitiveKeyword) -> ChangeType:
        """分类变更类型以选择最优更新策略"""
        if old.level.value < new.level.value:
            return ChangeType.LEVEL_UP
        elif old.level.value > new.level.value:
            return ChangeType.LEVEL_DOWN
        elif set(old.variants) != set(new.variants):
            return ChangeType.VARIANTS_CHANGE
        return ChangeType.UPDATE
```

### 7.4 全量重标注兜底

```python
class FullReannotationJob:
    """
    全量重标注任务
    
    触发条件:
    1. 系统升级，标注逻辑变更
    2. 关键词大规模重构
    3. 定期一致性校验发现偏差
    
    执行策略:
    - 低峰期执行 (夜间/周末)
    - 分批处理，避免数据库锁
    - 支持断点续传
    """
    
    async def run_full_reannotation(
        self,
        kb_id: Optional[str] = None,
        batch_size: int = 1000
    ):
        """执行全量重标注"""
        query = self.db.query(Chunk.chunk_id)
        if kb_id:
            query = query.filter(Chunk.kb_id == kb_id)
        
        total = query.count()
        processed = 0
        
        # 使用游标避免大OFFSET
        last_id = None
        while True:
            batch_query = query.order_by(Chunk.chunk_id).limit(batch_size)
            if last_id:
                batch_query = batch_query.filter(Chunk.chunk_id > last_id)
            
            batch = batch_query.all()
            if not batch:
                break
            
            chunk_ids = [c.chunk_id for c in batch]
            await self.engine._reannotate_chunks(chunk_ids, incremental=False)
            
            processed += len(batch)
            last_id = batch[-1].chunk_id
            
            print(f"Progress: {processed}/{total} ({processed/total*100:.1f}%)")
            
            # 每批之间短暂休眠，降低系统负载
            await asyncio.sleep(0.5)
```

### 8.2 协同关系矩阵

| 权限层级 | 控制对象 | 与关键词系统的关系 | 执行顺序 |
|---------|---------|-------------------|---------|
| **L1 文件类型** | 文件类型(DOC/EXCEL/VIDEO等) | 独立维度 | 1 |
| **L2 文档级** | 具体文档 | 独立维度 | 2 |
| **L3 字段级** | 文档内段落/列/单元格 | 独立维度 | 3 |
| **关键词级** | 内容语义敏感度 | **核心补充维度** | 4 |
| **L4 标签级** | 安全等级/业务标签 | 与关键词分类可联动 | 5 |

### 8.3 联动机制

```python
class UnifiedPermissionEngine:
    """
    统一权限决策引擎
    
    整合所有权限维度，输出最终决策。
    关键词敏感控制作为独立维度参与决策。
    """
    
    def __init__(
        self,
        file_type_filter,
        doc_permission_filter,
        field_permission_filter,
        keyword_level_filter: KeywordLevelFilter,
        tag_permission_filter
    ):
        self.filters = {
            "file_type": file_type_filter,
            "document": doc_permission_filter,
            "field": field_permission_filter,
            "keyword": keyword_level_filter,
            "tag": tag_permission_filter
        }
    
    async def evaluate_chunk_access(
        self,
        chunk: Chunk,
        user: User,
        query: str
    ) -> PermissionDecision:
        """
        综合评估Chunk访问权限
        
        所有维度都必须通过，任一维度拒绝则整体拒绝。
        """
        decisions = []
        
        # L1: 文件类型
        d1 = self.filters["file_type"].check(user, chunk.modality)
        decisions.append(("file_type", d1))
        if not d1.allowed:
            return PermissionDecision(deny=True, reason="File type not allowed")
        
        # L2: 文档级
        d2 = self.filters["document"].check(user, chunk.doc_id)
        decisions.append(("document", d2))
        if not d2.allowed:
            return PermissionDecision(deny=True, reason="Document access denied")
        
        # L3: 字段级
        d3 = self.filters["field"].check(user, chunk.doc_id, chunk.position_info)
        decisions.append(("field", d3))
        if not d3.allowed:
            return PermissionDecision(deny=True, reason="Field access denied")
        
        # ★ 关键词级
        d4 = self.filters["keyword"]._evaluate_chunk(chunk, user.clearance)
        decisions.append(("keyword", d4))
        if d4[0] == FilterDecision.BLOCK:
            return PermissionDecision(
                deny=True, 
                reason=f"Keyword level exceeded: {d4[1]}",
                keyword_blocked=True
            )
        
        # L4: 标签级
        d5 = self.filters["tag"].check(user, chunk.tags)
        decisions.append(("tag", d5))
        if not d5.allowed:
            return PermissionDecision(deny=True, reason="Tag access denied")
        
        return PermissionDecision(allow=True, dimension_decisions=decisions)
```

### 8.4 性能优化建议

```
关键词系统性能优化:

1. 预过滤策略
   - Milvus检索expr中直接加入 max_keyword_level <= user_level
   - 减少需要从磁盘读取的候选Chunk数量

2. 分层缓存
   - 关键词配置缓存: Redis TTL=30s
   - AC自动机缓存: 进程内内存，配置变更时重建
   - 用户Clearance缓存: Redis TTL=5min

3. 异步标注
   - Chunk标注在摄取Pipeline中异步完成
   - 不阻塞用户上传响应

4. 批量更新
   - 关键词变更触发的重标注使用批量处理
   - 每批500-1000个Chunk

5. 索引优化
   - PostgreSQL: ChunkMetadata.sensitive_keywords 使用GIN索引
   - PostgreSQL: ChunkMetadata.max_keyword_level 使用B-tree索引
   - Milvus: max_keyword_level 作为主过滤标量字段
```
