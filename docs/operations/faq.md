# 企业级私有化多模态 RAG 系统 — 常见问题解答（FAQ）

> 版本：v1.0  
> 更新日期：2026-06-15  
> 适用对象：最终用户、KB 管理员、Super Admin、技术支持

---

## 一、基础使用

### Q1：系统支持哪些文件格式？

**A**：当前支持以下格式：

- 文档：PDF、Word（DOC/DOCX）、PPT、TXT、Markdown
- 表格：Excel（XLS/XLSX）、CSV
- 图片：PNG、JPG/JPEG、WebP
- 视频：MP4、MOV、AVI（自动切片+关键帧提取）
- 链接：网页 URL（自动抓取正文及图片 alt 文本）

> 音频格式正在规划中，预计后续版本支持。

---

### Q2：上传文档后多久可以被搜索到？

**A**：取决于文件大小和系统负载：

- 普通文档（< 10 MB）：通常 30 秒~2 分钟。
- 大视频/大 PDF（> 100 MB）：可能需要 5~15 分钟。
- 可在 **任务中心** 查看解析进度，状态变为“已索引”后即可检索。

---

### Q3：为什么上传的文档搜不到？

**A**：请按以下顺序排查：

1. 检查文档状态是否为“已索引”。
2. 检查文档密级与当前用户权限是否匹配。
3. 检查搜索时是否选择了正确的知识库或模态过滤。
4. 检查问题是否与文档内容语义相关（尝试用更通用的关键词）。
5. 如果文档刚上传，可能 Worker 队列拥堵，请稍等或联系管理员。

---

### Q4：可以一次上传多个文件吗？

**A**：可以。系统支持批量上传，建议单次不超过 100 个文件，总大小不超过系统配置的上限（默认 500 MB）。

---

### Q5：如何对答案进行反馈？

**A**：在问答结果下方点击 **点赞** 或 **点踩**。点踩后可选择原因（答案错误 / 未引用 / 引用不准 / 答非所问），管理员可在评测工作台查看反馈数据。

---

## 二、权限与账号

### Q6：系统的权限模型是怎样的？

**A**：系统采用 RBAC + ABAC 混合模型，并支持五级权限穿透：

```
文件类型 → 文档 → 字段/段落 → 标签 → 关键词
```

可精确控制“谁能看到什么内容”。

---

### Q7：普通用户如何申请访问某个知识库？

**A**：

1. 在知识库列表找到目标知识库。
2. 点击 **申请权限**。
3. 填写申请理由并提交。
4. KB 管理员或 Super Admin 在消息中心审批。
5. 审批通过后刷新页面即可访问。

---

### Q8：字段级权限支持哪些文档类型？

**A**：目前支持 Word 和 Excel 文档：

- Word：可按段落或章节设置可见性。
- Excel：可按单元格、列、工作表设置可见性。

字段级权限通常用于财务、人事、合同等敏感内容场景。

---

### Q9：用户组权限如何继承？

**A**：权限继承遵循“用户组权限 ∪ 个人权限”的并集规则：

- 用户加入用户组后，自动获得该组的所有权限。
- 管理员也可单独给用户授权。
- 最终有效权限 = 用户组权限 + 个人直接权限。

---

### Q10：离职员工账号如何处理？

**A**：Super Admin 可在 **系统管理** → **用户管理** 中：

1. 禁用账号（保留历史审计日志）。
2. 或删除账号（谨慎操作，删除后无法恢复）。

建议优先使用“禁用”，以满足合规审计要求。

---

## 三、技术配置

### Q11：如何配置 Embedding / Re-rank / 生成模型？

**A**：Super Admin 进入 **系统管理** → **模型配置**：

- Embedding 模型：配置服务地址、向量维度、是否默认。
- Re-rank 模型：配置服务地址、Top-K 重排数量。
- 生成模型：配置 Provider、API Key、Base URL、温度、最大 Token。

> 配置保存后即时生效，无需重启服务。

---

### Q12：支持哪些大模型 Provider？

**A**：当前已验证 minimax，设计上兼容 OpenAI 协议。后续可扩展至：

- 私有部署模型（如 vLLM、Ollama、Xinference）
- 商业 API（OpenAI、Azure OpenAI、智谱、通义千问等）

---

### Q13：向量库可以替换吗？

**A**：当前默认使用 Milvus，数据层通过统一接口封装。若需替换为 pgvector 或其他向量库，需二次开发，但业务代码改动范围可控。

---

### Q14：系统的部署方式是什么？

**A**：支持 Docker Compose 一键部署，包含：

- 前端 Nginx
- FastAPI 后端
- Celery Worker（ingest / embed / permission-sync）
- PostgreSQL、Milvus、Redis、RabbitMQ、MinIO
- Kong 网关、Prometheus、Grafana

生产环境建议基于 Kubernetes 部署，配合蓝绿发布脚本。

## 四、故障排查

### Q15：文档解析失败怎么办？

**A**：

1. 在文档列表查看失败原因。
2. 常见原因：
   - 文件损坏或加密 PDF。
   - 文件格式不符合预期。
   - Worker 资源不足（内存/CPU）。
3. 可尝试点击 **重新处理**。
4. 多次失败后，请联系管理员查看后端日志。

---

### Q16：搜索响应很慢怎么办？

**A**：

1. 检查是否开启了重排序模型（会增加 100~500 ms 延迟）。
2. 检查知识库文档量级，超过千万级需联系管理员扩容 Milvus。
3. 检查网络延迟，尤其是私有化部署中的跨机房访问。
4. 查看 Grafana 监控，确认后端服务负载。

---

### Q17：生成的答案出现错误信息怎么办？

**A**：

1. 点击 **点踩** 并填写原因。
2. 检查引用来源是否准确。
3. 尝试调整问题表述，使其更具体。
4. 管理员可通过评测工作台分析 bad case，优化分块策略或重排序模型。

---

### Q18：忘记密码无法登录怎么办？

**A**：

1. 如果配置了企业微信/飞书/LDAP SSO，可通过 SSO 登录。
2. 否则联系 Super Admin 重置密码。
3. 系统当前不开放用户自助找回密码（出于企业安全管控要求）。

---

### Q19：审计日志可以导出吗？

**A**：可以。Super Admin 可在 **系统管理** → **审计日志** 中按时间范围、用户、操作类型筛选并导出为 CSV 或 JSON。

---

### Q20：如何反馈产品缺陷或新需求？

**A**：

1. 通过系统内 **帮助与反馈** 提交工单。
2. 联系技术支持邮箱：it-support@company.com。
3. 企业客户可联系专属客户成功经理。
