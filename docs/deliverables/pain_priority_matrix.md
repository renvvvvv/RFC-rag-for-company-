## 1. 矩阵说明

本矩阵从「业务影响程度」和「解决紧迫性」两个维度评估企业 RAG 场景中的核心痛点，用于指导产品迭代与交付优先级。

- **影响程度**：该痛点对业务效率、安全合规、用户体验的影响大小。
- **紧迫性**：该痛点是否在 MVP 阶段必须解决，或是否阻塞上线/签单。
- **优先级**：P0（立即解决）、P1（尽快解决）、P2（长期规划）。

## 2. 痛点清单

| 编号 | 痛点描述 | 影响对象 | 影响程度 | 紧迫性 | 优先级 | 对应产品能力 |
|------|----------|----------|----------|--------|--------|--------------|
| P01 | 企业知识分散在邮件、网盘、IM、本地电脑，检索困难 | 全体员工 | 高 | 高 | **P0** | 统一多模态知识库 |
| P02 | 通用 SaaS RAG 无法满足数据主权和细粒度权限要求 | 安全/合规部门 | 高 | 高 | **P0** | 私有化部署 + 五级权限穿透 |
| P03 | 大模型生成答案存在"幻觉"，缺乏来源引用 | 业务用户 | 高 | 高 | **P0** | 引用溯源 + 检索增强生成 |
| P04 | 图片、视频、Excel 等非文本知识难以被统一检索 | 业务部门 | 高 | 中 | **P0** | 多模态 RAG Pipeline |
| P05 | 传统关键词搜索无法理解语义，多次尝试才能定位信息 | 业务用户 | 中 | 高 | **P0** | 语义检索 + 混合检索 |
| P06 | 权限配置复杂，批量授权和继承规则校验缺失 | KB 管理员 | 中 | 中 | **P1** | 权限管理端增强 |
| P07 | 大视频/音频解析慢，进度不可见 | 业务用户 / 管理员 | 中 | 中 | **P1** | 任务中心 + 进度推送 |
| P08 | 系统全量部署资源占用高，低配环境难以运行 | 运维部门 | 中 | 高 | **P1** | 轻量部署模式 |
| P09 | 默认配置存在安全风险（默认密码、占位告警通道） | 安全/运维部门 | 高 | 高 | **P0** | 安全加固向导 |
| P10 | 缺乏量化的检索与生成效果评估机制 | 算法/管理员 | 中 | 中 | **P1** | 评测工作台 |
| P11 | 答案反馈无法有效回流优化系统 | 算法/管理员 | 中 | 低 | **P1** | 反馈闭环 + Bad Case 分析 |
| P12 | 缺乏与企微/飞书/钉钉等 IM 集成 | 业务部门 | 中 | 低 | **P2** | IM 机器人集成 |
| P13 | 多语言 UI 和移动端适配缺失 | 业务用户 | 低 | 低 | **P2** | i18n + 响应式适配 |
| P14 | 缺乏知识图谱，复杂关系型问题回答能力弱 | 算法/业务 | 中 | 低 | **P2** | 知识图谱模块 |

## 3. 优先级分组

### P0 — 必须解决（影响高且紧迫）

| 编号 | 痛点 | 解决策略 |
|------|------|----------|
| P01 | 知识分散 | 构建统一知识库，支持多模态 ingestion |
| P02 | 权限与数据主权 | 私有化部署 + 五级权限穿透（文件类型/文档/字段/标签/关键词） |
| P03 | 答案幻觉 | 检索增强生成 + 答案引用溯源 + 关键词拦截 |
| P04 | 多模态知识利用 | 文档/Excel/图片/视频/链接 Pipeline |
| P05 | 语义理解不足 | Dense 语义检索 + Sparse 关键词检索 + RRF 混合融合 |
| P09 | 默认配置安全风险 | 强制替换默认密码、密钥，提供安全加固 Checklist |

### P1 — 尽快解决（影响中高，紧迫性中）

| 编号 | 痛点 | 解决策略 |
|------|------|----------|
| P06 | 权限配置复杂 | 批量授权、权限继承可视化、规则冲突检测 |
| P07 | 解析进度不可见 | 任务中心实时进度、WebSocket 推送 |
| P08 | 全量部署资源高 | 提供轻量模式（pgvector + 本地存储 + Redis broker） |
| P10 | 效果难量化 | 评测数据集、检索指标、生成指标、A/B 对比 |
| P11 | 反馈未闭环 | 点赞/点踩收集、Bad Case 聚类、趋势报告 |

### P2 — 长期规划（影响或紧迫性较低）

| 编号 | 痛点 | 解决策略 |
|------|------|----------|
| P12 | IM 集成缺失 | 企微/飞书/钉钉机器人回调与消息推送 |
| P13 | 多语言与移动端 | i18n 框架、响应式布局 |
| P14 | 复杂关系问题 | 知识图谱：实体抽取、关系抽取、图谱检索 |

## 5. 落地建议

### 5.1 MVP 阶段（Sprint 1-6）

优先完成 P0 痛点对应的功能：

- 统一知识库 + 多模态 ingestion
- 私有化部署 + 五级权限穿透
- 检索增强生成 + 引用溯源
- 混合检索（语义 + 关键词）
- 默认安全配置加固

### 5.2 上线后 1 个月内

补齐 P1 痛点：

- 轻量部署模式作为低配环境默认选项。
- 任务中心实时进度与解析失败重试。
- 权限管理端批量授权与继承规则校验。
- 评测工作台数据集导入与指标报告。

### 5.3 长期 roadmap

按业务需求启动 P2 能力：

- 与高频 IM 工具集成，扩大使用场景。
- 引入知识图谱，提升复杂关系型问题回答能力。
- 多语言 UI 与移动端适配，服务跨国/外勤团队。
