### 3.3 API Key 作用域执行（**5/7 通过**）

| 测试 | 结果 |
|---|---|
| search-chat Key → /external/chat | ✅ 通过 |
| search-chat Key → /external/upload | ✅ 403 拦截 |
| search-only Key → /external/chat | ✅ 403 拦截 |
| search-only Key → /external/search | ✅ 通过（合理） |
| search-only Key → /kb/create | ✅ 403 拦截 |
| admin JWT → /api/v1/external/* | ❌ 401（NEW-1 双轨鉴权） |
| admin (L0) → 创建 scope=[*] Key | ❌ 403（NEW-2，符合 L4 才能申请） |

### 3.4 新发现的问题

#### NEW-1: 内部 JWT 与外部 API Key 鉴权双轨
- admin JWT token 调 `/api/v1/external/*` 返回 401 "Invalid API key"
- 外部端点只接受 API Key 鉴权，与内部 JWT 完全隔离
- 影响：admin 在外部端点无法工作（必须创建 API Key）

#### NEW-2: L0 用户无法申请 scope=[*] 的万能 Key
- L0 admin 被拒绝创建全权限 Key
- 符合 L4 才能申请的设计（但用户报告里未提及）

### 3.5 修复建议（50 行代码可完成）

**P0-1: 搜索越权（chat.py:46）**
```python
# 在 _retrieve_and_generate 起始处加：
from app.services.permission_service import permission_service
for kb_id in request.kb_ids:
    if not await permission_service.check_kb_access(current_user.id, kb_id):
        raise PermissionDeniedException("无权访问该知识库")
```

**P0-2: 权限授予越权（permissions.py:grant）**
```python
# 加 1 行：
if not is_admin(current_user):
    raise PermissionDeniedException("需要管理员权限")
```

**P0-3: 文档重处理（documents.py:reprocess）**
```python
# 在 get_document 之后加 1 行：
await _require_kb_access(db, current_user, doc.kb_id)
```

**NEW-1 建议**：在 chat.py 接受 JWT 时也允许 API Key 鉴权，或在外部端点路由加 fallback。
